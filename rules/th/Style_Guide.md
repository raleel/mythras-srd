# ชื่อคู่มือสไตล์ (h1)

ใช้หัวข้อ h1 (`# Heading Title`) สำหรับชื่อบทหลัก หัวข้อ hx ทั้งหมดจะปรากฏในแถบด้านข้างเป็นบุ๊กมาร์ก

หากชื่อไม่ใช่หัวข้อ h1 แถบด้านข้างจะมีข้อผิดพลาดในการทำซ้ำที่แสดงหน้าและหัวข้อ 'ชื่อ' h2

---
## หัวข้อส่วน (h2)

ใช้หัวข้อ h2 (`## Section Heading`) ที่นำหน้าด้วยเส้นแบ่ง (`---`) สำหรับแต่ละส่วนหลักใหม่

### หัวข้อย่อยเพิ่มระดับ h-level ขึ้น 1

ใช้หัวข้อ h3 (`### Subsection Heading`) สำหรับแต่ละหัวข้อย่อยใหม่ และหัวข้อ h4 (`#### Sub-Subsection Heading`) สำหรับแต่ละหัวข้อย่อยย่อยใหม่


### ลำดับ h-level ของหัวข้อย่อย: 2 > 3 > 4

หัวข้อย่อยจะสลับกันในแถบด้านข้างหากลำดับ h-level ไม่ได้รับการจัดการอย่างถูกต้อง

ตัวบ่งชี้หัวข้อควรอ่านเพิ่มขึ้นหนึ่งหรือไม่มีเลย

h2 ตามด้วย h2 หรือ h3 นั้นดี h3 ตามด้วย h3 หรือ h4 ก็ดีเช่นกัน

### ข้อผิดพลาดของหัวข้อย่อย
h2 ตามด้วย h4 ทำให้เกิดปัญหา แถบด้านข้างจะวางหัวข้อย่อยไว้เหนือหัวข้อในสารบัญ

### การลดระดับ h-level

สิ่งนี้ไม่น่าจะเป็นปัญหา เช่น h4 เป็น h3 หรือ h2 ก็ดี

## หลักการจัดรูปแบบ

รูปแบบต่อไปนี้เป็นมาตรฐานสำหรับ SRD _Imperative_ เหล่านี้ในรูปแบบ Markdown

### ตัวหนา, ตัวเอียง และเส้นแบ่ง

- ใช้เครื่องหมายดอกจันคู่ (`**Bold**`) สำหรับข้อความ **ตัวหนา**
- ใช้เครื่องหมายขีดล่าง (`_Italic_`) สำหรับข้อความ _ตัวเอียง_
  - _หมายเหตุ: หลีกเลี่ยงการใช้เครื่องหมายดอกจันเดี่ยวสำหรับตัวเอียง!_
- ใช้เครื่องหมายแบ็กสแลช (`\_`) เพื่อ 'หลีกเลี่ยง' อักขระ: \_ไม่เอียง\_
- ใช้เครื่องหมายอัญประกาศมุม (``` `code` ```) เพื่อสร้าง `code blocks`
- ใช้เครื่องหมายยัติภังค์สามขีด (`---`) เพื่อแทรกเส้นแบ่งแนวนอน

### หลักการสำหรับตัวหนาและตัวเอียง

พยายามยึดตามรูปแบบต่อไปนี้...

#### ชื่อเรื่องตัวหนา

ใช้ชื่อเรื่องตัวหนาเมื่ออธิบายคาถา, เอฟเฟกต์ ฯลฯ
- **ชื่อเวทมนตร์:** คำอธิบาย

#### ชื่อเรื่องตัวเอียง

ใช้ชื่อเรื่องตัวเอียงสำหรับ 'รายการย่อย' ของคาถาประเภทต่างๆ ฯลฯ
- **เวทมนตร์:** คำอธิบาย
  - **_เวทมนตร์รูปแบบ A:_** คำอธิบาย
  - **_เวทมนตร์รูปแบบ B:_** คำอธิบาย

#### การอ้างอิงชื่อเรื่องตัวเอียง

ใช้ชื่อเรื่องตัวเอียงเมื่ออ้างอิงถึงหนังสือ เช่น _Mythras Imperative_ หรือกฎหลักของ _Mythras_

## การจัดรูปแบบรายการ

### รายการที่ไม่มีลำดับ / จุดหัวข้อ

- รายการที่ไม่มีลำดับสร้างด้วยเครื่องหมายยัติภังค์
- รายการที่ไม่มีลำดับสร้างด้วยเครื่องหมายยัติภังค์
  - เพิ่มระดับการเยื้องด้วย 2 ช่องว่างหรือแท็บ

### รายการที่มีลำดับ / จุดตัวเลข

1. รายการตัวเลขสร้างด้วยตัวเลขและจุด/มหัพภาค
2. รายการตัวเลขสร้างด้วยตัวเลขและจุด/มหัพภาค
- ผสมกับจุดหัวข้อ
	- และการเยื้อง
3. ผสมตัวเลขกลับเข้ามา
   - และจุดหัวข้อ

### รายการที่มีหัวข้อ

สำหรับรายการหัวข้อย่อยที่เกี่ยวข้องกัน ให้ทำหัวข้อและเครื่องหมายโคลอนเป็นตัวหนา
- **ตัวอย่าง 1:** บลา บลา บลา...
- **ตัวอย่าง 2:** บลา บลา บลา...

### รายการที่มีหัวข้อตัวเอียง

สำหรับรายการหัวข้อย่อยที่เกี่ยวข้องกัน ให้ทำหัวข้อและเครื่องหมายโคลอนเป็นตัวหนาและตัวเอียง
- **_ตัวอย่าง 1:_** บลา บลา บลา...
- **_ตัวอย่าง 2:_** บลา บลา บลา...

## ตาราง

ใส่หัวข้อย่อยให้ตารางเพื่อให้ปรากฏเป็นบุ๊กมาร์กในแถบด้านข้าง โดยเยื้องหนึ่งระดับจากส่วนหลัก

~ทำให้แถวหัวข้อเป็นตัวหนา~
การทำให้แถวหัวข้อเป็นตัวหนาจริง ๆ แล้วจะแสดงผลเป็นข้อความปกติ CSS เริ่มต้นทำให้แถวนั้นเป็นตัวหนาอยู่แล้ว

### การจัดแนวข้อความในตาราง

ใช้แถวที่ 2 สำหรับการจัดแนวเนื้อหา: `:--`, `:-:`, หรือ `--:`
สิ่งนี้จะส่งผลต่อทั้งคอลัมน์

#### เค้าโครงพื้นฐานของตาราง

| **คอลัมน์ 1** | **คอลัมน์ 2** |
| :-: | :-- |
| 1 | อัลฟ่า |
| 2 | เบต้า |
| 3 | แกมม่า |

### หลักการจัดแนวข้อความ

- ใช้การจัดกึ่งกลางสำหรับตัวเลขหรือรายการค่าที่คล้ายกัน
- ใช้การจัดซ้ายสำหรับประโยคของข้อความ

#### ตัวอย่างตารางการจัดแนว

| **ตัวเลข** | **คำอธิบาย** |
| :-: | :-- |
| 1 | ข้อความอธิบาย |
| 2 | ข้อความอธิบาย |
| 3 | ข้อความอธิบาย |

#### ตัวอย่าง: ตารางความรุนแรงของไฟ

| **ความรุนแรง** | **ตัวอย่าง** | **เวลาในการจุดไฟ** | **ความเสียหาย** |
| :-: | :-- | :-: | :-: |
| 1 | เทียน | 1d4 | 1d2 |
| 2 | คบไฟ | 1d3 | 1d4 |
| 3 | กองไฟ | 1d2 | 1d6 |
| 4 | ไฟไหม้เต็มห้อง | 1d2 | 2d6 |
| 5 | ลาวาภูเขาไฟ | ทันที | 3d6 |

## รายการเวทมนตร์และพลัง

เริ่มต้นส่วน 'ใหม่' ด้วยหัวข้อ h2 สำหรับรายการคาถา, พลัง ฯลฯ ที่มีอยู่ จากนั้นใช้ h3 สำหรับแต่ละรายการ

คาถาและพลังมักจะนำเสนอในรายการแบบเรียบเดี่ยวตามประเภทเวทมนตร์ (เวทมนตร์พื้นบ้าน, เทวนิยม, ฯลฯ) แต่หากรายการนั้นต้องการส่วนย่อยของตัวเอง เช่น ความเชี่ยวชาญของโรงเรียนหรือลัทธิ ให้เพิ่มระดับหัวข้ออย่างเหมาะสม ตัวอย่างเช่น h2 สำหรับรายการปาฏิหาริย์เทวนิยม, h3 สำหรับลัทธิ: มิธราส และ h4 สำหรับปาฏิหาริย์ของลัทธิมิธราสแต่ละรายการ

โดยเฉพาะอย่างยิ่งในกรณีของคาถา แต่ในการอธิบายใดๆ ที่ใช้คุณสมบัติคำสำคัญ ให้แสดงเป็นข้อความตัวเอียง

ข้อความอธิบายและกฎพิเศษใช้ข้อความปกติ

### การจัดรูปแบบรายการคาถา

```
## h2 List name

### List item

_Attributes_

Description text.
```

## บล็อกค่าสถานะสิ่งมีชีวิต
เริ่มต้นส่วน 'ใหม่' ด้วยหัวข้อ h2 สำหรับรายการ บล็อกค่าสถานะสิ่งมีชีวิตใช้ตารางและมีความซับซ้อนมากขึ้นในด้านการจัดรูปแบบและข้อกำหนดพื้นที่ เพื่อความชัดเจน หลีกเลี่ยงส่วนย่อยและเพิ่มเส้นแบ่งระหว่างรายการ

แต่ละตัวอย่างที่แสดงรายการควรมีดังต่อไปนี้...

- **ชื่อสิ่งมีชีวิต:** เส้นแบ่งตามด้วยหัวข้อ h3 สำหรับชื่อสิ่งมีชีวิต

- **รายการ MEG:** ลิงก์ภายนอกไปยังสิ่งมีชีวิตที่ระบุบน [Mythras Encounter Generator](https://mythras.skoll.xyz/)

- **รายละเอียดหลัก:** ชื่อ/ประเภทสิ่งมีชีวิต, คุณสมบัติ, ทักษะ และคำอธิบายพิเศษหรือบันทึกตัวละครตามความจำเป็น

## ข้อผิดพลาดของชุมชน

### พิมพ์ผิดและความสับสน

การแก้ไขเพื่อ 'ปรับปรุง' ข้อความของผู้เขียนควรน้อยที่สุด แต่การพิมพ์ผิดและข้อผิดพลาดอื่น ๆ ควรแก้ไขในบรรทัดโดยไม่ต้องมีเครื่องหมายบอก

### แผงข้อมูลข้อผิดพลาด

ในกรณีที่กฎกำกวมเกินไป, ไม่สมบูรณ์, หรือผิดพลาดอย่างง่ายๆ ให้เพิ่มหมายเหตุข้อผิดพลาดของชุมชนที่สามารถยุบได้

อธิบายปัญหาด้วยตัวเอียง โดยใช้ตัวเอียงหนาสำหรับชื่อปัญหาเท่านั้น - _ไม่มีหัวข้อ h(x)!_ - เพื่อไม่ให้เกิดรายการในแถบด้านข้างที่สับสน

หากมีการเพิ่มข้อความทดแทน ให้ใส่ไว้ในเครื่องหมายคำพูดคู่ด้วยตัวเอียงหนา โดยนำหน้าด้วย "_Community Errata:_" ในข้อความตัวเอียงแบบปกติ

กฎเพิ่มเติม หากจำเป็น ควรอธิบายด้วยข้อความปกติ

## คำเตือนเบ็ดเตล็ด

### บรรทัดว่าง / การขึ้นบรรทัดใหม่

เว้นบรรทัดว่างระหว่างรายการประเภทต่างๆ หัวข้อ, เส้นแบ่ง, ตาราง ฯลฯ สิ่งเหล่านี้จะไม่ปรากฏให้เห็นในโหมดอ่าน แต่การไม่มีอาจทำให้เกิดปัญหาการจัดรูปแบบ

### ข้อความหลายย่อหน้าที่เป็นตัวหนาหรือตัวเอียง

แต่ละย่อหน้าจะต้องถูกทำเครื่องหมายว่าเป็นข้อความตัวหนาหรือตัวเอียง บรรทัดใหม่ระหว่างย่อหน้าจะทำให้การจัดรูปแบบเสียไป ทิ้งเครื่องหมายการจัดรูปแบบที่ถูกทอดทิ้งไว้

### การสะกดคำแบบสากล

อำนาจที่ถดถอยและน่าตำหนิได้เห็นสมควรที่จะลบการสะกดคำแบบบริติชทั้งหมดออก เพื่อใช้การสะกดคำแบบสหรัฐฯ ที่อ่อนแออย่างชัดเจนแทน โปรดระวังคำภาษาบริติชที่ลงท้ายด้วย -OUR: _armour_, _favour_ เป็นต้น
