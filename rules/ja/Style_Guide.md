# スタイルガイドのタイトル (h1)

メインの章タイトルにはh1見出し（`# Heading Title`）を使用します。すべてのhx見出しは、サイドバーにブックマークとして表示されます。

タイトルがh1見出しでない場合、サイドバーにページとh2「タイトル」見出しが重複して表示されるエラーが発生します。

---
## セクション見出し (h2)

新しい主要セクションごとに、区切り線（`---`）で区切られたh2見出し（`## Section Heading`）を使用します。

### サブセクション見出し hレベルを1増やす

新しいサブセクションにはh3見出し（`### Subsection Heading`）を、新しいサブサブセクションにはh4見出し（`#### Sub-Subsection Heading`）を使用します。

### サブセクションのh進行：2 > 3 > 4

h進行が正しく処理されていない場合、サブ見出しがサイドバーで混同されます。

見出しマーカーは1つ増えるか、何も増えないかのどちらかであるべきです。

h2の後にh2またはh3が続くのは問題ありません。h3の後にh3またはh4が続くのも問題ありません。

### サブセクションのエラー
h2の後にh4が続くと問題が発生します。サイドバーは、サブ見出しを目次内の見出しよりも上に配置します。

### hレベルを減らす

これは問題にならないようです。例：h4からh3またはh2への変更は問題ありません。

## 書式設定の規則

以下のスタイルは、これらの_Imperative_ SRDのMarkdown形式における標準です。

### 太字、斜体、区切り線

- **太字**のテキストには二重アスタリスク（`**Bold**`）を使用します。
- _斜体_のテキストにはアンダーバー（`_Italic_`）を使用します。
   - _注：斜体に単一のアスタリスクを使用するのは避けてください！_
- 文字を「エスケープ」するにはバックスラッシュ（`\_`）を使用します: \_Not Italic\_。
- `code blocks`を作成するには角度のある引用符（``` `code` ```）を使用します。
- 水平区切り線を挿入するには3つのハイフン（`---`）を使用します。

### 太字と斜体の規則

以下のパターンに沿うようにしてください...

#### 太字のタイトル

呪文や効果などを説明する際には、太字のタイトルを使用します。
- **魔法のタイトル:** 説明。

#### 斜体のタイトル

バリアント呪文などの「サブリスト」には斜体のタイトルを使用します。
- **魔法:** 説明。
  - **_魔法バリアント-A:_** 説明。
  - **_魔法バリアント-B:_** 説明。

#### 斜体のタイトル参照

_Mythras Imperative_ やコアの_Mythras_ ルールセットのような本を参照する際には、斜体のタイトルを使用します。

## リストの書式設定

### 順序なしリスト / 箇条書き

- 順序なしリストはハイフンで作られます。
- 順序なしリストはハイフンで作られます。
  - 2つのスペースまたはタブでインデントレベルを上げます。

### 順序付きリスト / 番号付きリスト

1. 番号付きリストは数字とピリオドで作られます。
2. 番号付きリストは数字とピリオドで作られます。
- 箇条書きと混在
	- そしてインデント
3. 番号を再び混ぜる
   - そして箇条書き

### 見出し付きリスト

関連するサブ見出しのリストには、タイトルとコロンを太字にします。
- **例1:** ほにゃらら...
- **例2:** ほにゃらら...

### 斜体見出し付きリスト

関連するサブ見出しのリストには、タイトルとコロンを太字かつ斜体にします。
- **_例1:_** ほにゃらら...
- **_例2:_** ほにゃらら...

## 表

表にはサブヘッダーを付け、サイドバーに親セクションから一段階インデントされたブックマークとして表示されるようにします。

~見出し行を太字にしてください。~
ヘッダー行を太字にしても、実際には通常のテキストでレンダリングされます。デフォルトのCSSでその行が太字になっています。

### 表のテキスト配置

コンテンツの配置には2行目を使用します：`:--`、`:-:`、または`--:`。
これは列全体に影響します。

#### 表の基本的なレイアウト

| **列1** | **列2** |
| :-: | :-- |
| 1 | アルファ |
| 2 | ベータ |
| 3 | ガンマ |

### テキスト配置の規則

- 数字や類似の値のリストには中央揃えを使用します。
- テキストの文には左揃えを使用します。

#### 配置例の表

| **数字** | **説明** |
| :-: | :-- |
| 1 | 説明テキスト |
| 2 | 説明テキスト |
| 3 | 説明テキスト |

#### 例：火炎強度表

| **強度** | **例** | **着火時間** | **ダメージ** |
| :-: | :-- | :-: | :-: |
| 1 | ろうそく | 1d4 | 1d2 |
| 2 | たいまつ | 1d3 | 1d4 |
| 3 | キャンプファイヤー | 1d2 | 1d6 |
| 4 | 部屋いっぱいの大火災 | 1d2 | 2d6 |
| 5 | 火山溶岩 | 即時 | 3d6 |

## リンク

SRDではリンクの多くの用途がありますが、ここではいくつかの主要なケースのみを扱います。

### リンクの書式設定

リンクの書式設定は角括弧で囲まれたテキストで行われます。表示されるリンクテキストには角括弧を、リンク先のURLには丸括弧を使用します：`[Link Text](URL)`。

これらのドキュメントで最も一般的に使用される外部リンクは、[Mythras Encounter Generator](https://mythras.skoll.xyz)でしょう。
```
[Mythras Encounter Generator](https://mythras.skoll.xyz)
```

### 外部リンクと内部リンク

外部リンクはインターネット経由で他のページに移動し、`http://example.com` URLを必要とします。また、デフォルトでは_常に_新しいブラウザタブで開きます。

文書を閲覧中に複数のタブが乱雑になるのを避けるため、内部リンクとブックマークはGitHubリポジトリで表示されるMarkdownファイルを参照する相対URLで形成されます。これらの内部リンクは、新しいタブではなく、リンクされたセクションに素早くスクロールします。

#### 内部リンクの例

- **章リンク：** メイン（h1）セクションの先頭を指します。例：[戦闘](0005_Combat.md)の章。

  - リンク形式：`[Combat](0005_Combat.md)`

- **サブセクションリンク：** 上記のように文書を参照しながら、`?id=`をその直後にリンク先の見出しとともに追記することで、章内の見出しを指します。例：[跳躍攻撃](0005_Combat.md?id=leaping-attacks)

  - リンク形式：`[Leaping Attacks](0005_Combat.md?id=leaping-attacks)`

#### サブセクションへの外部リンク

外部ドキュメントの既知のセクション、例えば別のSRDの特定の表を参照する機会があるかもしれません。書式設定は上記のサブセクションリンクと同じですが、外部のHTTP URLを使用します。

例として、[Classic Fantasy Imperative SRD](https://cfi-srd.mythras.net/)からの[帝国単位からメートル法への換算表](https://cfi-srd.mythras.net/#/Appendix_B_Conversion_Tables?id=imperial-weights-and-measures-to-metric-conversion)を挙げます。

```
[Imperial to Metric conversions table](https://cfi-srd.mythras.net/#/Appendix_B_Conversion_Tables?id=imperial-weights-and-measures-to-metric-conversion)
```

## 魔法と能力のリスト

利用可能な呪文、能力などのリストには、h2見出しで「クリーンな」セクションを開始し、各リストアイテムにはh3を使用します。

呪文と能力は通常、魔法タイプ（民間魔法、信仰、など）ごとの単一のフラットなリストで提示されますが、リストが学派やカルトの専門分野のような独自のサブセクションを必要とする場合、見出しレベルを適切に上げてください。例えば、信仰の奇跡リストにはh2、カルト：ミトラスにはh3、各ミトラスカルトの奇跡にはh4を使用します。

特に呪文の場合ですが、キーワード属性を使用するすべての説明では、それらを斜体テキストでリストします。

説明テキストと特殊ルールは通常のテキストを使用します。

### 呪文リストの書式設定

```
## h2 List name

### List item

_Attributes_

Description text.
```

<details>

<summary>例：書式設定された呪文リスト（ブロック引用） </summary>

---
> ## 呪文の説明
>
> ### 回避
> _即時、遠隔_
>
> 回避は、範囲内の別の呪文を打ち消すために使用されます。回避は、カウンター魔法の反応アクションを使用して、攻撃呪文を無力化するために反応的に唱えることができます。

</details>

## 乗り物のステータスブロック
リストにはh2見出しで「クリーンな」セクションを開始し、カテゴリとリストアイテムには通常通りhレベルを上げます。

リストアップされた各例は、太字の項目タイトルを持つ箇条書きリストとしてステータスと説明を提示します。

### 乗り物のステータスブロックの書式設定

```
## h2 List name

### Category

#### List Item

- **Stat 1:**  [Value]

- **Stat 2:**  [Value]

- **Description:** Descriptive text.
```

## クリーチャーステータスブロック
リストにはh2見出しで「クリーンな」セクションを開始します。クリーチャーステータスブロックは、書式設定とスペース要件の点でより複雑です。明瞭さのために、サブセクションを避け、リストアイテム間に区切り線を追加してください。

リストアップされた各例には、以下が含まれるべきです...

- **クリーチャーのタイトル：** 区切り線の後にクリーチャー名のh3ヘッダー。

- **MEGエントリ：** [Mythras Encounter Generator](https://mythras.skoll.xyz/)の指定されたクリーチャーへの外部リンク。

- **基本詳細：** クリーチャー名/タイプ、属性、スキル、および必要に応じて特別な説明やキャラクターノート。

- **武器詳細：** 武器名と以下の属性の列を含む簡略化された説明：サイズ（ここに力を含める）、間合い（ここに射程を含める）、ダメージ、装甲点/ヒットポイント、および特殊ルール。

- **ヒット箇所と装甲：** 位置表はクリーチャーの解剖学的構造によって異なりますが、装甲とダメージを追跡する手段として、従来のヒト型生物の場合でも含めるべきです。

### クリーチャーステータスブロックの書式設定

```
## Bestiary/List Name

Descriptive text. 

---
### Example Creature
[MEG Entry](https://mythras.skoll.xyz/enemy_template/8626/)

| Creature Name | Example Creature  |
| :-- | :-- |
| Key Attributes | STR 11, CON 14, SIZ 11, DEX 11, INT 11, POW 11, CHA 7 |
| Other notes | Values and details | 
```

## コミュニティエラッタ

### タイプミスと混乱

著者のテキストを「クリーンアップ」するための編集は最小限に抑えるべきですが、タイプミスやその他の間違いは、目印なしにインラインで変更すべきです。

### エラッタ情報パネル

特定のルールがあまりにも曖昧、不完全、または単純に間違っている場合、折りたたみ可能なコミュニティエラッタノートを追加してください。

問題のタイトルには太字の斜体のみを使用し（_h(x)ヘッダーは使用しないでください！_）、混乱を招くサイドバーエントリを作成しないように、問題を斜体で説明してください。

置換テキストが追加される場合は、二重引用符で囲み、太字斜体にし、通常のウェイトの斜体で「_Community Errata:_」を先行させてください。

必要に応じて、追加ルールは通常のウェイトのテキストで提示されるべきです。

#### エラッタ書式設定テンプレート

```
<details>

<summary>* Community Errata</summary>

_**Issue Title**_

_Community Errata:_
_**"Replacement text if required."**_

_Explanation text stating how the rule as written causes problems._

</details>
```

#### エラッタの例

<details>

<summary>* コミュニティエラッタ</summary>

_**登攀**_

_Community Errata:_
_**「着用している防具は、着用している防具点の平均に等しい量だけ登攀者を妨げる。これは、各部位の着用防具点を合計し、その数値を部位の数で割ることで算出する。」**_

_書かれたルールはすぐに問題を引き起こします。6APの防具を1つ着用するだけで、人間は移動力0になり、鋼鉄製のヘルメットを着用し、他には何も身につけていない人間は、全く登攀できないほど過負荷になるでしょう。これは不合理に制限的であるため、上記の編集版が提案されています。元の著者が書かれたルールを改訂した場合、これは更新される予定です。_

</details>

<details>

<summary>* コミュニティエラッタ</summary>

_**歩行速度：歩く、走る、全力疾走**_

_『Mythras Imperative』の公開版には、異なる歩行速度で移動するための明示的なルールがありませんが、それらはMythrasのコア・ルールブックでカバーされています。上記の歩行速度の乗数は、コミュニティエラッタの一部として追加されました。_

_必要に応じて、歩行速度は次のように扱われるべきです：_

- _歩行よりも速い歩行速度は移動力に倍率を適用します：走る場合は3倍、全力疾走の場合は5倍。_

- _走るまたは全力疾走の歩行速度中、キャラクターは攻撃や呪文の詠唱といったほとんどの先制行動を試みることができません。いくつかの例外として…_
  - _突撃ルールを使用する際の攻撃。_
  - _スカーミッシング特性を持つ遠隔武器の発射。_
- _一般的なルールとして、許可された行動は、走る場合は1段階難しく、全力疾走の場合は2段階難しく実行されるべきです。_

</details>

## その他の警告

### 空行 / 改行

異なる種類のエントリの間には空行を置いてください。ヘッダー、区切り線、表など。これらは読み取りビューでは表示されませんが、空行がないと書式設定の問題を引き起こす可能性があります。

### 太字または斜体の複数段落テキスト

各段落は太字または斜体テキストとしてマークする必要があります。段落間の改行は書式設定を破棄し、孤立した書式設定マークを残します。

### 国際的なスペル

退行的で嘆かわしい権力者たちは、客観的に劣る米国版を支持して、すべての英国式スペルを削除することを適切と見なしました。英国式の-OURの単語に注意してください：_armour_（装甲）、_favour_（好意）など。
