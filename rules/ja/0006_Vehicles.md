# 乗り物

『_Mythras Imperative_』における乗り物は、半抽象的な方法で扱われます。本質的に、これらはキャラクターをA地点からB地点へ移動させるための背景ツールであり、詳細な記述や統計はほとんど必要ありません。ただし、チェイスや戦闘状況で乗り物を表現する必要がある場合、ゲームマスターは乗り物の詳細を要求することがあります。

このセクションのルールは、ほとんどあらゆる種類の乗り物を抽象化された方法で表現することを可能にし、柔軟性を許容し、ストーリーテリングを強化することを意図しています。スターシップや宇宙船は、同様の原則に基づいていますが、少し異なります。

---
## 統計

すべての乗り物は、その構造点、外殻、速度、システム、特性、そしてシールド（設定に存在する場合）によって記述されます。

- **[サイズ](0006_Vehicles.md?id=vehicle-size-table):** すべての乗り物には6つのサイズ評価のいずれかがあり、それが他の特性の値または範囲を決定します。固有のサイズが大きく、宇宙の過酷な環境のため、スターシップには独自の範囲の値があります。
- **[外殻](0006_Vehicles.md?id=terrestrial-hulls):** 装甲点を表します。
- **[構造点](0006_Vehicles.md?id=terrestrial-hulls):** 乗り物の命中点を表します。
- **[シールド](0006_Vehicles.md?id=shields):** 高度なエネルギー操作技術がある設定では、乗り物を保護するシールドが存在する場合がありますが、持続的な攻撃を受けるとアブレーション（消耗）することがあります。
- **[システム](0006_Vehicles.md?id=systems):** 乗り物の生命線となるコンポーネント、動力機関、乗員区画などを表します。
- **[速度](0006_Vehicles.md?id=speed):** すべての乗り物には基本的な速度評価があります。
- **[特性](0006_Vehicles.md?id=traits):** 乗り物には1つ以上の特性がある場合があります。『_Mythras Imperative_』のルールでクリーチャーに割り当てられる能力や戦闘スタイル特性と同様に、特性は乗り物の性質を反映した追加のゲーム記述と利点を提供します。
- **[武器](0006_Vehicles.md?id=weapons):** 特性と同じく、設定が許す限り、乗り物には武器が装備されている場合があります。いくつかの提案された例と値が以下に詳述されています。

### 外殻のサイズ、種類、構造点

乗り物のサイズと目的は、その外殻の厚さや内部構造の弾力性に影響を与えます。例えば、モーターサイクルは速度と敏捷性のために作られているため、最小限の外殻しか持ちません。ランド・アイアンクラッド（蒸気機関またはディーゼルエンジンで動く巨大な履帯または車輪付きの装甲獣）は、戦場用に設計されたはるかに大きな乗り物であるため、はるかに大きな外殻を持ちます。外殻の比較サイズ（および通常、武器からどれだけの命中弾に耐えられるか）は、以下の乗り物サイズと外殻の表にまとめられています。

乗り物がダメージを受けた場合、入ってくるダメージはその外殻値によって軽減されます。残ったダメージは乗り物の構造点から差し引かれ、貫通ダメージに等しいパーセンテージの確率でシステムに影響を与えます。乗り物の構造点がゼロになった場合、それは完全に破壊されるか、ひどく損傷してスクラップにされるかのいずれかです。

### シールド

ディフレクターシールドは、装備できる乗り物に追加の防御層を提供します。シールドは、そのアブレーション（消耗）性質のため、外殻とは少し異なる働きをします。入ってくるダメージがシールド値以下の場合、それはすべてブロックされます。ダメージがシールド値を超える場合、その超過分だけシールドの強度が減少します。

_例えば、兵員輸送車はエネルギーシールド強度が12ポイントあります。輸送車がレーザーで11ダメージを受けた場合、それは単に貫通せず、追加の効果はありません。一方で、輸送車が15ダメージを与える爆発を受けた場合、超過分の3ポイントによってシールド強度は9に低下します。_

ディフレクターシールドの強度がゼロになると、それは崩壊します。残ったダメージは外殻に引き継がれます。

### システム

乗り物が外殻を貫通するダメージを受けるたびに、その貫通ダメージに等しい確率で生命線となるシステムに影響が及ぶ可能性があります。

_例えば、スピードボート（外殻3、構造点24の乗り物）に10ダメージが適用されたとします。外殻は装甲として働き、ダメージを7に軽減します。このダメージは構造点24から減算され、乗り物のシステムの一つにダメージを与える7%のチャンスがあります。_

システムが耐えられるダメージは、乗り物のサイズに純粋に基づいています。これは実際のダメージ量ではなく、システムが受けられるヒット数でモデル化されています。小型乗り物のシステムへの1ヒットは、そのシステムを即座に破壊します。中型乗り物のシステムを破壊するには2ヒット、大型乗り物には3ヒットが必要です。以下に示す[システムダメージ表](0006_Vehicles.md?id=system-damage-table)を参照してください。

損傷しているがまだ破壊されていないシステムは、受けたヒット数に比例して機能が失われます。これは、システムタスクへのパーセンテージ削減または難易度段階ペナルティのいずれかです。特定の影響は、サブシステムが何を制御しているかに基づき、以下の[システムコンポーネントダメージ表](0006_Vehicles.md?id=system-component-damage-table)に詳述されています。どのシステムがヒットしたかによっては、影響は壊滅的になる可能性があります。

_例えば、ランド・アイアンクラッド（システムヒット5の巨大な乗り物）が駆動装置に2回ヒットした場合、その動力は40%失われ、低速速度は2段階低下して「のろい」になります。同様に、大型貨物船（システムヒット3）がエンジン/燃料システムに2回ヒットした場合、速度に加えて電子システムの使用にも2段階のペナルティを受けます。3回目のヒットはエンジン/燃料システムとその船を破壊します。_

### 速度

速度、加速度、抗力、慣性などの物理学は複雑な問題であるため、いくらかの簡略化が必要です。そのため、乗り物には抽象的な速度評価が与えられます。速度特性は、ある乗り物が別の乗り物を追い越せるか、追いつけるかを大まかに判断するためにのみ使用されるべきです。

速度評価は、乗り物の名目上の運用速度を表します。緊急時には、エンジンへの摩耗や損傷のリスクを伴いますが、この性能を1段階上げることができます。すべての乗り物の種類で同じ表が使用されますが、同じクラスの乗り物同士のみを比較するよう常識を適用すべきです。例えば、スターシップとスポーツカーがどちらも「高速」という同じ加速率を持つ場合でも、宇宙船の方が比較にならないほど速いことは明らかです。

乗り物の速度を標準値以上に上げることは、運転判定に成功すれば可能です。通常、この追加速度は短期間しか持続できません。1d6分を超えると、乗り物は標準速度に戻すか、動力機関に損傷を与えるリスクを冒さなければなりません。クリティカルな運転判定では、高速期間を1d12分に延長できます。

運転手は乗り物の速度を2段階上げようと試みることができますが、これには超人的な運転判定が必要であり、乗り物がこの速度を維持できる最大時間は1d3分です。クリティカルサクセスでも増加した速度の持続時間は延長されません。

### 特性

乗り物が持てる特性の最大数は、外殻のサイズと種類によって異なります。

すべての乗り物には固有の特性があります。それは、主に設計された移動モードです。したがって、航空機は本質的に「空中」特性を持ち、ボートは「水上」特性を持つなどです。この固有の特性は、利用可能な特性の数にはカウントされません。ただし、乗り物が2つ目の移動モードを含む場合（例えば、飛行する潜水艦など）、その場合、「空中」特性は利用可能な特性の1つとして取得する必要があります。

設定の技術レベルによっては、利用できない特性もあります。ゲームマスターは、どの特性が適用できるかについて、独自の判断を下す必要があります。

#### 空中

この乗り物は大気圏内飛行が可能です。

#### 全地形対応

この特性は地上車両で使用されなければなりません。この乗り物は、履帯、高度なサスペンションと駆動システム、または強力な駆動差動装置を使用して、荒れた、困難な、急な地形を横断できます。

#### 掘削

この乗り物は地面を掘り進みトンネルを掘るように設計されています。その外殻値×5メートルに等しい最大深度に達することができます。

#### カモフラージュ

この乗り物はカモフラージュ塗装または模倣センサーを備えており、周囲の環境に溶け込むことができます。乗り物を視覚的に発見する試みは、1段階難しくなります。

#### 貨物

この乗り物は乗客ではなく貨物を運搬するために特別に設計されています。客室は最小限に抑えられ、乗り物の残りの利用可能な空間は、適切な区画、ツール、さらには環境調節（生鮮食品輸送用の冷凍区画など）を備えた貨物収容に充てられます。貨物を積載すると、乗り物の速度は通常の外殻サイズが許容する速度より2段階低くなります。

#### 運搬船

この乗り物は、民間または軍事目的の、小型で少人数で運用される、またはリモート制御可能な車両を運搬するために特別に設計されています。この特性を持つ乗り物は、少なくとも巨大でなければなりません。搭載される小型艇の数は、任務の種類や目的によって異なりますが、最低10隻から、乗り物の構造点評価に等しい数まで可能です。

#### 建設

この乗り物は重建設作業のために装備されています。この特性を持つすべての乗り物は「全地形対応」特性も持ちますが、速度は通常の外殻サイズが許容する速度より2段階低くなります。この乗り物には、クレーンジブ、ショベル、ダンプカートなどの作業に適したツールが装備されています。

#### EMP耐性

この乗り物は電磁パルス攻撃からシールドされています。

#### 強化された性能

この乗り物の速度評価は、そのサイズの最大値より1段階高くなります。

#### 射出座席

緊急時には、射出座席が乗員を数百メートル上空に射出し、その後パラシュートを展開して安全な着地を可能にします。パラシュートはオプションです。ヘリコプターには推奨されません（カモフ Ka-52 アリゲーターは注目すべき例外です）。

#### FTL（超光速）

宇宙船専用。この船には超光速エンジン（ハイパードライブ、ワープドライブ、ワームホールジェネレーターなど）が搭載されています。FTLドライブには独立した速度評価があり、本質的にスターシップが1パーセク（3.26光年、または31兆キロメートル、または19兆マイル）を横断できる速度を以下のように測定します。

##### FTL速度評価表

| 速度評価 | 1パーセク横断の係数 |
| :------- | :----------: |
| のろい   |      10      |
| 鈍い     |      8       |
| 遅い     |      7       |
| 並み     |      6       |
| 穏やか   |      5       |
| 中速     |      4       |
| 高速     |      3       |
| 速い     |      2       |
| 俊足     |      1       |

係数の時間は、設定のFTL開発がどの程度進んでいるかによって、ゲームマスターが決定します。例えば、基本係数が1日の場合、鈍いFTLドライブを持つ船は10日かけて航海を完了しますが、俊足のFTLドライブを持つ船は1日で航海を完了します。FTL技術が著しく非効率な場所では、係数の時間は日ではなく月単位で測定されるかもしれません。

船は往復航海のための燃料貯蔵、または目的地での補給手段を持っていると仮定されます。

設定にもよりますが、あらゆるサイズのスターシップにFTLドライブを搭載できると仮定されます。

FTLドライブは、恒星から1.5天文単位（約2億2500万キロメートル、または2億7000万マイル）以内では起動できません。これは、船がFTLシステムを起動する前に、火星の距離まで亜光速ドライブを使用する必要があることを意味します。

#### 地上車両

この乗り物は地上移動が可能です。

#### ホバー

この乗り物は、エアクッションまたは反重力リパルサーを使用して地面を浮上します。

#### 豪華

この乗り物は、特定のレベルの贅沢さで乗客を運ぶために特別に設計されています。余分な貨物スペースは、豪華な客室、プール、ホットタブ、乗客の気まぐれに応じるための追加の乗務員などに充てられています。

#### レール

この乗り物はレールシステムに依存しています。この特性を持つ乗り物は、通常の外殻サイズが許容する速度より3段階高い速度評価を自然に達成できますが、当然ながらレールシステム上での移動に限定されます。

#### 弾力性

すべてのシステムは、通常サイズによって決定されるよりも1ヒット多く耐えることができます。この特性は累積可能です。したがって、「弾力性2」は、システムが通常の2ヒット多く耐えられることを示し、2特性ポイントを消費します。

#### 水上

この乗り物は水に浮きます。

#### 宇宙船

この乗り物は真空から密閉され、宇宙放射線からシールドされており、設計によっては大気圏を離着陸し、大気圏内で飛行を維持する能力があるかもしれません。

#### ステルス

航空機専用。この乗り物は、レーダーやその他の長距離センサーシステムによる探知を避けるように設計されています。電子センサーを使用して乗り物を発見する試みは、1段階難しくなります。

#### 潜水可能

この乗り物は、海洋表面下を潜水し、外殻評価×10メートルまでの作動深度に到達できます。崩壊深度（潜水艇が押しつぶされずに到達できる絶対最大深度）は、作動深度の1.5倍です。

#### 優れた操縦性

この乗り物の操縦性は「容易」と評価されます。

#### 監視スイート

この乗り物には、オンボードコンピュータ、通信基地局、盗聴器信号受信機、および100メートル以内のスマートフォン信号を傍受するための装置が搭載されています。

#### 頑丈

この特性を持つ民間車両は、軍事列から外殻を引き、自動的に最大構造点を持ちます。この特性を持つ軍事車両は、次の行から外殻と構造点を引き継ぎます。

#### トラクタービーム

宇宙船専用。この船にはトラクタービームが搭載されており、自身の外殻評価より最大2段階小さい外殻評価を持つ車両を捕らえて引き寄せることができます。したがって、巨大なスターシップは中型サイズまでの船を引き寄せ、保持することができます。

#### VTOL（垂直離着陸）

垂直離着陸。この特性は、ヘリコプターや、リフトを生成するためにローターを組み合わせる同様の車両に固有のものです。固定翼航空機に適用される場合、これは空中姿勢、上昇、降下を制御するために固定または可動のスラスターを使用することを示します。

#### ウォーカー

この乗り物は関節のある脚を使って推進します。ウォーカー車両は、そのサイズに関わらず「穏やか」の速度にキャップされます。しかし、ウォーカーは、他の移動手段を備えた車両では不安定になるような高さに建造することができます。

この車両は水中移動が可能です。

## 操縦性と操縦

回避行動や急な操縦には、運転手またはパイロットが乗り物の制御を維持するために、運転/操縦判定の成功が必要です。ほとんどの乗り物は、時折のストレスや負担に耐えるように設計および製造されていますが、ゲームマスターは、その野心に応じて、特定の操縦がより高い難易度段階での判定を必要とすると判断する場合があります。失敗は乗り物の制御を失うことを意味します。制御喪失表でロールしてください。

制御喪失表は地上車両にのみ適用されることに注意してください。同様の効果は航空機や宇宙車両にも適用できますが、ゲームマスターは特定の影響の理由を調整する必要があります。例えば、航空機は横滑りする代わりにスピンに陥るかもしれません。

また、乗り物の固有の操縦性は、運転判定にさらなる影響を与える場合があります。操縦性は「容易」、「標準」、「困難」、「難攻不落」、「超人的」のいずれかであり、操縦を実行する際の運転判定の開始点を決定します。その後、操縦の難易度段階が操縦性に適用され、最終的な難易度が決定されます。

例えば、高性能モーターサイクルは内蔵されたジャイロスタビライザーにより「容易」な操縦性を持ちます。運転手は追跡者を回避するために180度のスキッドターンを行いたいと考えており、ゲームマスターはその操縦を追加で1難易度段階難しいと評価します。したがって、ライダーは運転判定を「容易」ではなく「標準」で行わなければなりません。もし安定装置のないモーターサイクルで同じ操縦を試みていた場合、運転判定は「困難」となるでしょう。

「優れた操縦性」は特性です。乗り物の説明にこの特性がない場合、大型以下の乗り物はデフォルトで「標準」の操縦性を持ちます。超大型および巨大な乗り物は、それぞれ本質的に「難攻不落」および「超人的」の操縦性特性を持ち、ゲームマスターは、乗り物のサイズのために特定の操縦が単に不可能であると裁定する自由があります。

##### 制御喪失表

| 1d100 | 結果                                                                         |
| :---: | :--------------------------------------------------------------------------- |
| 01-25 | **蛇行。** 制御喪失は一時的なもの。乗り物の速度は5秒間1段階低下する。            |
| 26-40 | **横滑り。** 運転手は乗り物の制御を維持するために奮闘しなければならない。乗り物の速度は10秒間2段階低下する。 |
| 41-50 | **激しい横滑り。** 乗り物は間違った方向を向き、15秒間停止する。               |
| 51-60 | **横転。** 乗り物は横滑りし横転し、構造点に3d10ダメージを受ける。乗員は耐久力判定に成功しないと1d3の部位に1d10ダメージを受ける。 |
| 61-70 | **激しい横転。** 上記と同じだが、乗り物は3d10+10ダメージを受け、乗員は耐久力判定に成功しても1d10ダメージ、失敗すれば2d10ダメージを受ける。 |
| 71-80 | **全損。** 上記と同じだが、乗り物は構造点0になる。乗員は激しい横転と同じダメージを受ける。 |
| 81-90 | **爆発。** 上記と同じだが、乗り物の燃料システムが1d20+10秒以内に点火・爆発する。脱出できない場合、乗員は1d6の部位にさらに1d6の火傷ダメージを受ける。 |
| 91-98 | **即時爆発。** 上記と同じだが、爆発は即時である。                            |
| 99-00 | **壊滅的な衝突。** 乗員は耐久力判定に成功しないと即死。全損と同じダメージは関係なく受ける。 |
