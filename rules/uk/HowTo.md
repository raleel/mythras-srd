# Передумови

1. Базове розуміння [мови розмітки Markdown](https://www.markdownguide.org/basic-syntax/).

2. Вам потрібно буде створити безкоштовний обліковий запис на [github](https://github.com/), якщо у вас його ще немає. Рекомендується використовувати назву вашої НРИ, оскільки вона стане частиною URL-адреси, яку люди використовуватимуть для доступу до вашого SRD.

3. Ви можете виконати весь цей процес через браузер, не знаючи Git. Хоча вивчення github desktop дозволить вам тестувати будь-які зміни локально, перш ніж розгортати їх для світу.

> Якщо у вас є зареєстроване доменне ім'я, ви можете використовувати його в цьому процесі для створення SRD, як це зробив я: https://srd.7thextinctionrpg.com

# Процес

## Створення репозиторію на Github

1. Увійдіть до Github

2. Перейдіть за публічним шаблоном https://github.com/7thExtinctionrpg/RPG_SRD_Template

3. Натисніть "Use this template" (Використати цей шаблон)

![Step3](/_media/Step3.png)

4. Оберіть "Create a new repository" (Створити новий репозиторій)

5. Введіть назву репозиторію

![Step5](/_media/Step5.png)

6. Оберіть 'Private' (Приватний)

7. Оберіть 'Create repository' (Створити репозиторій)

Копіювання шаблону займе 5-30 секунд, після чого у вас буде власна копія.

8. Переконайтеся, що у вас є файл ".nojekyll" – це порожній файл, необхідний при розміщенні веб-сайту на Github, інакше SRD не відображатиметься належним чином. *(Повідомляє GitHub Pages не запускати опубліковані файли через Jekyll).*

## Додавання вашого контенту

### Файли *.md

Файли .md — це файли Markdown. Markdown — це полегшена мова розмітки, яку можна використовувати для додавання елементів форматування до текстових документів. Залежно від обсягу вашого вмісту, ви можете або розмістити всю інформацію про вашу гру в одному файлі *.md, або, як я зробив на [7thextinctionrpg.com](https://srd.7thextinctionrpg.com), створити окремі файли для кожного розділу. "sample.md" надає деякі приклади форматування.

Мова Markdown дуже легка для вивчення, повний список синтаксису дивіться за посиланням, наданим у пункті 2 розділу "Передумови" вище.

### _Sidebar.md

Це ваше меню, панель навігації в лівій частині веб-сайту. Додавши файл Markdown до вашого репозиторію, вам потрібно додати його сюди, щоб його вміст було включено. Дотримуйтесь існуючого форматування.

## Увімкнення Github Pages для перетворення репозиторію на веб-сайт

1. У вашому репозиторії натисніть 'Settings' (Налаштування)

![Website-Step1](/_media/Website-Step1.png)

2. Оберіть 'Pages' у лівому бічному меню

![Website-Step2](/_media/Website-Step2.png)

3. Якщо ви не зробили репозиторій 'public' (публічним), у вас є два варіанти:
    a. Зробити репозиторій публічним, що дозволить вам використовувати Github Pages для безкоштовного розміщення вашого SRD-сайту.
    b. Оновити ваш обліковий запис Github (на момент написання цього тексту це коштує $4 на місяць або $48 щорічної плати).

4. У розділі 'Branch' (Гілка) оберіть 'main' і натисніть save (зберегти)

5. Зачекайте 10-20 секунд, потім оновіть сторінку

6. Тепер ви побачите свою URL-адресу, наприклад https://coweater.github.io/mysrd/

![Website-Step6](/_media/Website-Step6.png)

## Зміна шрифтів

1. Відредагуйте файл /_assets/style.css

2. У розділі 'body' ви знайдете siteFont та headingFont

3. Змініть їх на потрібні вам шрифти

## Використання плагіна Fontawesome та/або Material Icons для відображення безкоштовних іконок у вашому SRD

### Fontawesome

Використовуйте Fontawesome, щоб отримати безкоштовні іконки та анімовані іконки. Передплатники можуть використовувати Pro Icons.

1. Знайдіть іконку на [Fontawesome](https://fontawesome.com/icons)
2. Оберіть іконку
3. У спливаючому вікні, на вкладці HTML, скопіюйте все, що знаходиться між " "

```html
<i class="fa-solid fa-dice-d20"></i>
```

4. Розмістіть скопійований текст між `<p>`, без пробілів.

```html
<p><i class="fa-solid fa-dice-d20"></i></p>
```

### Material Icons

Використовуйте Material Icons для більшої кількості безкоштовних іконок

1. Знайдіть іконки на [Material Icons](https://fonts.google.com/icons)
2. Скопіюйте та вставте веб-посилання span безпосередньо у ваші файли Markdown

```html
<span class="material-symbols-outlined">
dice_5
</span>
```
