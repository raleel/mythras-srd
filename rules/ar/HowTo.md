# المتطلبات الأساسية

1. فهم أساسي [للغة Markdown](https://www.markdownguide.org/basic-syntax/).

2. ستحتاج إلى إنشاء حساب [github](https://github.com/) مجاني إذا لم يكن لديك واحد. يوصى باستخدام اسم لعبة تقمص الأدوار الخاصة بك (RPG) حيث سيصبح هذا جزءًا من عنوان URL الذي سيستخدمه الأشخاص للوصول إلى SRD الخاص بك.

3. يمكنك إنجاز هذه العملية بالكامل عبر المتصفح دون معرفة Git. ومع ذلك، فإن تعلم GitHub Desktop سيتيح لك اختبار أي تغييرات محليًا قبل نشرها للعالم.

> إذا كان لديك اسم نطاق مسجل، فيمكنك استخدامه مع هذه العملية لإنشاء SRD كما فعلت أنا: https://srd.7thextinctionrpg.com

# العملية

## إنشاء المستودع على GitHub

1. سجل الدخول إلى GitHub

2. انتقل إلى القالب العام https://github.com/7thExtinctionrpg/RPG_SRD_Template

3. انقر على "Use this template" (استخدم هذا القالب)

![Step3](/_media/Step3.png)

4. اختر "Create a new repository" (إنشاء مستودع جديد)

5. أدخل اسم المستودع

![Step5](/_media/Step5.png)

6. اختر "Private" (خاص)

7. اختر "Create repository" (إنشاء مستودع)

سيستغرق الأمر من 5 إلى 30 ثانية لنسخ القالب، ثم ستحصل على نسختك الخاصة.

8. تأكد من وجود الملف ".nojekyll" هذا ملف فارغ مطلوب عند استضافة موقع ويب على GitHub وإلا فلن يتم عرض SRD بشكل صحيح. *(يخبر صفحات GitHub بعدم تشغيل الملفات المنشورة من خلال Jekyll).*

## إضافة المحتوى الخاص بك

### ملفات *.md

ملفات `.md` هي ملفات Markdown، وMarkdown هي لغة ترميز خفيفة الوزن يمكنك استخدامها لإضافة عناصر تنسيق إلى مستندات نص عادي. بناءً على حجم المحتوى الخاص بك، يمكنك إما وضع جميع معلومات لعبتك في ملف `.md` واحد، أو كما فعلت أنا في [7thextinctionrpg.com](https://srd.7thextinctionrpg.com) إنشاء ملفات منفصلة لكل فصل. يقدم "sample.md" بعض أمثلة التنسيق.

لغة Markdown سهلة التعلم للغاية، استخدم الرابط المقدم في النقطة 2 ضمن المتطلبات الأساسية أعلاه للحصول على قائمة كاملة بالنحو.

### _Sidebar.md

هذا هو قائمتك، جزء التنقل على الجانب الأيسر من الموقع الإلكتروني. بمجرد إضافة ملف Markdown إلى المستودع الخاص بك، تحتاج إلى إضافة الملف هنا بحيث يتم تضمين محتوياته. اتبع التنسيق الحالي.

## تفعيل صفحات GitHub لتحويل المستودع إلى موقع ويب

1. في المستودع الخاص بك، انقر على "Settings" (الإعدادات)

![Website-Step1](/_media/Website-Step1.png)

2. اختر "Pages" (الصفحات) من القائمة الجانبية اليسرى

![Website-Step2](/_media/Website-Step2.png)

3. إذا لم تكن قد جعلت المستودع "عامًا" (public)، فلديك خياران:
    أ. اجعل المستودع عامًا، مما سيسمح لك بعد ذلك باستخدام صفحات GitHub لاستضافة موقع SRD الخاص بك مجانًا.
    ب. قم بترقية حسابك على GitHub (وقت كتابة هذا، يكلف 4 دولارات شهريًا أو رسوم سنوية قدرها 48 دولارًا).

4. ضمن "Branch" (الفرع)، اختر "main" وانقر على "Save" (حفظ)

5. انتظر من 10 إلى 20 ثانية ثم قم بتحديث الصفحة

6. سترى الآن عنوان URL الخاص بك مثل https://coweater.github.io/mysrd/

![Website-Step6](/_media/Website-Step6.png)

## استخدام مكون Fontawesome و/أو Material Icons الإضافي لعرض أيقونات مجانية في SRD الخاص بك

### Fontawesome

استخدم Fontawesome للحصول على أيقونات مجانية وأيقونات متحركة. يمكن للمشتركين استخدام أيقونات Pro

1. ابحث عن الأيقونة على [Fontawesome](https://fontawesome.com/icons)
2. اختر الأيقونة
3. في النافذة المنبثقة، تحت علامة التبويب HTML، انسخ كل شيء بين " "

```EXAMPLE
<i class="fa-solid fa-dice-d20"></i>
```

4. ضع النص المنسوخ بين `:` بدون مسافات

```EXAMPLE
fa-solid fa-dice-d20
```

### Material Icons

استخدم Material Icons للمزيد من الأيقونات المجانية

1. ابحث عن الأيقونات على [Material Icons](https://fonts.google.com/icons)
2. انسخ وأدخل رابط نطاق الويب مباشرة في ملفات Markdown الخاصة بك

```markup
<span class="material-icons">face</span>
```
