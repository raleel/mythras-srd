# 스타일 가이드 (h1)

주요 챕터 제목에는 h1 제목(`# Heading Title`)을 사용합니다. 모든 hx 제목은 사이드바에 북마크로 표시됩니다.

제목이 h1 제목이 아닌 경우, 사이드바에 페이지와 h2 '제목'이 중복되어 표시되는 오류가 발생합니다.

---
## 섹션 제목 (h2)

새로운 주요 섹션마다 구분선(`---`) 앞에 h2 제목(`## Section Heading`)을 사용합니다.

### 소제목은 h-레벨을 1씩 증가

새로운 소제목에는 h3 제목(`### Subsection Heading`)을 사용하고, 새로운 하위 소제목에는 h4 제목(`#### Sub-Subsection Heading`)을 사용합니다.

### 소제목 h-진행: 2 > 3 > 4

h-진행이 올바르게 처리되지 않으면 사이드바에서 소제목이 섞입니다.

제목 표시는 1씩 증가하거나 아무것도 증가하지 않아야 합니다.

h2 다음에 h2 또는 h3가 오는 것은 괜찮습니다. h3 다음에 h3 또는 h4가 오는 것도 괜찮습니다.

### 소제목 오류
h2 다음에 h4가 오면 문제가 발생합니다. 사이드바는 소제목을 목차의 해당 제목보다 위에 배치합니다.

### h-레벨 감소

이것은 문제가 되지 않는 것 같습니다. 예: h4에서 h3 또는 h2로 가는 것은 괜찮습니다.

## 서식 규칙

다음 스타일은 Markdown 형식으로 된 이 _Imperative_ SRD의 표준입니다.

### 굵게, 기울임꼴 및 구분선

- **굵게** 표시할 텍스트에는 이중 별표(`**Bold**`)를 사용합니다.
- _기울임꼴_ 텍스트에는 밑줄(`_Italic_`)을 사용합니다.
  - _참고: 기울임꼴에 단일 별표를 사용하지 마십시오!_
- 문자를 '이스케이프'하려면 역슬래시(`\_`)를 사용합니다: \_기울임꼴 아님\_.
- `code blocks`를 만들려면 꺾쇠 따옴표(``` `code` ```)를 사용합니다.
- 가로 구분선을 삽입하려면 하이픈 3개(`---`)를 사용합니다.

### 굵게 & 기울임꼴 규칙

다음 패턴을 따르도록 노력하세요...

#### 굵은 제목

주문, 효과 등을 설명할 때 굵은 제목을 사용합니다.
- **마법 제목:** 설명.

#### 기울임꼴 제목

변형 주문 등의 '하위 목록'에는 기울임꼴 제목을 사용합니다.
- **마법:** 설명.
  - **_마법 변형-A:_** 설명.
  - **_마법 변형-B:_** 설명.

#### 기울임꼴 제목 참조

_Mythras Imperative_ 또는 코어 _Mythras_ 규칙서와 같은 책을 참조할 때 기울임꼴 제목을 사용합니다.

## 목록 서식

### 순서 없는 목록 / 글머리 기호

- 순서 없는 목록은 하이픈으로 만듭니다.

1. 번호 매기기 목록은 숫자와 마침표로 만듭니다.
2. 번호 매기기 목록은 숫자와 마침표로 만듭니다.

## 표

표에는 소제목을 지정하여 사이드바에 부모 섹션에서 한 단계 들여쓰기된 북마크로 표시되도록 합니다.

~제목 행을 굵게 표시합니다.~
제목 행을 굵게 표시하면 실제로는 일반 텍스트로 렌더링됩니다. 기본 CSS는 해당 행을 굵게 처리합니다.

### 표 텍스트 정렬

내용 정렬을 위해 2행을 사용합니다: `:--`, `:-:`, 또는 `--:`.
이는 전체 열에 영향을 미칩니다.

#### 표의 기본 레이아웃

| **열 1** | **열 2** |
| :-: | :-- |
| 1 | 알파 |
| 2 | 베타 |
| 3 | 감마 |

### 텍스트 정렬 규칙

- 숫자나 유사한 값 목록에는 가운데 정렬을 사용합니다.
- 문장 텍스트에는 왼쪽 정렬을 사용합니다.

#### 정렬 예시 표

| **번호** | **설명** |
| :-: | :-- |
| 1 | 설명 텍스트 |
| 2 | 설명 텍스트 |
| 3 | 설명 텍스트 |

#### 예시: 화재 강도 표

| **강도** | **예시** | **점화 시간** | **피해** |
| :-: | :-- | :-: | :-: |
| 1 | 양초 | 1d4 | 1d2 |
| 2 | 횃불 | 1d3 | 1d4 |
| 3 | 모닥불 | 1d2 | 1d6 |
| 4 | 방을 가득 채우는 대화재 | 1d2 | 2d6 |
| 5 | 화산 용암 | 즉시 | 3d6 |

## 링크

SRD에는 링크의 많은 용도가 있으며 여기서는 몇 가지 주요 경우만 다룹니다.

### 링크 서식

링크 서식은 괄호 안의 텍스트로 지정됩니다. 표시될 링크 텍스트에는 대괄호를, 대상 URL에는 둥근 괄호를 사용합니다: `[Link Text](URL)`.

이 문서에서 가장 일반적으로 사용되는 외부 링크는 [Mythras Encounter Generator](https://mythras.skoll.xyz)입니다.
```
[Mythras Encounter Generator](https://mythras.skoll.xyz)
```

### 외부 링크 vs 내부 링크

외부 링크는 인터넷을 통해 다른 페이지로 이동하며 `http://example.com` URL이 필요하며, 기본적으로 _항상_ 새 브라우저 탭에서 열립니다.

읽고 있는 문서가 여러 탭으로 어수선해지는 것을 방지하기 위해, 내부 링크 및 북마크는 GitHub 저장소에 보이는 Markdown 파일을 참조하는 상대 URL로 형성됩니다. 새 탭 대신, 이러한 내부 링크는 연결된 섹션으로 빠르게 스크롤됩니다.

#### 내부 링크 예시

- **챕터 링크:** 주 (h1) 섹션의 시작 부분을 가리킵니다. 예: [전투](0005_Combat.md) 챕터.

  - 링크 형식: `[Combat](0005_Combat.md)`

- **소섹션 링크:** 위에 설명된 문서뿐만 아니라 `?id=`와 대상 제목이 바로 뒤따르는 방식으로 챕터 내의 제목을 참조하는 링크입니다. 예: [도약 공격](0005_Combat.md?id=leaping-attacks)

  - 링크 형식: `[Leaping Attacks](0005_Combat.md?id=leaping-attacks)`

#### 소섹션으로의 외부 링크

외부 문서의 특정 섹션, 예를 들어 다른 SRD의 특정 표를 참조해야 할 경우가 있을 수 있습니다. 서식은 위 소섹션 링크와 동일하며, 외부 HTTP URL만 사용합니다.

예를 들어, [Classic Fantasy Imperative SRD](https://cfi-srd.mythras.net/)에서 [제국 단위에서 미터법 변환 표](https://cfi-srd.mythras.net/#/Appendix_B_Conversion_Tables?id=imperial-weights-and-measures-to-metric-conversion)입니다.

```
[Imperial to Metric conversions table](https://cfi-srd.mythras.net/#/Appendix_B_Conversion_Tables?id=imperial-weights-and-measures-to-metric-conversion)
```

## 마법 및 능력 목록

사용 가능한 주문, 능력 등의 목록을 위해 h2 제목으로 '깨끗한' 섹션을 시작한 다음, 각 목록 항목에 h3를 사용합니다.

주문 및 능력은 일반적으로 마법 유형(민속 마법, 신성 마법 등)별로 단일 평면 목록으로 제시되지만, 목록에 학교 또는 컬트 특수성과 같은 자체 소섹션이 필요한 경우 적절하게 제목 수준을 높입니다. 예를 들어, 신성 마법 기적 목록에 h2, 컬트: 미트라스에 h3, 각 미트라스 컬트 기적에 h4를 사용합니다.

특히 주문의 경우, 키워드 속성을 사용하는 모든 설명에서 기울임꼴 텍스트로 나열합니다.

설명 텍스트와 특수 규칙은 일반 텍스트를 사용합니다.

### 주문 목록 서식

```
## h2 List name

### List item

_Attributes_

Description text.
```

## 차량 능력치 블록
목록을 위해 h2 제목으로 '깨끗한' 섹션을 시작한 다음, 카테고리 및 목록 항목에 평소처럼 h-레벨을 증가시킵니다.

각 나열된 예시는 굵게 표시된 제목으로 된 글머리 기호 목록으로 능력치와 설명을 제시합니다.

### 차량 능력치 블록 서식

```
## h2 List name

### Category

#### List Item

- **Stat 1:**  [Value]

- **Stat 2:**  [Value]

- **Description:** Descriptive text.
```

## 생명체 능력치 블록
목록을 위해 h2 제목으로 '깨끗한' 섹션을 시작합니다. 생명체 능력치 블록은 표를 사용하며 서식 및 공간 요구 사항 측면에서 더 복잡합니다. 명확성을 위해 소섹션을 피하고 목록 항목 사이에 구분선을 추가합니다.

각 나열된 예시에는 다음이 포함되어야 합니다...

- **생명체 제목:** 구분선 뒤에 생명체 이름의 h3 헤더.

- **MEG 항목:** [Mythras Encounter Generator](https://mythras.skoll.xyz/)의 지정된 생명체에 대한 외부 링크.

- **핵심 세부 사항:** 생명체 이름/유형, 속성, 기술 및 필요한 특별 설명 또는 캐릭터 노트.

- **무기 세부 사항:** 무기 이름 및 다음 속성을 위한 열이 있는 요약된 설명: 크기(여기에 힘 포함), 사거리(여기에 범위 포함), 피해, AP/HP 및 모든 특별 규칙.

- **타격 부위 및 갑옷:** 부위 표는 생명체의 해부학적 구조에 따라 달라지지만, 전통적인 휴머노이드의 경우에도 갑옷 및 피해 추적 수단으로 포함되어야 합니다.

### 생명체 능력치 블록 서식

```
## Bestiary/List Name

Descriptive text. 

---
### Example Creature
[MEG Entry](https://mythras.skoll.xyz/enemy_template/8626/)

| Creature Name | Example Creature  |
| :-- | :-- |
| Key Attributes | STR 11, CON 14, SIZ 11, DEX 11, INT 11, POW 11, CHA 7 |
| Other notes | Values and details | 
```

## 커뮤니티 에라타

### 오타 및 혼동

저자의 텍스트를 '정리'하기 위한 편집은 최소한으로 유지되어야 하지만, 오타 및 기타 실수는 별도의 표시 없이 바로 수정되어야 합니다.

### 에라타 정보 패널

주어진 규칙이 너무 모호하거나, 불완전하거나, 단순히 틀린 경우, 접을 수 있는 커뮤니티 에라타 메모를 추가합니다.

문제를 기울임꼴로 설명하고, 문제의 제목에만 굵은 기울임꼴을 사용합니다. - _h(x) 헤더는 사용하지 마세요!_ - 혼동을 주는 사이드바 항목을 만들지 않기 위함입니다.

교체 텍스트가 추가되는 경우, 일반 기울임꼴로 "_Community Errata:_" 앞에 붙여서 굵은 기울임꼴로 이중 따옴표 안에 넣습니다.

필요한 추가 규칙은 일반 텍스트로 제시되어야 합니다.

#### 에라타 서식 템플릿

```
<details>

<summary>* Community Errata</summary>

_**Issue Title**_

_Community Errata:_
_**"Replacement text if required."**_

_Explanation text stating how the rule as written causes problems._

</details>
```

_**등반**_

_커뮤니티 에라타:_
_**"착용한 갑옷은 착용한 갑옷 점수의 평균만큼 등반을 방해합니다. 이는 각 부위의 착용한 갑옷 점수를 모두 더하고 그 수치를 부위 수로 나누어 구합니다."**_

_작성된 규칙은 빠르게 문제를 일으킵니다. 6AP 갑옷 한 조각만 착용해도 인간은 이동 0이 되어, 강철 헬멧만 착용하고 다른 것은 아무것도 착용하지 않은 인간은 전혀 등반할 수 없을 정도로 과도한 부담을 느낍니다. 이는 비합리적으로 제한적이며, 따라서 위에서 제안된 수정된 버전이 제시되었습니다. 이 규칙은 원 저자들이 수정할 경우 업데이트될 예정입니다._

<summary>* 커뮤니티 에라타</summary>

_**보행: 걷기, 달리기, 전력 질주**_

_출판된 _Mythras Imperative_ 버전에는 다른 보행 방식으로 이동하는 명시적인 규칙이 없지만, Mythras 핵심 규칙서에는 다루어져 있습니다. 위 보행 배율은 커뮤니티 에라타의 일부로 추가되었습니다._

_필요한 경우 보행을 다음과 같이 처리하는 것으로 충분할 것입니다:_

- _걷기보다 빠른 보행은 이동 속도에 배율을 적용합니다: 달리기는 3배, 전력 질주는 5배._

- _달리기 또는 전력 질주 보행 중에는 캐릭터는 공격하거나 주문을 시전하는 등 대부분의 선제적 행동을 시도할 수 없습니다. 몇 가지 예외는 다음과 같습니다..._
  - _돌격 규칙을 사용할 때의 공격._
  - _소규모 전투 특성을 가진 원거리 무기를 발사하는 경우._
- _일반적으로 허용되는 모든 행동은 달릴 때 1단계, 전력 질주할 때 2단계 더 어려워진다고 간주해야 합니다._

## 기타 경고

### 빈 줄 / 캐리지 리턴

다른 유형의 항목(헤더, 구분선, 표 등) 사이에는 빈 줄을 남겨 두십시오. 이러한 빈 줄은 읽기 보기에서 보이지 않지만, 없으면 서식 문제가 발생할 수 있습니다.

### 굵게 또는 기울임꼴 여러 단락 텍스트

각 단락은 굵게 또는 기울임꼴 텍스트로 표시되어야 합니다. 단락 사이의 새 줄은 서식을 깨뜨려 고아 서식 표시를 남길 수 있습니다.

### 국제 철자

퇴행적이고 한심한 세력들이 모든 영국식 철자를 객관적으로 더 약한 미국식 철자로 바꾸기로 결정했습니다. 영국식 -OUR 단어: _armour_, _favour_ 등을 주의 깊게 살펴보십시오.
