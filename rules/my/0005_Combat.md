# တိုက်ခိုက်ခြင်း

တိုက်ခိုက်ခြင်းသည် RPG (Roleplaying Games) များ၏ အရေးပါသော အင်္ဂါရပ်တစ်ခုဖြစ်သည်။ လက်စားချေရန် မိတ်ဆွေမဟုတ်သူများအကြား တိုက်ခိုက်ခြင်းမှသည် စစ်ပွဲကာလတွင် စစ်တပ်များ တိုက်ခိုက်ခြင်းအထိ ပြဿနာများကို ဖြေရှင်းရန် အကြမ်းဖက် ထိပ်တိုက်ရင်ဆိုင်မှုများ ရှိခဲ့သည်။ သို့သော် တိုက်ခိုက်ခြင်းသည် အသက်ဆုံးရှုံးသည်အထိ ရက်စက်ကြမ်းကြုတ်သော တိုက်ပွဲတစ်ခု ဖြစ်စရာမလိုပါ။ အချို့သော တိုက်ခိုက်မှုများသည် ဒဏ်ရာမရဘဲ အဆုံးသတ်သွားသော သူရဲကောင်းဆန်သော လက်ရွေးစင် တိုက်ပွဲများ ဖြစ်နိုင်သည်၊ သို့မဟုတ် ပေါ့ပါးသော ဘားခန်း ရန်ပွဲများ ဖြစ်နိုင်သည်။ _Mythras Imperative_ သည် တိုက်ခိုက်ခြင်း၏ အဓိက အစိတ်အပိုင်းများကို ရှင်းပြရန် ဝေါဟာရအချို့ကို အသုံးပြုထားသည်။ ၎င်းတို့ကို အောက်ပါအပိုင်းများတွင် ရှင်းပြထားသော်လည်း အကျဉ်းချုပ်အားဖြင့် ၎င်းတို့မှာ-
- **တိုက်ခိုက်ရေးပုံစံ (Combat Style):** ယဉ်ကျေးမှု သို့မဟုတ် အသက်မွေးဝမ်းကျောင်းမှု၏ အစိတ်အပိုင်းအဖြစ် သင်ယူထားသော လက်နက်အုပ်စုတစ်ခုကို ကိုင်စွဲရာတွင် ကျွမ်းကျင်မှု။
- **တိုက်ခိုက်ရေးအချီများ (Combat Rounds):** ငါးစက္ကန့်စီရှိသော အချိန်ပိုင်းများ။
- **တိုက်ခိုက်ရေးလုပ်ဆောင်ချက်များ (Combat Actions):** လုပ်ဆောင်ချက်မှတ် (Action Point) သုံးစွဲခြင်းဖြင့် အချီတစ်ခုအတွင်း ဇာတ်ကောင်တစ်ဦး လုပ်ဆောင်နိုင်သော လုပ်ရပ်များ။
- **လက်နက်အရွယ်အစား (Weapon Size):** သီးခြားလက်နက်တစ်ခုကို ပိတ်ဆို့ရန် မည်မျှခက်ခဲခြင်း။
- **ထိတွေ့မှု (Engagement):** အခြေအနေအရ တိုက်ခိုက်သူများသည် တိုက်ရိုက်ပြိုင်ဘက်ကို တိုက်ခိုက်နိုင်ခြင်း သို့မဟုတ် အတိုက်ခိုက်ခံရနိုင်ခြင်း။
- **အထူးအကျိုးသက်ရောက်မှုများ (Special Effects):** တိုက်ခိုက်သူတစ်ဦးက အသာရရှိပါက လုပ်ဆောင်နိုင်သော တိုက်ခိုက်ရေး လှုပ်ရှားမှုများ။

## တိုက်ခိုက်ရေးပုံစံများ

တိုက်ခိုက်ရေးပုံစံ (Combat Style) သည် သဘာဝအရ ဆက်စပ်နေသော လက်နက်များစွာ၏ 'အတွဲအစည်း' ကို ကိုယ်စားပြုပြီး လက်နက်တစ်ခုချင်းစီကို သီးခြားစီ သင်ယူရမည့် လိုအပ်ချက်ကို ရှောင်ရှားပေးသည်။

တိုက်ခိုက်ရေးကျွမ်းကျင်မှု (Combat Skill) ၏ အရေးအပါဆုံး အချက်မှာ ဇာတ်ကောင်သည် ကျွမ်းကျင်မှုရှိသော လက်နက်တစ်ခုစီကို တစ်ခုတည်းအနေဖြင့် သော်လည်းကောင်း၊ ပေါင်းစပ်ပြီး သော်လည်းကောင်း မည်သို့အသုံးပြုရမည်ကို သင်ယူခြင်းပင်ဖြစ်သည်။ ၎င်းတို့ကို လိုအပ်သလို အပြန်အလှန် အသုံးပြုနိုင်သည်။ တိုက်ခိုက်ရေး လေ့ကျင့်မှုသည် အကောင်းဆုံး အခြေအနေများအောက်တွင် သီးခြားလက်နက်တစ်ခု သို့မဟုတ် လက်နက်များကို အသုံးပြုခြင်းကိုသာ အာရုံစိုက်သည်မဟုတ်ဘဲ ပြင်းထန်သော အားနည်းချက်တစ်ခုတွင် ရှိနေသောအခါ ဘာလုပ်ရမည်ကိုလည်း အကျုံးဝင်သည်။ ထိုသို့သော ဘက်စုံလေ့ကျင့်မှုသည် လက်နက်ကျိုးပဲ့ခြင်း သို့မဟုတ် လက်နက်ချခြင်းတို့ အမြဲတမ်း ဖြစ်နိုင်ခြေရှိသော စစ်မြေပြင်၏ ခန့်မှန်းရခက်သော ဖြစ်ရပ်များအတွက် တိုက်ခိုက်သူတစ်ဦးကို ပြင်ဆင်ရာတွင် အဓိက အစိတ်အပိုင်းတစ်ခုဖြစ်သည်။

ထို့ကြောင့်၊ Hyberborean လူရိုင်းတစ်ဦးသည် လှံ၊ လက်ပုဆိန်၊ လှံတံနှင့် ဒိုင်းတို့ကို မည်သို့အသုံးပြုရမည်ကို သိရှိရန် မျှော်လင့်ထားနိုင်သကဲ့သို့ Sumerian စစ်သည်တော်တစ်ဦးသည် ဓားတို၊ တုတ်၊ ဒိုင်းနှင့် လေးတို့ကို လေ့ကျင့်ထားနိုင်သည်။ ပုံစံတစ်ခုတည်းတွင် လက်နက်မည်မျှ ပါဝင်သင့်သည်ကို ဆုံးဖြတ်ခြင်းသည် နောက်ဆုံးတွင် ဂိမ်းမာစတာနှင့် ဂိမ်း၏ ဆက်တင်အပေါ် မူတည်သည့် ရွေးချယ်မှုဖြစ်သည်။

၎င်းတို့၏ပုံစံတွင် မပါဝင်သော လက်နက်တစ်ခုကို အသုံးပြုရန် ကြိုးပမ်းသော ဇာတ်ကောင်များသည် ကျွမ်းကျင်မှု ဒဏ်ကြေး တစ်ခု သို့မဟုတ် ထို့ထက်ပိုသော အခက်အခဲအဆင့်များကို ခံစားရမည်။

တိုက်ခိုက်ရေးပုံစံများသည် လက်နက်ကျွမ်းကျင်မှု အုပ်စုတစ်ခုကို သင်ယူခြင်းထက် ကျော်လွန်သော လုပ်ဆောင်ချက်တစ်ခုကို ပိုင်ဆိုင်သည်။ ပုံစံတစ်ခုစီကို သီးခြားယဉ်ကျေးမှု သို့မဟုတ် အသက်မွေးဝမ်းကျောင်းမှုတစ်ခုမှ သင်ကြားပေးသောကြောင့် ၎င်းတို့သည် ထိုအဖွဲ့က ပုံမှန်တိုက်ခိုက်သော ပတ်ဝန်းကျင် သို့မဟုတ် စစ်ရေးနည်းဗျူဟာများအတွက် မွေးရာပါ ပိုမိုကောင်းမွန်အောင် ပြင်ဆင်ထားသည်။ ဥပမာအားဖြင့်၊ မြင်းစီး ကုန်သည်တစ်ဦးသည် ကုန်းနှီးပေါ်မှ တိုက်ခိုက်ရန် သင်ယူခြင်း သို့မဟုတ် ခြေလျင်တပ်သားတစ်ဦးသည် အနီးကပ် ဖွဲ့စည်းမှုဖြင့် တိုက်ခိုက်ရန် လေ့ကျင့်ခြင်းတို့ဖြစ်သည်။ ထို့ကြောင့် အပိုဆုကြေးတစ်ခုအနေဖြင့် အချို့သော တိုက်ခိုက်ရေးပုံစံများကို အခြေအနေအချို့အောက်တွင် အားသာချက်ပေးသော စရိုက်လက္ခဏာ (Traits) တစ်ခု သို့မဟုတ် ထို့ထက်ပို၍ သတ်မှတ်ပေးနိုင်သည်။ စရိုက်လက္ခဏာများကို ထိုပုံစံ၏ လက်နက်များနှင့် အခြေအနေများတွင်သာ အသုံးပြုရန် ရည်ရွယ်သည်။ ထင်ရှားသည်မှာ စရိုက်လက္ခဏာများစွာရှိပြီး အများစုမှာ သီးခြားခေတ်ကာလ သို့မဟုတ် ကမ်ပိန်းကမ္ဘာအတွက် သီးသန့်ဖြစ်သည်။

##### တိုက်ခိုက်ရေးပုံစံ စရိုက်လက္ခဏာများ ဇယား

| စရိုက်လက္ခဏာ | ဖော်ပြချက် |
| :-- | :-- |
| သားရဲကျောပေါ်မှ လှံစွပ် (Beast-back Lancer) | ဤတိုက်ခိုက်ရေးပုံစံဖြင့် မြင်းစီး တိုက်ခိုက်ရာတွင် ထိမှန်ရန် အခက်အခဲဒဏ်ကြေး တစ်ဆင့်မရှိပါ။ |
| မျက်ကွယ်တိုက်ခိုက်ခြင်း (Blind Fighting) | အလင်းရောင်အားနည်းခြင်း သို့မဟုတ် ယာယီမျက်စိကွယ်ခြင်းကြောင့် ဖြစ်ပေါ်လာသော ဒဏ်ကြေးများကို လျစ်လျူရှုနိုင်သည်။ |
| စွန့်စားရဲသူ (Daredevil) | လက်နက်မဲ့တိုက်ခိုက်ရေးတွင် လဲကျမသွားဘဲ ထိုးနှက်ချက်များကို ရှောင်တိမ်းရန် ရှောင်တိမ်းကျွမ်းကျင်မှု (Evade skill) ကို အသုံးပြုနိုင်သည်။ |
| ခံစစ်စိတ်ရှိသူ (Defensive Minded) | ထိုအချီတွင် ထိုးစစ်လုပ်ဆောင်ချက် (offensive action) မယူပါက သင်၏လက်နက် အရွယ်အစားကို တစ်ဆင့်တိုးမြှင့်ပေးသည်။ |
| အဖွဲ့လိုက်တိုက်ခိုက်ခြင်း (Formation Fighting) | ဘေးကင်းသော စစ်သည်သုံးဦး သို့မဟုတ် ထို့ထက်ပိုသော အဖွဲ့အား အနီးကပ်ဖွဲ့စည်းမှုသို့ ဝင်ရောက်ခွင့်ပြုပြီး ပွင့်နေသော သို့မဟုတ် ရှုပ်ထွေးသော ပြိုင်ဘက်များကို အားနည်းချက်ဖြစ်စေကာ ၎င်းတို့ ထိတွေ့ပါက ရန်သူတစ်ဦးစီ၏ လုပ်ဆောင်ချက်မှတ် (Action Points) ကို တစ်မှတ်လျှော့ချပေးသည်။ |
| လဲကျအောင်ထိုးခြင်း (Knockout Blow) | ရုတ်တရက် တိုက်ခိုက်သောအခါ မည်သည့် ထိမှန်သည့်နေရာကိုမဆို အလှည့်များအစား မိနစ်များကြာရှည်သည်ဟု မှတ်ယူပါ။ |
| မြင်းစီးတိုက်ခိုက်ခြင်း (Mounted Combat) | Ride skill ဖြင့် တိုက်ခိုက်မှုအပေါ် ကန့်သတ်ထားသော ကျွမ်းကျင်မှု အမြင့်ဆုံးကို လျစ်လျူရှုရန် ဇာတ်ကောင်ကို ခွင့်ပြုသည်။ |
| အဝေးပစ်ပစ်ခတ်ကျွမ်းကျင်သူ (Ranged Marksman) | အဝေးပစ်လက်နက် (ranged weapon) အသုံးပြုသောအခါ ကျပန်းထိမှန်သည့်နေရာ (Hit Location) လှိမ့်ခြင်းကို ဘေးချင်းကပ်ကိုယ်ခန္ဓာအစိတ်အပိုင်းတစ်ခုသို့ ရွှေ့ပါ။ |
| ပစ်ခတ်ပြေးလွှားခြင်း (Skirmishing) | ဤပုံစံသည် လမ်းလျှောက်နေစဉ် သို့မဟုတ် ပြေးနေစဉ် အဝေးပစ်တိုက်ခိုက်မှုများကို စတင်ခွင့်ပြုသည်။ |
| လက်နက်ပစ်ခြင်း (Throw Weapons) | ပုံစံရှိ မည်သည့်လက်နက်ကိုမဆို ကျွမ်းကျင်မှုဒဏ်ကြေးမရှိဘဲ ပစ်ခွင့်ပြုသည်၊ သို့သော် ဤနည်းဖြင့် အသုံးပြုသောအခါ လက်နက်၏ ထိခိုက်မှု (damage) ကို တစ်ဝက်လျှော့ချသည်။ |
| လက်နက်မဲ့စွမ်းရည် (Unarmed Prowess) | အသုံးပြုသူအား ၎င်း၏လက်နက်မဲ့ ပိတ်ဆို့ခြင်းနှင့် တန်ပြန်ခြင်းများကို 'အလယ်အလတ်' အရွယ်အစားအဖြစ် မှတ်ယူခွင့်ပြုပြီး လက်နက်ကိုင်ပြိုင်ဘက်များမှ မိမိကိုယ်ကို ပိုမိုကောင်းမွန်စွာ ကာကွယ်နိုင်သည်။ |

## တိုက်ခိုက်ခြင်းကို လုပ်ဆောင်ခြင်း

၎င်း၏ပြင်းထန်သော သဘာဝကို အလေးပေးရန်အတွက် တိုက်ခိုက်ခြင်းကို ထိုးနှက်ချက်တစ်ခုချင်းစီအလိုက် မှတ်တမ်းတင်ထားသည်။ မှတ်တမ်းတင်ခြင်းကို လွယ်ကူစေရန်အတွက် တိုက်ပွဲများကို ငါးစက္ကန့်စီရှိသော တိုက်ခိုက်ရေးအချီများ (Combat Rounds) အဖြစ် ပိုင်းခြားထားသည်။ ဤကာလအတွင်း တိုက်ခိုက်သူများသည် လက်ချင်းထိတိုက်ခိုက်မှုများ ပြုလုပ်နိုင်သည် သို့မဟုတ် ၎င်းတို့ကို ကာကွယ်နိုင်သည်၊ တိုက်ပွဲထဲသို့ ဝင်ရောက်နိုင်သည် သို့မဟုတ် ထွက်နိုင်သည်၊ အဝေးပစ်လက်နက် (Ranged Weapon) ကို ပစ်နိုင်သည် သို့မဟုတ် ပစ်ပေါက်နိုင်သည် အစရှိသည်တို့ လုပ်ဆောင်နိုင်သည်။ လက်တွေ့ဘဝနှင့် ဆင်တူစွာ၊ တိုက်ပွဲများ (တစ်ကြိမ်ဝင်ရောက်ပြီးနောက်) သည် မိနစ်များစွာကြာမြင့်ခြင်းထက် စက္ကန့်ပိုင်းအတွင်း ပြီးဆုံးလေ့ရှိသည်။ ဂိမ်းစည်းမျဉ်းများအရ ဆိုလျှင် ပါဝင်သူများသည် Initiative ကို ပြန်လည်သတ်မှတ်ရန်၊ ၎င်းတို့၏ နည်းဗျူဟာအခြေအနေကို ပြန်လည်ဆန်းစစ်ရန် သို့မဟုတ် ပြန်လည်မတိုက်ခိုက်မီ ခေတ္တအနားယူရန်အတွက် ဆုတ်ခွာခြင်းမပြုပါက တိုက်ခိုက်ရေးအချီအများအပြား ကြာမြင့်နိုင်သည်။ ကောင်းမွန်စွာလိုက်ဖက်သော ပြိုင်ဘက်များကြား တိုက်ပွဲများ သို့မဟုတ် ပြိုင်ဘက်များ၏ ဆက်တိုက်တိုက်ခိုက်မှုများသာ ပိုမိုကြာရှည်လေ့ရှိသည်။

## သေနတ်များ

_Mythras Imperative_ ဂိမ်းတွင် သေနတ်များကို အသုံးပြုပါက သေနတ်ပစ်ခတ်မှု၏ ပြင်းထန်မှုကို ထည့်သွင်းစဉ်းစားရန် အချက်အချို့ရှိပါသည်။ ပထမအချက်မှာ လူ့တုံ့ပြန်မှုများထက် ပိုမိုမြန်ဆန်စွာ သွားလာသော လေဆာရောင်ခြည်များ သို့မဟုတ် ကျည်ဆံများကို ရှောင်ရှားရန် မဖြစ်နိုင်သလောက်ပင်။ တင်ပြထားသည့် တိုက်ခိုက်ရေးစည်းမျဉ်းများကို အသုံးပြုပါက၊ ၎င်းသည် ရှောင်တိမ်းကျွမ်းကျင်မှု (Evade) ကို အသုံးပြု၍ ဧရိယာမှ ရှောင်တိမ်းရန် မကြိုးစားပါက မည်သည့်ထိမှန်မှုမဆို သားကောင်အား တစ်ခု သို့မဟုတ် ထို့ထက်ပိုသော အထူးအကျိုးသက်ရောက်မှုများ (Special Effects) ကို ချက်ချင်းခံစားရစေမည်ဟု ဆိုလိုသည်။ ၎င်းသည် လုပ်ဆောင်ချက်မှတ် (Action Point) စီးပွားရေးကိုလည်း ပြောင်းလဲစေသည်။ မိမိကိုယ်ကို တုံ့ပြန်မှုဖြင့် ကာကွယ်ရန် မှတ်များကို အသုံးမပြုနိုင်ပါက ဇာတ်ကောင်အများစုသည် ၎င်းတို့၏လုပ်ဆောင်ချက်အားလုံးကို ပစ်ခတ်ရန် အသုံးပြုပြီး ရန်သူများ၏ အရည်အချင်းမရှိခြင်းကို လောင်းကစားကြမည်ဖြစ်သည်။ ဤပုံစံပြောင်းလဲမှုကို ကိုင်တွယ်ရန် ဂိမ်းမာစတာများနှင့် ကစားသမားနှစ်ဦးစလုံးသည် သေနတ်တိုက်ခိုက်မှုအပေါ် ၎င်းတို့၏အမြင်နှင့် ချဉ်းကပ်ပုံကို ပြောင်းလဲရန် လိုအပ်နိုင်သည်။ မှတ်သားသင့်သည့်အချက်အချို့မှာ-

- သေနတ်တိုက်ခိုက်မှုသည် အန္တရာယ်ရှိသည်။
- ယေဘုယျအားဖြင့် လူများသည် အခြားသူများကို ပစ်ခတ်ခြင်းထက် မိမိကိုယ်ကို အပစ်မခံရအောင် ဦးစားပေးသည်။
- ကာကွယ်မှု၏ အသုံးအများဆုံးနည်းလမ်းမှာ ခိုလှုံရာယူခြင်းဖြစ်သည်။
- ထိုးစစ်ပစ်ခတ်မှု (Suppressing fire) သည် ရန်သူများ ပြန်မပစ်ခတ်နိုင်အောင် တားဆီးရန် အကောင်းဆုံးနည်းလမ်းဖြစ်သည်။
- သေနတ်များသည် မြဲမြံစွာ ထောက်ပြီး/သို့မဟုတ် ပစ်မှတ်ထားချိန်ယူခြင်းမရှိပါက အံ့ဩဖွယ်ကောင်းလောက်အောင် တိကျမှုမရှိပါ။
- သင့်ကို ပစ်ခတ်ခံရသောအခါ မကောင်းသည့်အရာများ ဖြစ်ပေါ်တတ်သည်။

မိမိကိုယ်ကို သေနတ်ဖြင့်ပစ်ခတ်မှုမှ ကာကွယ်ခြင်းသည် ကိုယ်ကာကွယ်ဝတ်စုံ၊ ခိုလှုံရာအသုံးပြုခြင်း၊ နှင့် ပြိုင်ဘက်များ ပြန်မပစ်နိုင်အောင် ကြိုးပမ်းခြင်းတို့ဖြစ်သည်။ ရလဒ်အနေဖြင့် သေနတ်တိုက်ပွဲအများစုသည် အလွန်နည်းဗျူဟာကျသည် - ထိုးစစ်ပစ်ခတ်မှုများစွာ၊ ဘေးတိုက်လှုပ်ရှားမှုများ၊ နှင့် နက်ရှိုင်းသော ခိုလှုံရာမှ ရန်သူများကို ထုတ်ပစ်ခြင်း - ဤကာလများအတွင်း ဇာတ်ကောင်များသည် ရန်သူများ ပစ်ခတ်မှု ရပ်သွားသည်အထိ အချီများစွာ ကုန်ဆုံးသွားပြီး တုံ့ပြန်နိုင်စွမ်းမရှိဘဲ ဖြစ်နေနိုင်သည်... ၎င်းကို ကစားသမားအချို့က အလွန်ငြီးငွေ့ဖွယ်ကောင်းသည်ဟု ယူဆနိုင်သည်။

သေနတ်ပစ်ခတ်မှုတစ်ခုလုံး ဒဏ်ရာမရှိဘဲ ပြီးဆုံးသွားခြင်း၊ တစ်ဖက်က ခဲယမ်းကုန်သွား၍ လက်နက်ချခြင်း သို့မဟုတ် တိုက်ပွဲမှ လုံးဝထွက်ပြေးခြင်းတို့ ဖြစ်နိုင်သည်။ အဘယ်ကြောင့်ဆိုသော် လူအနည်းငယ်ကသာ အလိုအလျောက်ပစ်ခတ်နိုင်သော လက်နက်တစ်ခုမှ ကျည်ဆံများဖြင့် အပေါက်ဖောက်ခံရမည့် အန္တရာယ်ကို ရင်ဆိုင်လိုကြသည်မဟုတ်လား။

ဤတိုက်ခိုက်ရေးစည်းမျဉ်းများတစ်လျှောက်တွင် သီးခြားလုပ်ဆောင်ချက်အမျိုးအစားများ၊ အထူးအကျိုးသက်ရောက်မှုများ (Special Effects) နှင့် အခြေအနေများသည် သေနတ်များဖြင့် တိုက်ခိုက်ခြင်းအတွက်သာ သက်ဆိုင်ပြီး ၎င်းတို့ကို သက်ဆိုင်ရာအတိုင်း အမှတ်အသားပြုထားသည်။

## တိုက်ခိုက်ရေးအချီများနှင့် ရှေ့ဦးတည်ချက်

တိုက်ခိုက်ရေးအချီများ (Combat Rounds) သည် အရေးကြီးသော ဝေါဟာရအချို့ကို အသုံးပြုသည်။

- **ရှေ့ဦးတည်ချက် (Initiative):** တိုက်ခိုက်ရေးအချီ၏ အလှည့်တစ်ခုအတွင်း မည်သူက မည်သည့်အချိန်တွင် လုပ်ဆောင်သည်ကို ဆုံးဖြတ်သော အစီအစဉ်။ ဂိမ်းမာစတာသည် ရှေ့ဦးတည်ချက်တန်ဖိုးများကို အမြင့်ဆုံးမှ အနိမ့်ဆုံးသို့ ရေတွက်သွားပြီး ပါဝင်သူတစ်ဦးစီသည် ၎င်းတို့၏နံပါတ်သို့ ရောက်ရှိသောအခါ ၎င်းတို့၏အလှည့် (Turn) ကို ယူရန်အခွင့်အရေးရရှိသည်။
- **အလှည့်များ (Turns):** ငါးစက္ကန့်ကြာ တိုက်ခိုက်ရေးအချီတစ်ခုစီအတွင်း ပါဝင်သူတစ်ဦးစီသည် ၎င်းတို့ပိုင်ဆိုင်သော လုပ်ဆောင်ချက်မှတ် (Action Points) အရေအတွက်နှင့် ၎င်းတို့ကို မည်သို့အသုံးပြုသည်အပေါ် မူတည်၍ အလှည့်တစ်ခု သို့မဟုတ် ထို့ထက်ပို၍ ယူရန် အခွင့်အရေးရှိသည်။ ၎င်းတို့၏အလှည့်တွင် ပါဝင်သူများသည် လုပ်ဆောင်ချက်တစ်ခု (Combat Action) ကို လုပ်ဆောင်ပြီး ၎င်းတွင် ကြေညာချက်၊ ဖြစ်နိုင်သော ရွေ့လျားမှု၊ လိုအပ်သော အံစာတုံးလှိမ့်ခြင်းများ (တုံ့ပြန်မှုများ (Reactions) အပါအဝင်) နှင့် လုပ်ဆောင်ချက်ကို ဖြေရှင်းခြင်းတို့ ပါဝင်သည်။ ပါဝင်သူတစ်ဦးဦးတွင် လုပ်ဆောင်ချက်မှတ်များ ကျန်ရှိနေပါက ကစားသမားများသည် ရှေ့ဦးတည်ချက်အစီအစဉ်အတိုင်း နောက်ထပ်အလှည့်တစ်ခုစီကို ယူနိုင်သည်။ လုပ်ဆောင်ချက်မှတ်များ မကျန်တော့ပါက သို့မဟုတ် ကျန်ရှိနေသောမှတ်များကို တုံ့ပြန်မှုများအတွက် သိမ်းဆည်းထားပါက နောက်ထပ်တိုက်ခိုက်ရေးအချီသို့ ဆက်လက်လုပ်ဆောင်သည်။

ဝေါဟာရတစ်ခုစီကို အောက်တွင် ပိုမိုအသေးစိတ် ဖော်ပြထားသည်။

#### ရှေ့ဦးတည်ချက်

ရှေ့ဦးတည်ချက် (Initiative) သည် တိုက်ခိုက်ရေးအချီတစ်ခုအတွင်း ပါဝင်သူများ၏ လုပ်ဆောင်ချက် အစီအစဉ်ကို ဆုံးဖြတ်သည်။ ၎င်းကို တိုက်ပွဲစတင်ချိန်တွင် လှိမ့်ရပြီး ဇာတ်ကောင်တစ်ဦးစီ မည်သည့်အချိန်တွင် လုပ်ဆောင်သည်ကို ဆုံးဖြတ်သည်။ အခြေအနေကို ပြောင်းလဲစေသည့် အရာတစ်ခုခု (ဥပမာ- သီးခြားတိုက်ခိုက်ရေးလုပ်ဆောင်ချက်များ သို့မဟုတ် အထူးအကျိုးသက်ရောက်မှုများ) မဖြစ်ပွားပါက ရှေ့ဦးတည်ချက်သည် ပြန်လည်လှိမ့်ရန် မလိုအပ်သည့်တိုင်အောင် ဆက်လက်ရှိနေမည်ဖြစ်သည်။

ရှေ့ဦးတည်ချက်ကို ပါဝင်သူတစ်ဦးစီက 1d10 လှိမ့်ပြီး ၎င်းတို့၏ ရှေ့ဦးတည်ချက်မွမ်းမံမှု (Initiative Modifier) ကို ပေါင်းထည့်ခြင်းဖြင့် တွက်ချက်သည်။ အမြင့်ဆုံးရလဒ်ရရှိသူက ဦးစွာလုပ်ဆောင်ပြီး ဒုတိယအမြင့်ဆုံးသူက ဆက်လက်လုပ်ဆောင်သည်၊ စသည်ဖြင့်။ ပါဝင်သူနှစ်ဦး သို့မဟုတ် ထို့ထက်ပို၍ ရမှတ်တူညီပါက DEX ပိုမိုမြင့်မားသူက ဦးစွာလုပ်ဆောင်မည်။ ဤသည်မှာ ဆက်လက်တူညီနေသေးပါက အမြင့်ဆုံးလှိမ့်သူက အခြားသူများထက် ဦးစွာသွားနိုင်ရန် အံစာတုံးတစ်ခုစီကို လှိမ့်စေပါ။

#### တိုက်ခိုက်ခြင်းတွင် လုပ်ဆောင်ချက်များ

ရှေ့ဦးတည်ချက်ကို ဆုံးဖြတ်ပြီးသည်နှင့် ပါဝင်သူများသည် တိုက်ခိုက်ရေးအချီတစ်ခုစီအတွင်း တိုက်ခိုက်ရေးလုပ်ဆောင်ချက်များစွာကို လုပ်ဆောင်နိုင်ခြေရှိသည်။ ဇာတ်ကောင်များအားလုံး အချီတစ်ခုလျှင် လုပ်ဆောင်ချက်မှတ် (Action Points) ၂ မှတ်ရရှိသော်လည်း ၎င်းတို့ မည်သည့်အချိန်တွင် လုပ်ဆောင်နိုင်သည်မှာ လုပ်ဆောင်ချက်သည် တက်ကြွစွာဆောင်ရွက်ခြင်း (proactive) သို့မဟုတ် တုံ့ပြန်ဆောင်ရွက်ခြင်း (Reactive) ဖြစ်ခြင်းအပေါ် မူတည်သည်။

တက်ကြွစွာဆောင်ရွက်ခြင်း (Proactive Actions) များကို ဇာတ်ကောင်၏ ကိုယ်ပိုင်အလှည့် (Turn) တွင်သာ ကြိုးပမ်းနိုင်သည်၊ ဆိုလိုသည်မှာ ၎င်း၏ ရှေ့ဦးတည်ချက် (Initiative) အတွင်းဖြစ်သည်။ တက်ကြွစွာဆောင်ရွက်ခြင်းများမှာ ဇာတ်ကောင်သည် စတင်လုပ်ဆောင်သူဖြစ်သည့် လုပ်ဆောင်ချက်များဖြစ်သည်၊ ဥပမာ- လက်နက်ဖြင့် တိုက်ခိုက်ခြင်း။

တုံ့ပြန်ဆောင်ရွက်ခြင်းများ (Reactive Actions) မှာ ဇာတ်ကောင်က မိမိကိုယ်ကို ဆန့်ကျင်ပြုလုပ်သော လုပ်ရပ်တစ်ခုကို တန်ပြန်ရန် သို့မဟုတ် ခုခံရန် လုပ်ဆောင်သည့် လုပ်ဆောင်ချက်များဖြစ်သည်။ ခြိမ်းခြောက်မှုတစ်ခုစီအတွက် တုံ့ပြန်မှုတစ်ကြိမ်သာ ခွင့်ပြုသည်။ ဥပမာအားဖြင့် တိုက်ခိုက်မှုကို ပိတ်ဆို့ရန် ကြိုးပမ်းခြင်း။

အသုံးမပြုရသေးသော လုပ်ဆောင်ချက်မှတ်များ (Action Points) သည် အချီတစ်ခုမှ နောက်တစ်ချီသို့ သယ်ဆောင်မသွားပါ။

## တိုက်ခိုက်ရေးလုပ်ဆောင်ချက်များ

တိုက်ခိုက်ရေးလုပ်ဆောင်ချက်များ (Combat Actions) သည် တိုက်ပွဲအတွင်း လုပ်ဆောင်နိုင်သည့် လုပ်ရပ်များဖြစ်သည်။ လက်နက်ဆွဲထုတ်ခြင်း၊ စာလုံးပေါင်းရွတ်ဆိုခြင်း သို့မဟုတ် တိုက်ခိုက်မှုတစ်ခုမှ ခုန်ရှောင်ခြင်းတို့သည် တိုက်ခိုက်ရေးလုပ်ဆောင်ချက်များ၏ ဥပမာများဖြစ်သည်။ အများစုသည် တိုက်ခိုက်ခြင်း၏ အစိတ်အပိုင်းများနှင့် သက်ဆိုင်သော်လည်း အချို့မှာ စာလုံးပေါင်းများနှင့် လက်နက်များ၏ နယ်ပယ်ပြင်ပရှိ လုပ်ဆောင်ချက်များနှင့် သက်ဆိုင်သည်။

ယခင်က ဖော်ပြခဲ့သည့်အတိုင်း ဇာတ်ကောင်တစ်ဦး အချီတစ်ခုလျှင် မည်မျှကြာကြာ လုပ်ဆောင်နိုင်သည်မှာ ၎င်းတို့ရရှိနိုင်သော လုပ်ဆောင်ချက်မှတ် (Action Points) များဖြင့် ကန့်သတ်ထားသည်။ ၎င်းတို့ကို တိုက်ခိုက်ရေးအချီ (Combat Round) တစ်လျှောက်လုံး အမျိုးမျိုးသော တိုက်ခိုက်ရေးလုပ်ဆောင်ချက်များ ပြုလုပ်ရန်အတွက် အသုံးပြုသည်။ တိုက်ခိုက်သူတစ်ဦး၏ လုပ်ဆောင်ချက်မှတ်များ ကုန်ဆုံးသွားသည်နှင့် ၎င်းတို့သည် ထိုအချီ၏ ကျန်ရှိသောကာလအတွက် လုပ်ဆောင်နိုင်တော့မည်မဟုတ်ဘဲ နောက်အချီစတင်ချိန်တွင် မှတ်များ ပြန်လည်သတ်မှတ်သည်အထိ စောင့်ရမည်။

အခြားနည်းဖြင့် မဖော်ပြထားပါက မည်သည့်တိုက်ခိုက်ရေးလုပ်ဆောင်ချက်မဆို ("အခမဲ့" လုပ်ဆောင်ချက်များမှလွဲ၍) လုပ်ဆောင်ချက်မှတ် တစ်မှတ် ကုန်ကျသည်။ ထို့ကြောင့် တိုက်ခိုက်သူများသည် ၎င်းတို့၏လုပ်ဆောင်ချက်မှတ်များကို မည်သို့နှင့် မည်သည့်အချိန်တွင် အသုံးပြုမည်ကို ဂရုတစိုက်စဉ်းစားရန် လိုအပ်သည်။ ၎င်းတို့သည် တိုက်ပွဲ၏ ပြောင်းလဲနေသော အခြေအနေများအပေါ် တက်ကြွစွာ တုံ့ပြန်လျက် ၎င်းတို့လိုချင်သလောက် ပြင်းထန်စွာ သို့မဟုတ် ခံစစ်အနေအထားဖြင့် လုပ်ဆောင်နိုင်သည်။

### တက်ကြွသောလုပ်ဆောင်ချက်များ

အောက်ပါတို့သည် ဇာတ်ကောင်တစ်ဦးသည် ၎င်း၏အလှည့်တွင် လုပ်ဆောင်ချက်မှတ်တစ်ခုကို သုံးစွဲခြင်းဖြင့် ကြိုးပမ်းနိုင်သည့် လုပ်ဆောင်ချက်များဖြစ်သည်။ စာလုံးပေါင်းရွတ်ဆိုခြင်း သို့မဟုတ် ပြန်လည်တင်ဆောင်ခြင်းကဲ့သို့သော အချို့သောလုပ်ဆောင်ချက်များသည် ပြီးမြောက်ရန် အလှည့်များစွာ ကြာနိုင်ပြီး အလှည့်တစ်ခုစီသည် ၎င်း၏ကိုယ်ပိုင်လုပ်ဆောင်ချက်မှတ် ကုန်ကျသည်။

#### တိုက်ခိုက်ခြင်း

ဇာတ်ကောင်သည် လက်ချင်းထိတိုက်ခိုက်လက်နက်ဖြင့် တိုက်ခိုက်ရန် သို့မဟုတ် အဝေးပစ်လက်နက် (Ranged Weapon) ကို အသုံးပြုရန် ကြိုးပမ်းနိုင်သည်။

#### ထောက်ခြင်း

ဇာတ်ကောင်သည် ခိုင်မာသောရပ်တည်ချက်ကိုယူပြီး လာမည့်တိုက်ခိုက်မှု၏ ဦးတည်ရာသို့ ကိုယ်ကိုမှီကာ ထောက်ထားသည်။ Knockback (နောက်ပြန်လွင့်ကျခြင်း) သို့မဟုတ် Leaping Attacks (ခုန်ပေါက်တိုက်ခိုက်ခြင်း) ကို ခုခံရန်အတွက် ဇာတ်ကောင်၏ SIZ ကို 50% ပိုကြီးသည်ဟု သတ်မှတ်သည်။ Bash Special Effect ကို ခုခံရာတွင် SIZ သည် နှစ်ဆဖြစ်သည်။

#### မှော်ပညာရွတ်ဆိုခြင်း/စွမ်းအားသုံးခြင်း

ဇာတ်ကောင်သည် စာလုံးပေါင်းရွတ်ဆိုရန်၊ သီးခြားစွမ်းအားများကို အသက်သွင်းရန် စသည်ဖြင့် ကြိုးပမ်းနိုင်သည်။ ရှုပ်ထွေးသောမှော်ပညာများမှာ ရွတ်ဆိုခြင်းကို ပြီးမြောက်ရန် လုပ်ဆောင်ချက်များစွာ လိုအပ်နိုင်သည်။ ပြီးဆုံးသည်နှင့် မှော်ပညာကို စာလုံးပေါင်းရွတ်ဆိုသူ၏ နောက်အလှည့်အထိ မည်သည့်အချိန်တွင်မဆို ထုတ်လွှတ်နိုင်သည်။ ထိုအချိန်တွင် ၎င်းကို နောက်ပိုင်းအကျိုးသက်ရောက်မှုအတွက် ထိန်းသိမ်းထားနိုင်သော်လည်း ၎င်းကို နောက်ပိုင်းထုတ်လွှတ်ရန် ပြင်ဆင်မှုအနေဖြင့် ထိန်းသိမ်းထားရန် Hold Magic/Power လုပ်ဆောင်ချက် (အောက်တွင်ကြည့်ပါ) လိုအပ်သည်။

#### အကွာအဝေးပြောင်းခြင်း

ဇာတ်ကောင်သည် ပြိုင်ဘက်တစ်ဦးနှင့် လက်ချင်းထိတိုက်ခိုက်မှုတွင် ပါဝင်နေခြင်းမရှိပါက သို့မဟုတ် အဝေးပစ်တိုက်ခိုက်မှုအခြေအနေတွင် ရန်သူ၏ပစ်ခတ်မှုကြောင့် ပိတ်မိနေခြင်းမရှိပါက ပြိုင်ဘက်ထံသို့ နီးကပ်စွာ ချဉ်းကပ်ရန် သို့မဟုတ် နောက်ဆုတ်ရန် ကြိုးပမ်းနိုင်သည်။

#### တိုက်ခိုက်ရန်ပြေးခြင်း (Charge)

တိုက်ခိုက်ရန်ပြေးခြင်း (Charge Action) သည် ဇာတ်ကောင်အား ပြေးနှုန်း သို့မဟုတ် ပြေးနှုန်းအမြန်ဖြင့် ထိတွေ့မှုအကွာအဝေးသို့ ရွေ့လျားခွင့်ပြုပြီး တိုက်ခိုက်ခြင်း၏ အရှိန်အဟုန်ကို အသုံးပြု၍ ပိုမိုအားကောင်းသော တိုက်ခိုက်မှုတစ်ခု ပြုလုပ်နိုင်သည်။ ဤစည်းမျဉ်းများကို အောက်ပါ [Charging](0005_Combat.md?id=charging) အပိုင်းတွင် ပိုမိုအသေးစိတ် ဖော်ပြထားသည်။

#### နှောင့်နှေးခြင်း

ဇာတ်ကောင်သည် Reactive Actions (တုံ့ပြန်မှု လုပ်ဆောင်ချက်များ) ကို နောက်ပိုင်းတွင် လုပ်ဆောင်ရန်အတွက် လုပ်ဆောင်ချက်တစ်ခု သို့မဟုတ် ထို့ထက်ပို၍ ထိန်းသိမ်းထားသည်။ ဥပမာအားဖြင့် Interrupt (ဝင်ရောက်စွက်ဖက်ခြင်း) သို့မဟုတ် Parry (ပိတ်ဆို့ခြင်း)။ နှောင့်နှေးခြင်း၏ လုပ်ဆောင်ချက်မှတ် (Action Point) ကုန်ကျစရိတ်များကို နောက်ဆုံးတွင် လုပ်ဆောင်သည့် လုပ်ရပ်များဖြင့် အကျုံးဝင်သည်။ နှောင့်နှေးထားသော လုပ်ဆောင်ချက်များကို ဇာတ်ကောင်၏ နောက်အလှည့်မတိုင်မီ မလုပ်ဆောင်ပါက ဇာတ်ကောင်သည် Dithered (အောက်တွင်ကြည့်ပါ) ဖြစ်သည်ဟု မှတ်ယူပြီး လုပ်ဆောင်ချက်မှတ် ဆုံးရှုံးသည်။

#### လေလွင့်ခြင်း (Dither)

အခြားလုပ်ဆောင်ချက်တစ်ခုကို မရွေးချယ်ပါက မူလရွေးချယ်မှုမှာ ဇာတ်ကောင်သည် အသုံးမဝင်သောအရာကို လုပ်ဆောင်ရင်း ၎င်း၏အလှည့်ကို အလဟဿဖြစ်စေခြင်း။

#### မှော်ပညာ/စွမ်းအားကို ထိန်းသိမ်းခြင်း

စာလုံးပေါင်းရွတ်ဆိုခြင်း ပြီးစီးသည်နှင့် ဇာတ်ကောင်သည် မှော်ပညာ သို့မဟုတ် စွမ်းအားကို ယာယီ ထိန်းချုပ်ထားနိုင်သည်၊ ၎င်းကို ထုတ်လွှတ်ရန် အကောင်းဆုံးအချိန်ကို စောင့်ဆိုင်းရင်း ဖြစ်သည်။ မှော်ပညာကို ဇာတ်ကောင်က နောက်ဆက်တွဲအလှည့်များတွင် ဤလုပ်ဆောင်ချက်ကို ဆက်လက်လုပ်ဆောင်နေသမျှ ကာလပတ်လုံး ထိန်းသိမ်းထားနိုင်သည်၊ သို့သော် ၎င်းသည် စာလုံးပေါင်းနှင့် သက်ဆိုင်ပါက ရန်သူက Counter Spell (တန်ပြန်စာလုံးပေါင်း) တုံ့ပြန်မှုကို အခမဲ့ အသုံးပြုခွင့်ပြုသည်။ ထိန်းသိမ်းထားသော စာလုံးပေါင်းကို ရွတ်ဆိုရန်အတွက် တကယ်တမ်း ကျွမ်းကျင်မှုလှိမ့်ခြင်းကို ရွတ်ဆိုသည့်အခါမှသာ ပြုလုပ်သည်။

#### စီးနင်းခြင်း

ဇာတ်ကောင်သည် စီးနင်းသော သားရဲ သို့မဟုတ် ယာဉ်ပေါ်သို့ တက်နိုင်သည် သို့မဟုတ် ဆင်းနိုင်သည်။ အထူးသဖြင့် ကြီးမားသော စီးနင်းစရာများမှာ ပြီးမြောက်ရန် အလှည့်များစွာ လိုအပ်နိုင်သည်။

#### ရွေ့လျားခြင်း

ပြိုင်ဘက်တစ်ဦးနှင့် ထိတွေ့မှုမရှိပါက ဇာတ်ကောင်သည် ဂိမ်းမာစတာက အခြေအနေနှင့် သင့်တော်သည်ဟု ယူဆသော မည်သည့်အကွာအဝေးကိုမဆို ရွေ့လျားနိုင်သည်။

#### အသာစီးရယူခြင်း (Outmaneuver)

ဇာတ်ကောင်သည် ပြိုင်ဘက်များစွာကို Evade skills (ရှောင်တိမ်းကျွမ်းကျင်မှု) ၏ အုပ်စုလိုက် ပြိုင်ဆိုင်လှိမ့်ခြင်း (Opposed Roll) တွင် ပါဝင်နိုင်သည်။ ၎င်း၏လှိမ့်ခြင်းကို မကျော်နိုင်သူများသည် ထိုတိုက်ခိုက်ရေးအချီ (Combat Round) တွင် ၎င်းကို တိုက်ခိုက်နိုင်မည်မဟုတ်ပေ။ အောက်ပါ [Outmaneuvering](0005_Combat.md?id=outmaneuvering) အပိုင်းကို ကြည့်ပါ။

#### လက်နက်ပြင်ဆင်ခြင်း

ဇာတ်ကောင်သည် လက်နက် သို့မဟုတ် အခြားအရာဝတ္ထုကို ပြန်လည်ယူဆောင်ခြင်း၊ ဆွဲထုတ်ခြင်း၊ သေနတ်အိမ်ထဲသို့ ထည့်ခြင်း၊ ပြန်ထုတ်ခြင်း သို့မဟုတ် ပြန်လည်တင်ဆောင်ခြင်းတို့ လုပ်ဆောင်နိုင်သည်။ အနီးအနားရှိ ပြုတ်ကျနေသော အရာဝတ္ထုကို ပြန်လည်ယူဆောင်ရန် လုပ်ဆောင်ချက် ၂ ခု လိုအပ်သည်- တစ်ခုက အရာဝတ္ထုဆီသို့ ရွေ့လျားပြီး ကောက်ယူရန်၊ နောက်တစ်ခုက ပြန်လည်အသင့်အနေအထားသို့ ရောက်ရှိရန်။ အချို့သော ဒုံးပျံလက်နက်များသည် ပြန်လည်တင်ဆောင်ရန် လုပ်ဆောင်ချက်များစွာ လိုအပ်သည်။

#### ပြန်လည်တင်ဆောင်ခြင်း (သေနတ်များ)

လက်နက်ပြင်ဆင်ခြင်း (Ready Weapon) ကဲ့သို့ပင်။ ပြန်လည်တင်ဆောင်ချိန်များသည် လက်နက်အမျိုးအစားပေါ်မူတည်၍ အလွန်ကွာခြားနိုင်သည်။ တစ်ချက်ပစ် အနက်ရောင်ယမ်းမှုတ်သေနတ်များသည် ခဲယမ်းမဂ္ဂဇင်း သို့မဟုတ် ပါဝါအရင်းအမြစ်ပြောင်းလဲခြင်းထက် ပြန်လည်တင်ဆောင်ရန် ပိုမိုနှေးကွေးမည်ဖြစ်သည်။

#### ခြေထောက်ပြန်လည်ထိန်းမတ်ခြင်း

ပြိုင်ဘက်တစ်ဦးနှင့် ထိတွေ့မှုမရှိပါက ဇာတ်ကောင်သည် ချော်လဲခြင်း သို့မဟုတ် လဲကျခြင်းမှ ၎င်း၏ခြေထောက်ကို အလိုအလျောက် ပြန်လည်ထိန်းမတ်နိုင်သည်။ ထိတွေ့မှုရှိပါက ဇာတ်ကောင်သည် မတ်တပ်မရပ်မီ ပြိုင်ဘက်နှင့် Brawn သို့မဟုတ် Athletics ၏ Opposed Roll (ပြိုင်ဆိုင်လှိမ့်ခြင်း) ကို အနိုင်ရရမည်။ ဤပြိုင်ဆိုင်မှုသည် ဖြေရှင်းရန်အတွက် လုပ်ဆောင်ချက်မှတ် (Action Points) မကုန်ကျပါ။

#### ခိုးကြောင်ခိုးဝှက်ကြည့်ခြင်း (သေနတ်များ)

ခိုးကြောင်ခိုးဝှက်ကြည့်ခြင်း (Sneak Peek) သည် လက်ရှိပုန်းအောင်းနေသူတစ်ဦးအား နည်းဗျူဟာအခြေအနေကို ဆုံးဖြတ်ရန်အတွက် လျှင်မြန်စွာ ကြည့်ရှုခွင့်ပေးသည်။ ဤလုပ်ဆောင်ချက်သည် လုံခြုံသည်ဟု ယူဆရပြီး ကြည့်ရှုသူအား ပုန်းအောင်းရာသို့ ပြန်မဝင်မီ Perception လှိမ့်ခြင်းတစ်ကြိမ် ပြုလုပ်ခွင့်ပြုသည်။ သို့သော် Perception စစ်ဆေးမှုသည် ချော်လဲခြင်း (fumble) ဖြစ်ပါက ၎င်းတို့သည် ခေါင်းကို မိုက်မဲစွာ ထုတ်ဖော်မိခြင်း သို့မဟုတ် ရန်သူက ပစ်ခတ်ရန် အချိန်လုံလောက်အောင် မြင်ကွင်းထဲတွင် ရှိနေခြင်းတို့ ဖြစ်စေသည်။

#### လက်နက်ကို တည်ငြိမ်အောင် ထိန်းခြင်း (သေနတ်များ)

ဇာတ်ကောင်အား မြဲမြံသော ပစ်ခတ်မှုအနေအထားကို ထူထောင်ပြီး အလိုအလျောက်လက်နက်ကို တည်ငြိမ်အောင် ထောက်ပံ့နိုင်စေသည်။ ၎င်းသည် ဇာတ်ကောင်အား အနည်းဆုံး တစ်စိတ်တစ်ပိုင်း ပွင့်နေပြီး ခိုလှုံရာဖြင့် လုံးဝကာကွယ်ထားခြင်းမရှိရန် လိုအပ်သည်။ အဓိက အကျိုးကျေးဇူးမှာ Burst သို့မဟုတ် Full-Automatic ပစ်ခတ်မှု၏ နောက်ပြန်ကန်ဒဏ်ကြေးကို တစ်ဆင့်လျှော့ချပေးခြင်းဖြစ်သည်။ ဤအကျိုးသက်ရောက်မှုသည် Brace (ထောက်ခြင်း) နှင့် ဆင်တူသည်၊ ၎င်း၏အကျိုးကျေးဇူးများသည် ဇာတ်ကောင်ရွေ့လျားသည်အထိ ဆက်လက်ရှိနေမည်ဖြစ်သည်။

#### ရုန်းကန်ခြင်း

ဇာတ်ကောင်သည် အချို့သော တိုက်ခိုက်မှုအမျိုးအစားများ သို့မဟုတ် အထူးအကျိုးသက်ရောက်မှုများ၏ သားကောင်ဖြစ်ပါက ၎င်းတို့သည် အခြေအနေမှ မိမိကိုယ်ကို ထုတ်နုတ်ရန် ကြိုးပမ်းနိုင်သည်။ ဥပမာအားဖြင့် ရန်သူ၏ လက်ဆုပ်အတွင်းမှ ရုန်းထွက်ခြင်း။

#### ပစ်ခတ်မှုပုံစံပြောင်းလဲခြင်း (သေနတ်များ)

ပစ်ခတ်မှုပုံစံပြောင်းလဲခြင်း (Switch Fire) သည် အလိုအလျောက် သို့မဟုတ် ဘက်စုံသုံးသေနတ်ကို အသုံးပြုသူအား ၎င်းတို့၏လက်နက်ပေါ်ရှိ ရွေးချယ်မှုခလုတ်ကို ပြောင်းလဲ၍ အခြားပစ်ခတ်မှုပုံစံများကို အခွင့်ကောင်းယူရန် ခွင့်ပြုသည်။ ဥပမာအားဖြင့် သေနတ်တစ်လက်ကို semi-automatic မှ burst သို့ ပြောင်းနိုင်သည်၊ မတူညီသော ခဲယမ်းအမျိုးအစားများကို ရွေးချယ်နိုင်သည်၊ သို့မဟုတ် ပေါင်းစပ်လက်ပစ်ဗုံးပစ်ကိရိယာကဲ့သို့သော အပိုစနစ်တစ်ခုကို အွန်လိုင်းသို့ ယူဆောင်လာနိုင်သည်။ ခေတ်မီလက်နက်များသည် ပုံစံပြောင်းရန် ဇာတ်ကောင်၏အလှည့်တွင် လုပ်ဆောင်ချက်မှတ် (Action Point) တစ်မှတ် လိုအပ်သော်လည်း လုံလောက်စွာ တိုးတက်သော အဆင့်မြင့်နည်းပညာသုံးသေနတ်များသည် အီလက်ထရွန်နစ်စနစ်များ သို့မဟုတ် အသုံးပြုသူ၏စိတ်ကို စိတ်ပိုင်းဆိုင်ရာဖြင့် ဖတ်ရှုခြင်းဖြင့် အခမဲ့လုပ်ဆောင်ချက် (Free Action) အဖြစ် ခွင့်ပြုနိုင်သည်။

#### ခိုလှုံရာယူခြင်း

ခိုလှုံရာယူခြင်း (Take Cover) သည် တက်ကြွသောလုပ်ဆောင်ချက်တစ်ခုဖြစ်ပြီး သေနတ်ပစ်သူအား ၎င်းတို့၏ချက်ချင်းပတ်ဝန်းကျင်တွင် ရရှိနိုင်သော မည်သည့်ခိုလှုံရာနောက်သို့မဆို ပုန်းအောင်းနိုင်စေကာ ဝင်လာသောပစ်ခတ်မှုများမှ အတိုင်းအတာတစ်ခုအထိ ကာကွယ်မှုရရှိစေသည်။ Evade (ရှောင်တိမ်းခြင်း) နှင့်မတူဘဲ ၎င်းသည် အသုံးပြုသူအား လဲကျနေအောင် မထားခဲ့သော်လည်း ခိုလှုံရာတစ်မျိုးမျိုး ရရှိနိုင်မှုအပေါ် မူတည်သည်၊ ဥပမာ- စင်္ကြံတစ်ခု၏ထောင့်တစ်ဝိုက်တွင် ပုန်းအောင်းခြင်း သို့မဟုတ် ဘားတစ်ခုရှိ စားပွဲနောက်တွင် ဒူးထောက်ခြင်း။ အခြေအနေအရ ရရှိနိုင်သော ခိုလှုံရာသည် ဇာတ်ကောင်ကို လုံးဝကာကွယ်ရန် လုံလောက်နိုင်သည် သို့မဟုတ် မလုံလောက်နိုင်။ ခိုလှုံရာအမျိုးအစားသည် ၎င်း၏ကာကွယ်မှုအရည်အသွေးများကိုလည်း ဆုံးဖြတ်ပေးမည်။ ဥပမာအားဖြင့်၊ သံမဏိတံခါးထူကြီးတစ်ခုသည် ကျည်ဆံများကို မထိုးဖောက်နိုင်သော်လည်း သစ်သားနံရံပါးတစ်ရံသည် သံချပ်ကာမှတ် (Armor Points) ၄ မှတ်သာ ပေးနိုင်ဖွယ်ရှိသည်။ အချို့သောပစ္စည်းများ၏ ကာကွယ်မှုအရည်အသွေးများနှင့် ပတ်သက်သည့် အထွေထွေလမ်းညွှန်ချက်များအတွက် [Inanimate Objects](0004_Game_System.md?id=inanimate-objects) အပိုင်းရှိ ဇယားကို ကြည့်ပါ။

### တုံ့ပြန်လုပ်ဆောင်ချက်များ

ဤစာရင်းသည် တိုက်ခိုက်ရေးအချီ (Combat Round) အတွင်း မည်သည့်အချိန်တွင်မဆို နီးကပ်လာသော ခြိမ်းခြောက်မှုတစ်ခုကို တုံ့ပြန်မှုအဖြစ် အသုံးပြုနိုင်သည့် တုံ့ပြန်မှုများ (Reactions) ကို သတ်မှတ်သည်။ ယခင်စာရင်းတွင်ကဲ့သို့ တုံ့ပြန်မှုတစ်ခုလုပ်ဆောင်ရန် လုပ်ဆောင်ချက်မှတ် (Action Point) တစ်မှတ် ကုန်ကျသည်။

#### မှော်ပညာကို တန်ပြန်ခြင်း

ဇာတ်ကောင်သည် ဝင်လာသော စာလုံးပေါင်းကို ဖျက်ရန် သို့မဟုတ် တန်ပြန်ရန် ကြိုးပမ်းနိုင်သည်။ ၎င်းသည် တန်ပြန်မှော်ပညာတွင် ရွတ်ဆိုချိန် (Casting Time) တစ်အလှည့်ရှိသည်ဟု ယူဆသည်၊ သို့မဟုတ်ပါက ၎င်းကို ကြိုတင်ပြင်ဆင်ပြီး Hold Magic Action ကို အသုံးပြု၍ ယာယီထိန်းသိမ်းထားရမည်။ ဤနည်းဖြင့် မှော်ပညာကို အောင်မြင်စွာ ကြားဖြတ်ခြင်းသည် ပစ်မှတ်များစွာ သို့မဟုတ် သက်ရောက်မှုဧရိယာများရှိသော စာလုံးပေါင်းများပင်ဖြစ်စေ စာလုံးပေါင်းတစ်ခုလုံးကို ပယ်ဖျက်သည်ဟု ယူဆရသည်။

#### ရှောင်တိမ်းခြင်း

ဇာတ်ကောင်သည် ဝင်လာသော ဒုံးပျံများ သို့မဟုတ် တိုက်ခိုက်ရန်ပြေးခြင်း (charging attack) ကဲ့သို့သော ခြိမ်းခြောက်မှုများမှ မိမိကိုယ်ကို လွတ်အောင် ခုန်ရှောင်ရန် ၎င်းတို့၏ ရှောင်တိမ်းကျွမ်းကျင်မှု (Evade skill) ကို အသုံးပြုနိုင်သည်။ ၎င်းသည် စွမ်းရည် (Ability) သို့မဟုတ် အထူးအခြေအနေများဖြင့် လျော့ပါးသက်သာခြင်းမရှိပါက ဇာတ်ကောင်အား လဲကျနေအောင် ဖြစ်စေသည်။ ထို့ကြောင့် ဇာတ်ကောင်၏ နောက်အလှည့်ကို ပုံမှန်အားဖြင့် Regain Footing Action (ခြေထောက်ပြန်ထိန်းမတ်ခြင်းလုပ်ဆောင်ချက်) ကို အသုံးပြု၍ ပြန်မတ်တပ်ရပ်ရန် ကုန်ဆုံးစေသည်။

#### ဝင်ရောက်စွက်ဖက်ခြင်း

နှောင့်နှေးနေသော ဇာတ်ကောင်များအတွက်သာ။ ဤတုံ့ပြန်မှု လုပ်ဆောင်ချက် (Reactive Action) သည် ရန်သူ၏အလှည့် (Turn) ကို မည်သည့်အမှတ်တွင်မဆို ရပ်တန့်စေပြီး နှောင့်နှေးထားသော အလှည့် လုပ်ဆောင်ချက်ကို လုပ်ဆောင်စေသည်။ နည်းဗျူဟာအခြေအနေတွင် ပြောင်းလဲမှုမရှိဟု ယူဆပါက ဇာတ်ကောင်၏ လုပ်ဆောင်ချက် ပြီးဆုံးပြီးနောက် ရန်သူသည် ၎င်း၏အလှည့်ကို ဆက်လက်လုပ်ဆောင်သည်။ မူလကြေညာချက်ကို ဆက်လက်လုပ်ဆောင်နိုင်ခြင်းမရှိပါက ရန်သူ၏ လုပ်ဆောင်ချက်မှတ် (Action Point) သည် အလဟဿဖြစ်သွားသည်။ နှောင့်နှေးနေသော ဇာတ်ကောင်၏ အနီးနားမှ ဖြတ်သွားသော မည်သူ့ကိုမဆို လက်နက်ရောက်နိုင်သည့် အကွာအဝေးအတွင်း တိုက်ခိုက်ရန် အခွင့်အရေးအဖြစ်လည်း ဝင်ရောက်စွက်ဖက်ခြင်းကို အသုံးပြုနိုင်သည်။

#### ပိတ်ဆို့ခြင်း

ဇာတ်ကောင်သည် ဝင်လာသော တိုက်ခိုက်မှုကို ပိတ်ဆို့ခြင်း၊ ကိုယ်ကို မှီခြင်း၊ ခေါင်းငုံ့ခြင်းနှင့် ဘေးတိုက်ခြေလှမ်းများ ပေါင်းစပ်အသုံးပြု၍ ထိုးနှက်ချက်ကို အနည်းဆုံးဖြစ်အောင် ကာကွယ်ရန် ကြိုးပမ်းနိုင်သည်။

### အခမဲ့လုပ်ဆောင်ချက်များ

အခမဲ့လုပ်ဆောင်ချက်များ (Free Actions) (အောက်တွင်ဖော်ပြထားသည်) ကို တိုက်ခိုက်ရေးအချီ (Combat Round) အတွင်း မည်သည့်အချိန်တွင်မဆို လုပ်ဆောင်နိုင်ပြီး လုပ်ဆောင်ချက်မှတ် (Action Points) မကုန်ကျပါ။

#### အခြေအနေကို ဆန်းစစ်ခြင်း

ထိတွေ့မှုမရှိပါက ဇာတ်ကောင်သည် Perception roll (အာရုံခံစားမှုလှိမ့်ခြင်း) ကို လုပ်ဆောင်ချက်မှတ် ကုန်ကျခြင်းမရှိဘဲ လုပ်ဆောင်နိုင်သည်။ အောင်မြင်ပါက နည်းဗျူဟာအခြေအနေတွင် သက်ဆိုင်ရာပြောင်းလဲမှုများ (ဥပမာ- ရန်သူတစ်ဦး တိုက်ခိုက်ရန်ပြေးခြင်း (charge) စတင်နေသည်ကို သတိပြုမိခြင်း) ကို ဖော်ပြသည်။ ပြေးနေစဉ် ဆန်းစစ်ခြင်းသည် Hard Perception roll (ခက်ခဲသောအာရုံခံစားမှုလှိမ့်ခြင်း) ကို ဖြစ်ပေါ်စေပြီး Sprinting (အပြေးမြန်ခြင်း) သည် Formidable (ကြောက်မက်ဖွယ်) ဖြစ်သည်။

#### ပစ္စည်းချခြင်း

ပစ္စည်းတစ်ခုကို ချခြင်းသည် အခမဲ့လုပ်ဆောင်ချက်တစ်ခုဖြစ်သည်။

#### အချက်ပြခြင်း

ထိတွေ့မှုမရှိပါက ပါဝင်သူတစ်ဦး သို့မဟုတ် ထို့ထက်ပို၍ အချက်ပြခြင်း သို့မဟုတ် အချက်ပြခြင်း (အကယ်၍ ၎င်းတို့သည် အချက်ပြမှုကို သိမြင်နိုင်ပါက) သည် အခမဲ့လုပ်ဆောင်ချက်ဖြစ်သည်။

#### စကားပြောခြင်း

ဇာတ်ကောင်တစ်ဦးသည် တိုက်ခိုက်နေစဉ် မည်သည့်အချိန်တွင်မဆို စကားပြောနိုင်သည်။ သို့သော် ပြောဆိုသည်များကို ငါးစက္ကန့် သို့မဟုတ် ထို့ထက်နည်းသော အချိန်အတွင်း ပြောဆိုနိုင်သည့် စကားစုတိုများဖြင့် ကန့်သတ်သင့်သည်။ ဥပမာအားဖြင့်၊ 'မင်းနောက်ကွယ်ကို ကြည့်လိုက်' သို့မဟုတ် 'မင်းကို ငရဲရောက်ပါစေ' ဟူ၍ဖြစ်သည်။

#### ကံကောင်းမှတ်အသုံးပြုခြင်း

ကံကောင်းမှတ် (Luck Point) ကို အသုံးပြုခြင်း (ဥပမာ- သီးခြားရလဒ်တစ်ခုကို ပြန်လှိမ့်ရန်) သည် အခမဲ့လုပ်ဆောင်ချက်တစ်ခုဖြစ်သည်။

#### ကာကွယ်သည့်နေရာ

ဇာတ်ကောင်သည် ၎င်း၏လက်နက်များထဲမှ တစ်ခုကို ထိုဧရိယာကို ပိတ်ဆို့ရန်အတွက် မြဲမြံစွာထားခြင်းဖြင့် သီးခြားထိမှန်သည့်နေရာ (Hit Location) ကို ကာကွယ်သည်။ ထိုနေရာသို့ ထိမှန်သည့် မည်သည့်ထိုးနှက်ချက်မဆို ၎င်း၏အရွယ်အစားရှိသော တန်ပြန်လက်နက်တစ်ခုအတွက် ပုံမှန်အတိုင်း ထိခိုက်မှု (damage) အလိုအလျောက် လျှော့ချခံရသည်။ ကာကွယ်မှုသည် ထိုလက်နက်ကို တိုက်ခိုက်ရန် သို့မဟုတ် တက်ကြွစွာ တန်ပြန်ရန် အသုံးပြုသည်အထိ ဆက်လက်ရှိနေမည်။ ကာကွယ်သည့်နေရာကို သတ်မှတ်ခြင်း သို့မဟုတ် ပြောင်းလဲခြင်းကို ပြိုင်ဘက်က ဇာတ်ကောင်ကို တိုက်ခိုက်ရန် လှိမ့်မမီမီ လုပ်ဆောင်ရမည်။ ၎င်းတို့၏ ဒီဇိုင်းကြောင့် ဒိုင်းများသည် နေရာများစွာကို ကာကွယ်နိုင်သည်။

နောက်ထပ်ရှင်းပြချက်အတွက် အောက်ပါ [Passive Blocking](0005_Combat.md?id=passive-blocking) ကို ကြည့်ပါ။

## ရွေ့လျားမှု

အောက်ပါ စည်းမျဉ်းများသည် တိုက်ခိုက်ခြင်းတွင် ထိတွေ့မှု (engagement) ကို ထိန်းချုပ်သည်။

- ရွေ့လျားမှုကို Move (ရွေ့လျားခြင်း) သို့မဟုတ် Charge (တိုက်ခိုက်ရန်ပြေးခြင်း) လုပ်ဆောင်ချက်ဖြင့် လုပ်ဆောင်သည်။
- Move Action (ရွေ့လျားခြင်းလုပ်ဆောင်ချက်) သည် ဇာတ်ကောင်အား ၎င်းတို့၏ အခြေခံရွေ့လျားနှုန်း (Base Movement Rate) နှင့် ညီမျှသော အကွာအဝေးကို ရွေ့လျားခွင့်ပြုသည်။
- ရွေ့လျားမှုကို ထိတွေ့မှုမရှိသော ဇာတ်ကောင်များမှသာ လုပ်ဆောင်နိုင်သည် (Outmaneuvering ကဲ့သို့သော အထူးအခြေအနေများမှလွဲ၍)။
- အချို့သော အခြေအနေများ (တိုက်ခိုက်ရန်ပြေးခြင်း (charging) ကဲ့သို့) မှလွဲ၍ ဇာတ်ကောင်သည် ရည်ရွယ်ထားသော ပြိုင်ဘက်၏ ထိတွေ့မှုအကွာအဝေးသို့ ဝင်ရောက်သည်နှင့် သိသာထင်ရှားသော လှုပ်ရှားမှုအားလုံး ရပ်တန့်သွားသည်။
- ထိတွေ့မှုမှတဆင့် တိုက်ခိုက်ရန်ပြေးခြင်းမရှိပါက ပြိုင်ဘက်၏ ထိတွေ့မှုနယ်မြေသို့ ဝင်ရောက်ခြင်းသည် တစ်ဦးကိုတစ်ဦး ထိတွေ့မှုထဲသို့ ရောက်ရှိစေသည်။
- ထိတွေ့ပြီးသည်နှင့် ဇာတ်ကောင်များသည် ပြိုင်ဘက်နှင့် ဝေးရာသို့ မရွေ့လျားနိုင်ပါ၊ ဦးစွာ လက်ချင်းထိတိုက်ခိုက်မှုမှ ဆုတ်ခွာခြင်းမပြုမီ။
- နှောင့်နှေးခြင်း (Delay) တိုက်ခိုက်ရေးလုပ်ဆောင်ချက်ကို အသုံးပြုနေသော ထိတွေ့မှုမရှိသော ရန်သူတစ်ဦးကို ကျော်လွန်သွားရန် ကြိုးပမ်းခြင်းသည် (သီးခြားခြွင်းချက်အနေဖြင့်) ထိုရန်သူအား ရွေ့လျားနေသော ဇာတ်ကောင်များကို ၎င်းတို့ဖြတ်သွားစဉ် တိုက်ခိုက်ရန် သို့မဟုတ် ဇာတ်ကောင်တစ်ဦး၏ တိုးတက်မှုကို ပိတ်ဆို့ရန် ခွင့်ပြုပြီး နောက်အလှည့် သို့မဟုတ် အချီတွင် လက်ချင်းထိတိုက်ခိုက်မှုကို ဖြစ်ပေါ်စေသည်။

### အခြေအနေအလိုက် ရွေ့လျားနှုန်းများ

ဤရွေ့လျားနှုန်းများသည် အခြေအနေအလိုက် ဖြစ်ပေါ်ပြီး အမြဲတမ်း ရွေးချယ်စရာမရှိပါ။ အများစုကို အခြားနေရာများတွင် အသေးစိတ်ဖော်ပြထားသော်လည်း အဆင်ပြေစေရန်အတွက် ဤနေရာတွင် ထည့်သွင်းထားသည်။ ဤအခြေအနေအလိုက်နှုန်းများစွာကို သတ်မှတ်ထားသော ကန့်သတ်ချက်များအောက်တွင် ပုံမှန်ရွေ့လျားနှုန်းအပြင် အသုံးပြုနိုင်သည်။ ထိုသို့သော အချက်အလက်များကို အောက်ပါ သီးခြားဖော်ပြချက်များတွင် အသေးစိတ်ဖော်ပြထားသည်။

#### တောင်တက်ခြင်း

တက်နိုင်သော မျက်နှာပြင် သို့မဟုတ် အရာဝတ္ထုတစ်ခု၏ ဘေးတွင် ရွေ့လျားသော သို့မဟုတ် အလှည့်ကို စတင်သော ဇာတ်ကောင်သည် အခြေခံရွေ့လျားနှုန်း (Base Movement Rate) နှင့် ညီမျှသော နှုန်းဖြင့် တက်နိုင်သည်။ ဇာတ်ကောင်သည် လမ်းလျှောက်နှုန်းထက် ပိုမြန်စွာ ရွေ့လျားခဲ့ခြင်းမရှိနိုင်ဘဲ ကျန်ရှိသော လှုပ်ရှားမှုရှိရမည်၊ တက်နိုင်သော အကွာအဝေးကို အသုံးပြုပြီးသား လှုပ်ရှားမှုဖြင့် လျှော့ချသည်။ ဝတ်ဆင်ထားသော သံချပ်ကာသည် ဝတ်ဆင်ထားသော သံချပ်ကာမှတ် (Armor Points) စုစုပေါင်းပေါ်မူတည်၍ တောင်တက်သူများကို အဟန့်အတားဖြစ်စေသည်။

#### တောင်တက်ခြင်း

_မူရင်းစာသားမှာ "ဝတ်ဆင်ထားသော သံချပ်ကာသည် တောင်တက်သူများကို ဝတ်ဆင်ထားသော အလေးဆုံးပစ္စည်း၏ သံချပ်ကာမှတ် (Armor Points) နှင့် ညီမျှသော ပမာဏဖြင့် အဟန့်အတားဖြစ်စေသည်။" ဟူ၍ဖြစ်သည်။_

#### တွားသွားခြင်း

#### ခုန်ခြင်း

ခုန်နိုင်သော အတားအဆီးတစ်ခု၏ ဘေးတွင် ရွေ့လျားသော သို့မဟုတ် အလှည့်ကို စတင်သော ဇာတ်ကောင်သည် ၎င်းကို ကျော်ခုန်ရန် ကြိုးပမ်းနိုင်သည်။ ဇာတ်ကောင်သည် ခုန်မည့်အမှတ်အထိ မည်သည့်နှုန်းဖြင့်မဆို ရွေ့လျားနေနိုင်သည်။

အောင်မြင်သော Athletics လှိမ့်ခြင်း (Athletics roll) သည် မိမိကိုယ်ပိုင်အရပ်၏ နှစ်ဆအထိ အလျားလိုက်ခုန်နိုင်သည် သို့မဟုတ် မိမိကိုယ်ပိုင်အရပ်၏ တစ်ဝက်အထိ ဒေါင်လိုက်ခုန်နိုင်သည် (အကယ်၍ ဇာတ်ကောင်သည် အနည်းဆုံး ငါးမီတာ ပြေးဆွဲနိုင်ခြေရှိပါက)။ ရပ်နေသည့်အနေအထားမှ ခုန်ပါက ဤအကွာအဝေးများကို တစ်ဝက်လျှော့ချသည်။

သယ်ဆောင်ထားသော စုစုပေါင်း 'အရာဝတ္ထု' ၏ ငါးပုံတစ်ပုံဖြင့် ခုန်နိုင်သည့်အကွာအဝေးကို မီတာများဖြင့် လျှော့ချပါ။ ဝတ်ဆင်ထားသော သံချပ်ကာသည် ခုန်သူများကို အဟန့်အတားဖြစ်စေပြီး ပေါ့ပါးသော သံချပ်ကာဖြင့် ဖုံးအုပ်ထားသော နေရာတိုင်းကို အထက်တွင်ဖော်ပြခဲ့သည့် တွက်ချက်မှုတွင် 'အရာဝတ္ထု' ၁ ခုအဖြစ် မှတ်ယူပြီး လေးလံသော သံချပ်ကာသည် 'အရာဝတ္ထု' ၂ ခုအဖြစ် မှတ်ယူသည်။

#### ရေကူးခြင်း

ရေနံတားတစ်ခု၏ ဘေးတွင် ရွေ့လျားသော သို့မဟုတ် အလှည့်ကို စတင်သော ဇာတ်ကောင်သည် အခြေခံရွေ့လျားနှုန်း (Base Movement Rate) နှင့် ညီမျှသော နှုန်းဖြင့် ရေကူးနိုင်သည်၊ အသုံးပြုပြီးသော လှုပ်ရှားမှုများကို နုတ်ပြီးဖြစ်သည်။ နောက်ထပ်အချက်အလက်များအတွက် [Skills](0003_Skills.md?id=swim-strcon) အပိုင်းရှိ ရေကူးဖော်ပြချက်ကို ကြည့်ပါ။

ပုံမှန်အခြေအနေများအောက်တွင် ဇာတ်ကောင်၏ အချီတစ်ခုလျှင် အမြင့်ဆုံးရေကူးနှုန်းသည် ၎င်းတို့၏ ရွေ့လျားနှုန်း (Movement Rate) နှင့် ညီမျှသည်၊ ဇာတ်ကောင်၏ STR ထက် ပိုမိုသယ်ဆောင်ထားသော 'အရာဝတ္ထု' တစ်ခုစီအတွက် -၁ လျှော့ချသည်။ ဝတ်ဆင်ထားသော သံချပ်ကာသည် ရေကူးသူများကို အဟန့်အတားဖြစ်စေပြီး ပေါ့ပါးသော သံချပ်ကာဖြင့် ဖုံးအုပ်ထားသော နေရာတိုင်းကို အပို 'အရာဝတ္ထု' ၁ ခု သယ်ဆောင်ထားသည်ဟု မှတ်ယူပြီး လေးလံသော သံချပ်ကာဖြင့် ဖုံးအုပ်ထားသော နေရာတိုင်းကို အပို 'အရာဝတ္ထု' ၂ ခု သယ်ဆောင်ထားသည်ဟု မှတ်ယူသည်။ စုစုပေါင်း ENC က ရေကူးအကွာအဝေးကို ၀ သို့မဟုတ် ထို့ထက်နည်းအောင် လျှော့ချပါက ဇာတ်ကောင်သည် အလိုအလျောက် ရေနစ်ခြင်း စတင်သည် (ကြည့်ရန် [Asphyxiation](0004_Game_System.md?id=asphyxiation))။

## တိုက်ခိုက်ခြင်း မည်သို့လုပ်ဆောင်သည်

_Mythras Imperative_ တွင် တိုက်ခိုက်ခြင်းကို ဖြေရှင်းရာတွင် ထိုးနှက်ချက် သို့မဟုတ် ပိတ်ဆို့ခြင်း တစ်ခုစီသည် ဓားထိုးခြင်း၊ လှည့်ပတ်ခုတ်ခြင်း၊ စာလုံးပေါင်းရွတ်ဆိုခြင်း သို့မဟုတ် လက်နက်တစ်ချက်ပစ်ခြင်းတို့ကို ကိုယ်စားပြုသည်။ မည်သည့်ထိုးစစ်လုပ်ဆောင်ချက်မဆို တုံ့ပြန်မှုဖြင့် ခုခံရန် အခွင့်အရေးကို ခွင့်ပြုသည်။ ထို့ကြောင့် ဇာတ်ကောင်တစ်ဦးသည် ပြိုင်ဘက်ကို တိုက်ခိုက်နိုင်ပါကပင် ရန်သူအား ၎င်း၏ထိုးနှက်ချက် မကျရောက်မီ ပိတ်ဆို့နိုင်သည်ရှိမရှိ ကြည့်ရှုရန် ၎င်းတို့၏ကိုယ်ပိုင်လှိမ့်ခြင်းကို ခွင့်ပြုထားသည်။ တူညီသော ဒဿနကို ဒုံးပျံလက်နက်များက ခိုလှုံရာသို့ ခုန်ဆင်းသော ပစ်မှတ်များကို တိုက်ခိုက်သည်ဖြစ်စေ ဧရာမရေဘဝဲ၏ သံလက်ဆုပ်မှ လွတ်မြောက်ရန် ကြိုးစားသည်ဖြစ်စေ အသုံးပြုသည်။

### တိုက်ခိုက်မှုများနှင့် ပိတ်ဆို့ခြင်းများ

လက်ချင်းထိတိုက်ခိုက်ခြင်း (Close combat) ကို အောက်ပါ အဆင့်ဆင့်ဖြင့် ကိုင်တွယ်ဖြေရှင်းသည်။

1.  ၎င်း၏အလှည့်တွင် တိုက်ခိုက်သူသည် လုပ်ဆောင်ချက်မှတ် (Action Point) တစ်မှတ်ကို သုံးစွဲပြီး ၎င်း၏ တိုက်ခိုက်ရေးကျွမ်းကျင်မှု (Combat Skill) ဖြင့် လှိမ့်ကာ ရလဒ်ကို မှတ်သားသည်။
2.  လိုချင်ပါက ကာကွယ်သူသည် လုပ်ဆောင်ချက်မှတ်တစ်မှတ်ကို Parry (ပိတ်ဆို့ခြင်း) အတွက် သုံးစွဲပြီး ၎င်း၏ တိုက်ခိုက်ရေးကျွမ်းကျင်မှုဖြင့် လှိမ့်ကာ ရလဒ်ကို မှတ်သားသည်။
3.  ရလဒ်များ၏ အောင်မြင်မှုအဆင့်ကို Differential Roll (ကွာခြားမှုလှိမ့်ခြင်း) အရ နှိုင်းယှဉ်သည်။
4.  မည်သည့်ကွာခြားချက်မဆို ပိုမိုသာလွန်သော လှိမ့်ခြင်းရှိသည့် အောင်မြင်သော တိုက်ခိုက်သူအား Special Effects (အထူးအကျိုးသက်ရောက်မှုများ) တစ်ခု သို့မဟုတ် ထို့ထက်ပို၍ ရရှိစေသည်။
5.  တိုက်ခိုက်သူသည် Success (အောင်မြင်ခြင်း) သို့မဟုတ် Critical (ပြင်းထန်အောင်မြင်ခြင်း) ရရှိပါက လက်နက်ထိခိုက်မှု (damage) ကို လှိမ့်ပြီး ၎င်း၏ Damage Modifier (ထိခိုက်မှုမွမ်းမံမှု) (ရှိပါက) ကို အသုံးပြုနိုင်သည်။ သက်ဆိုင်ပါက ထိုးနှက်ချက်အတွက် ထိမှန်သည့်နေရာ (Hit Location) ကို ဆုံးဖြတ်သည်။
6.  ကာကွယ်သူသည် Success (အောင်မြင်ခြင်း) သို့မဟုတ် Critical (ပြင်းထန်အောင်မြင်ခြင်း) ရရှိပါက ဖြစ်ပေါ်လာသော ထိခိုက်မှုများကို ပါဝင်သော လက်နက်များ၏ နှိုင်းယှဉ်အရွယ်အစားအရ လျှော့ချသည်။
7.  ကျန်ရှိသော ထိခိုက်မှုများကို သဘာဝကာကွယ်မှု သို့မဟုတ် ဝတ်ဆင်ထားသော ကာကွယ်မှု၏ သံချပ်ကာမှတ် (Armor Points) ဖြင့် လျှော့ချသည်။

မည်သည့် အထူးအကျိုးသက်ရောက်မှုများ (Special Effects) ကိုမဆို ထိခိုက်မှု (damage) ဖြစ်ပေါ်သည်ရှိမရှိအပေါ် မူတည်သည်မဟုတ်ကြောင်း သတိပြုပါ။ ကာကွယ်သူက ပိုမိုမြင့်မားသော အောင်မြင်မှုအဆင့်ကို ရရှိသော်လည်း ဒဏ်ရာရနိုင်ခြေရှိသည်။

### လက်နက်အရွယ်အစား

_Mythras Imperative_ ရှိ လက်နက်တိုင်းသည် ၎င်းပေးနိုင်သော ထိခိုက်မှုမှသည် ၎င်းကို ကိုင်စွဲရန် လိုအပ်သော လက်အရေအတွက်အထိ မတူညီသော စရိုက်လက္ခဏာများစွာကို ပိုင်ဆိုင်သည်။ အရွယ်အစား (Size) သည် လက်နက်၏ ထုထည်၊ အသုံးချမှုနှင့် တည်ငြိမ်မှုတို့ ပေါင်းစပ်ထားခြင်းဖြစ်ပြီး လက်နက်၏ ထိခိုက်မှုကို ဖြစ်ပေါ်စေရန်နှင့် ပိတ်ဆို့ရန် စွမ်းရည်ကို ဆုံးဖြတ်ရန် အသုံးပြုသည်။ အရွယ်အစား အမျိုးအစားများမှာ သေးငယ် (Small)၊ အလယ်အလတ် (Medium)၊ ကြီးမား (Large)၊ ဧရာမ (Huge) နှင့် အလွန်ကြီးမား (Enormous) တို့ဖြစ်သည်။ သတ္တဝါများ၏ လက်နက်မဲ့တိုက်ခိုက်မှုများကို ၎င်းတို့၏ ရုပ်ပိုင်းဆိုင်ရာလက္ခဏာများအရ အလားတူ အမျိုးအစားများ သတ်မှတ်ပေးသည်။

### ထိခိုက်မှုလျှော့ချခြင်း

ကာကွယ်သူသည် ပိတ်ဆို့ရာတွင် အောင်မြင်ပါက ၎င်းတို့သည် တိုက်ခိုက်သူ၏ ထိခိုက်မှုကို (ရှိပါက) အသုံးပြုသော လက်နက်များ၏ နှိုင်းယှဉ်အရွယ်အစားအရ လျှော့ချနိုင်သည်။

- လက်နက် သို့မဟုတ် ဒိုင်းကို တူညီသော သို့မဟုတ် ပိုကြီးသော အရွယ်အစားဖြင့် တိုက်ခိုက်မှုကို ပိတ်ဆို့ခြင်းသည် ထိခိုက်မှုအားလုံးကို ကာကွယ်ပေးသည်။
- အရွယ်အစား တစ်ဆင့်လျော့နည်းသော လက်နက် သို့မဟုတ် ဒိုင်းဖြင့် ပိတ်ဆို့ခြင်းသည် ထိခိုက်မှုတစ်ဝက်ကိုသာ ကာကွယ်ပေးသည်။
- အရွယ်အစား နှစ်ဆင့် သို့မဟုတ် ထို့ထက်ပို၍ နည်းသော လက်နက် သို့မဟုတ် ဒိုင်းဖြင့် ပိတ်ဆို့ခြင်းသည် မည်သည့်ထိခိုက်မှုကိုမျှ ကာကွယ်နိုင်မည်မဟုတ်ပေ။

_ဥပမာအားဖြင့်၊ ကြီးမားသော ပုဆိန် (Huge) ကို ဒိုင်းကြီး (Kite shield) (Huge) ဖြင့် ပိတ်ဆို့ခြင်းသည် ထိခိုက်မှုအားလုံးကို ကာကွယ်နိုင်မည်ဖြစ်သည်။ ၎င်းကို Longsword (Large) ဖြင့် ပိတ်ဆို့ခြင်းသည် ထိခိုက်မှုကို တစ်ဝက်လျှော့ချမည်ဖြစ်ပြီး Shortsword (Medium) ဖြင့် ပိတ်ဆို့ခြင်းသည် မည်သည့်ထိခိုက်မှုကိုမျှ ရပ်တန့်နိုင်မည်မဟုတ်ပေ။_

### သံချပ်ကာ

ကာကွယ်သူ၏ Parry (ပိတ်ဆို့ခြင်း) ကို ထိုးဖောက်ဝင်ရောက်သော မည်သည့်ထိခိုက်မှုမဆို ၎င်းတို့ပိုင်ဆိုင်သော သဘာဝကာကွယ်မှု (ဥပမာ- အကြေးခွံများ သို့မဟုတ် ခရုခွံ) သို့မဟုတ် ဝတ်ဆင်ထားသော သံချပ်ကာ၏ Armor Points (သံချပ်ကာမှတ်) ဖြင့် ထပ်မံလျှော့ချသည်။ ကာကွယ်သူတွင် နှစ်မျိုးလုံးရှိပါက ထိခိုက်မှုကို လျှော့ချရန်အတွက် နှစ်မျိုးလုံး ပေါင်းစည်းသည်။

၎င်း၏ထုထည်နှင့် လှုပ်ရှားမှုကန့်သတ်မှုကြောင့် ဇာတ်ကောင်တစ်ဦးဝတ်ဆင်ထားသော သံချပ်ကာ၏ အမြင့်ဆုံး Armor Point (သံချပ်ကာမှတ်) တန်ဖိုးသည် ၎င်းတို့၏ Initiative roll (ရှေ့ဦးတည်ချက်လှိမ့်ခြင်း) တွင် ဒဏ်ကြေးအဖြစ် လုပ်ဆောင်သည်။

### လွဲချော်သော တိုက်ခိုက်မှုကို ပိတ်ဆို့ခြင်း

တိုက်ခိုက်သူက ၎င်း၏မူလ တိုက်ခိုက်မှုလှိမ့်ခြင်းကို လွဲချော်ပါက ကာကွယ်သူသည် လုပ်ဆောင်ချက်မှတ် (Action Point) တစ်မှတ်ကို Parry (ပိတ်ဆို့ခြင်း) အတွက် သုံးစွဲရန် ရွေးချယ်ခွင့်ရှိသည်။ မည်သို့ပင်ဖြစ်စေ လွဲချော်မည့် တိုက်ခိုက်မှုကို ပိတ်ဆို့ခြင်းသည် မသင့်တော်ဟု ထင်ရနိုင်သော်လည်း ကျွမ်းကျင်သော ကာကွယ်သူတစ်ဦးသည် ၎င်းကို ၎င်း၏အားသာချက်အဖြစ် အသုံးပြု၍ Special Effects (အထူးအကျိုးသက်ရောက်မှုများ) တစ်ခု သို့မဟုတ် ထို့ထက်ပို၍ ရယူနိုင်ပြီး ရန်သူကို အားနည်းစေခြင်း သို့မဟုတ် အချီတစ်ခုကို အခြားလုပ်ဆောင်ချက်တစ်ခုအတွက် သိမ်းဆည်းထားနိုင်သည်။

### ပိတ်ဆို့နိုင်စွမ်းမရှိခြင်း သို့မဟုတ် ပိတ်ဆို့လိုစိတ်မရှိခြင်း

ကာကွယ်သူသည် လုပ်ဆောင်ချက်မှတ်များ (Action Points) မကျန်ရှိတော့သောကြောင့် ပိတ်ဆို့နိုင်စွမ်းမရှိသည့် အခြေအနေတွင် သို့မဟုတ် ထိုးနှက်ချက်ကို ခံနိုင်ရည်ရှိကြောင်း ယုံကြည်၍ ပိတ်ဆို့ရန် ရွေးချယ်ခြင်းမပြုပါက ၎င်းတို့သည် Failure (မအောင်မြင်ခြင်း) အလိုအလျောက် လှိမ့်သည်ဟု မှတ်ယူသည်။ ၎င်းသည် အောင်မြင်သော တိုက်ခိုက်သူအား Special Effects (အထူးအကျိုးသက်ရောက်မှုများ) တစ်ခု သို့မဟုတ် ထို့ထက်ပို၍ ပေးနိုင်သည့် ရလဒ်ရှိသည်။

### မအောင်မြင်သောလှိမ့်ခြင်းများနှင့် ချော်လဲခြင်းများ

တိုက်ခိုက်သူနှစ်ဦးစလုံး ၎င်းတို့၏လှိမ့်ခြင်းများကို မအောင်မြင်ပါက သို့မဟုတ် ကာကွယ်သူသည် လွဲချော်သောတိုက်ခိုက်မှု၏ အကျိုးကျေးဇူးကို မယူရန် ဆုံးဖြတ်ပါက တိုက်ခိုက်မှု-ပိတ်ဆို့မှု အစဉ်လိုက်ပြီးဆုံးပြီး နောက်အမြင့်ဆုံးရှေ့ဦးတည်ချက်ရှိသော ပါဝင်သူထံသို့ တိုက်ခိုက်မှု ဆက်လက်လုပ်ဆောင်သည်။

## အထူးအကျိုးသက်ရောက်မှုများ

တိုက်ခိုက်ခြင်းသည် ပြိုင်ဘက်ကို ဒဏ်ရာရစေခြင်း သို့မဟုတ် သတ်ဖြတ်ခြင်းထက် ပိုမိုသည်။ တိုက်ခိုက်ရေးအနုပညာများသည် ရန်သူကို အနိုင်ယူရန် နည်းလမ်းများစွာကို သင်ကြားပေးသည်၊ ၎င်းတို့ကို ကူကယ်ရာမဲ့ဖြစ်စေခြင်း သို့မဟုတ် ထိခိုက်မှုဖြစ်စေရန် မလိုအပ်ဘဲ အရှုံးပေးရန် တွန်းအားပေးသည့် အခြေအနေများသို့ ရောက်ရှိစေခြင်းတို့ ဖြစ်သည်။ အထူးအကျိုးသက်ရောက်မှုများ (Special Effects) သည် ဤနည်းစနစ်များကို ကိုယ်စားပြုပြီး ၎င်းတို့သည် ဂိမ်းကစားရာတွင် မည်သို့ဖြစ်ပေါ်သည်ကို ထိန်းချုပ်သည်။

ပြိုင်ဘက်များ ၎င်းတို့၏ သက်ဆိုင်ရာ တိုက်ခိုက်ရေးကျွမ်းကျင်မှုများ (fighting skills) ဖြင့် ကွာခြားမှုလှိမ့်ခြင်း (Differential Roll) တွင် ပါဝင်သည့်အခါတိုင်း အောင်မြင်မှုအဆင့်များတွင် ဖြစ်ပေါ်လာသည့် ကွာခြားချက်သည် အထူးအကျိုးသက်ရောက်မှုများ (Special Effects) ဖြစ်ပေါ်လာရန် အခွင့်အလမ်းကို ဖော်ပြသည်။ ၎င်းသည် တိုက်ခိုက်သူတစ်ဦးက ၎င်း၏ပြိုင်ဘက်ကို အားနည်းချက်ရှိသော အခြေအနေတစ်ခုသို့ လှုပ်ရှားစေပြီး လိမ္မာပါးနပ်သော ပရိယာယ် သို့မဟုတ် နည်းဗျူဟာကို အသုံးပြု၍ အမြတ်ထုတ်နိုင်ပုံကို ထင်ဟပ်စေသည်။

ရရှိသော အထူးအကျိုးသက်ရောက်မှုများ (Special Effects) အရေအတွက်သည် Differential Levels of Success (အောင်မြင်မှုအဆင့်ကွာခြားမှု) ဇယားတွင် ဖော်ပြထားသည့်အတိုင်း အောင်မြင်မှုအဆင့်များအကြား ကွာခြားချက်အပေါ် မူတည်သည်။

ဖလှယ်မှုတစ်ခုအတွင်း အထူးအကျိုးသက်ရောက်မှုများ (Special Effects) ကို ရရှိပါက ထိခိုက်မှု (Damage) နှင့် ထိမှန်သည့်နေရာ (Hit Location) ကို လှိမ့်မမီမီ ရွေးချယ်ရမည်။ အကျိုးသက်ရောက်မှု (Effect) ကို အသုံးပြုပြီးနောက် လက်ခံရရှိသူသည် အထူးအကျိုးသက်ရောက်မှုအပြင် ပြင်းထန်စွာ ဒဏ်ရာရရှိသည့် အခြေအနေများတွင် Endurance (ခံနိုင်ရည်) လှိမ့်ခြင်းများကို ဖြေရှင်းသည်။

အထူးအကျိုးသက်ရောက်မှုများ (Special Effects) သည် မတူညီသော အခြေအနေများနှင့် နည်းစနစ်များစွာကို အကျုံးဝင်သည်။ အချို့ကို တိုက်ခိုက်မှုများအတွက် သီးသန့် ဒီဇိုင်းထုတ်ထားပြီး အချို့ကို ကာကွယ်မှုအတွက် ရည်ရွယ်ပြီး အချို့မှာ အခြေအနေနှစ်ခုလုံးအတွက် လိုက်လျောညီထွေဖြစ်အောင် အသုံးပြုနိုင်သည်။ သီးခြားအထူးအကျိုးသက်ရောက်မှုအချို့ကိုလည်း သီးခြားလက်နက်အမျိုးအစားများ သို့မဟုတ် သီးခြားအံစာတုံးလှိမ့်ခြင်းများအတွက် ကန့်သတ်ထားနိုင်သည်၊ ဥပမာအားဖြင့် ၎င်းတို့၏ ကျွမ်းကျင်မှုစစ်ဆေးမှုတွင် Critical (ပြင်းထန်အောင်မြင်ခြင်း) သို့မဟုတ် Fumble (ချော်လဲခြင်း) ရလဒ် လိုအပ်ခြင်းတို့ ဖြစ်သည်။

အထူးအကျိုးသက်ရောက်မှုများ နှစ်ခု သို့မဟုတ် ထို့ထက်ပို၍ ရရှိသည့်အခါ တိုက်ခိုက်သူသည် တစ်ခုစီအတွက် လိုအပ်ချက်များ ပြည့်မီပါက မည်သည့်အရာများကို ရွေးချယ်မည်ကို လွတ်လပ်စွာ ရောနှောရွေးချယ်နိုင်သည်။ အချို့သော အကျိုးသက်ရောက်မှုများကို စုပုံထားနိုင်သည်။ ဥပမာအားဖြင့် Critical Success (ပြင်းထန်အောင်မြင်ခြင်း) ကို လှိမ့်ပြီး Special Effects နှစ်ခုကို ရရှိသော တိုက်ခိုက်သူသည် သီးခြားထိုးစစ်လှုပ်ရှားမှု နှစ်ခုကို ရွေးချယ်မည့်အစား Maximize Damage (ထိခိုက်မှုကို အမြင့်ဆုံးမြှင့်တင်ခြင်း) ကို နှစ်ကြိမ်ရွေးချယ်နိုင်သည်။

### အထူးအကျိုးသက်ရောက်မှုများ ဖော်ပြချက်များ

#### မတော်တဆဒဏ်ရာ

ကာကွယ်သူသည် ပြိုင်ဘက်၏တိုက်ခိုက်မှုကို ၎င်း၏ကိုယ်ပိုင်လက်နက်ဖြင့် ချော်လဲပြီး မိမိကိုယ်ကို ဒဏ်ရာရစေမည့်ပုံစံဖြင့် ကာကွယ်ခြင်း သို့မဟုတ် လှည့်ပစ်ခြင်း။ တိုက်ခိုက်သူသည် ၎င်းတိုက်ခိုက်ရာတွင် အသုံးပြုခဲ့သော လက်နက်ဖြင့် ကျပန်းထိမှန်သည့်နေရာ (random Hit Location) တွင် မိမိကိုယ်ကို ထိခိုက်မှု (damage) လှိမ့်ရမည်။ လက်နက်မဲ့ဖြစ်ပါက တိုက်ခိုက်သူသည် အတွင်းပိုင်းတစ်ခုခုကို ပေါက်ပြဲခြင်း သို့မဟုတ် ကျိုးပဲ့ခြင်းဖြစ်ပြီး ထိခိုက်မှုလှိမ့်ခြင်းသည် မည်သည့်သံချပ်ကာကိုမဆို လျစ်လျူရှုသည်။

#### ထခြင်း

ကာကွယ်သူအား ယာယီပွင့်နေသည့်နေရာကို အသုံးပြု၍ ၎င်း၏ခြေထောက်ပေါ်သို့ ပြန်လည်မတ်တပ်ရပ်နိုင်စေသည်။

#### ရိုက်ပုတ်ခြင်း

တိုက်ခိုက်သူသည် ပြိုင်ဘက်ကို တမင်တကာ ချိန်ခွင်လျှာပျက်အောင် ရိုက်ပုတ်သည်။ ကာကွယ်သူ မည်မျှ နောက်သို့ သို့မဟုတ် ဘေးသို့ ယိုင်လဲသွားသည်မှာ အသုံးပြုသော လက်နက်အပေါ် မူတည်သည်။ ဒိုင်းများသည် ထိခိုက်မှုရမှတ် ၂ မှတ် (သံချပ်ကာ၊ ပိတ်ဆို့မှု စသည်တို့ကို မနုတ်မီ) တိုင်းအတွက် ပြိုင်ဘက်ကို တစ်မီတာ နောက်သို့ ရိုက်ပုတ်ပြီး ရိုက်နှက်လက်နက်များသည် ၃ မှတ်တိုင်းအတွက် တစ်မီတာ ရိုက်ပုတ်သည်။ ရိုက်ပုတ်ခြင်းသည် တိုက်ခိုက်သူ၏ SIZ နှစ်ဆအထိရှိသော သတ္တဝါများတွင်သာ အကျိုးသက်ရောက်သည်။ အကယ်၍ ထိခိုက်ခံရသူသည် အတားအဆီးတစ်ခုဆီသို့ နောက်သို့ တွန်းပို့ခံရပါက ၎င်းတို့သည် လဲကျခြင်း သို့မဟုတ် ချော်လဲခြင်းမှ ရှောင်ရှားရန် Acrobatics skill roll - သို့မဟုတ် Hard difficulty ဖြင့် Athletics - ကို ပြုလုပ်ရမည်။ သေနတ်များအတွက်၊ ရိုက်ပုတ်ခြင်းသည် ပြိုင်ဘက်ကို ရိုးရိုးအားဖြင့် နောက်သို့ ပစ်ခြင်းထက် ချိန်ခွင်လျှာပျက်အောင် ဖမ်းဆုပ်ခြင်းအပေါ် ပိုမိုမှီခိုသောကြောင့် ကျည်ဆန်များသည် အစွန်းပိုင်းကို ထိမှန်ပါက ပစ်မှတ်ကို ယိုင်ထိုးစေခြင်းမှာ လုံးဝသင့်လျော်သည်။ စွမ်းအင်လက်နက်များသည် Beam ၏ ရူပဗေဒကြောင့် မျက်နှာပြင်ပေါက်ကွဲမှုတစ်မျိုးမျိုး ဖြစ်ပေါ်ခြင်းမရှိပါက ထိုသို့သော အကျိုးသက်ရောက်မှုကို ဖြစ်ပေါ်စေနိုင်ခြေနည်းပါသည်။

#### သွေးထွက်ခြင်း

တိုက်ခိုက်မှုကြောင့် အဓိကသွေးကြောတစ်ခု ပွင့်သွားသည်။ ထိုးနှက်ချက်က သံချပ်ကာမှတ် (Armor Points) ကို ကျော်လွှားပြီး ပစ်မှတ်ကို ဒဏ်ရာရစေပါက ကာကွယ်သူသည် Endurance ၏ Opposed Roll (ပြိုင်ဆိုင်လှိမ့်ခြင်း) ကို မူလတိုက်ခိုက်မှုလှိမ့်ခြင်း (original attack roll) နှင့် ဆန့်ကျင်၍ ပြုလုပ်ရမည်။ ကာကွယ်သူ Fails (မအောင်မြင်) ပါက ၎င်းတို့သည် သွေးလွန်စွာ စတင်ထွက်သည်။ တိုက်ခိုက်ရေးအချီ (Combat Round) တစ်ခုစီ၏အစတွင် လက်ခံရရှိသူသည် ပင်ပန်းနွမ်းနယ်မှု (Fatigue) အဆင့်တစ်ဆင့် ဆုံးရှုံးပြီး ပြိုလဲကာ သေဆုံးနိုင်သည်။ သွေးထွက်ဒဏ်ရာများကို First Aid skill roll (ရှေးဦးသူနာပြုကျွမ်းကျင်မှုလှိမ့်ခြင်း) ကို အောင်မြင်စွာ ပြုလုပ်ခြင်းဖြင့် ရပ်တန့်နိုင်သော်လည်း လက်ခံရရှိသူသည် ဒဏ်ရာကို ပြန်မပွင့်စေဘဲ မည်သည့်ခွန်အားကြီးသော သို့မဟုတ် အကြမ်းဖက်သော လုပ်ဆောင်ချက်ကိုမျှ မလုပ်ဆောင်နိုင်တော့ပါ။ ဂိမ်းစနစ် (Game System) အပိုင်းရှိ Blood Loss (သွေးဆုံးရှုံးမှု) ကို ကြည့်ပါ။ သေနတ်များအတွက်၊ သေနတ်အများစုသည် အလွန်ကျဉ်းမြောင်းသော ဒဏ်ရာလမ်းကြောင်းများကို ဖန်တီးသောကြောင့် ဓားဖြင့်လှီးဖြတ်ခြင်း သို့မဟုတ် ထိုးနှက်ခြင်းလက်နက်များကဲ့သို့ အဓိကသွေးကြောကို ဖြတ်ခြင်း သို့မဟုတ် ဖြတ်တောက်ခြင်းသည် ပိုမိုခက်ခဲသည်။ ထို့ကြောင့် Bleed သည် Critical Success (ပြင်းထန်အောင်မြင်ခြင်း) အတွက်သာ ဖြစ်သင့်သည် - သေနတ်သည် shuriken၊ razor fléchettes၊ ပြန့်ကားသော စွမ်းအားနယ်ပယ်များ သို့မဟုတ် ထိုသို့သော ထူးခြားသော ခဲယမ်းများကို ပစ်ခတ်သည့် ထူးဆန်းသော လက်နက်တစ်ခုမဟုတ်ပါက။

#### ပြိုင်ဘက်ကို မျက်စိကွယ်စေခြင်း

Critical တွင် ကာကွယ်သူသည် သဲပစ်ခြင်း၊ ဒိုင်းမှနေ၍ နေရောင်ခြည်ကို ရောင်ပြန်ဟပ်ခြင်း သို့မဟုတ် တိုက်ခိုက်သူ၏ အမြင်အာရုံကို ခဏတာ အနှောင့်အယှက်ဖြစ်စေသော အခြားနည်းဗျူဟာတစ်ခုခုဖြင့် ၎င်း၏ပြိုင်ဘက်ကို ခဏတာ မျက်စိကွယ်စေသည်။ တိုက်ခိုက်သူသည် ၎င်း၏ Evade skill (ရှောင်တိမ်းကျွမ်းကျင်မှု) (သို့မဟုတ် ဒိုင်းကိုအသုံးပြုပါက Weapon skill) ၏ Opposed Roll (ပြိုင်ဆိုင်လှိမ့်ခြင်း) ကို ကာကွယ်သူ၏ မူလ Parry roll (ပိတ်ဆို့လှိမ့်ခြင်း) နှင့် ဆန့်ကျင်၍ ပြုလုပ်ရမည်။ တိုက်ခိုက်သူ Fails (မအောင်မြင်) ပါက မျက်စိကွယ်စေသော နည်းလမ်းပေါ်မူတည်၍ နောက် ၁d၃ အလှည့်အတွက် Hard သို့မဟုတ် Formidable (ကြောက်မက်ဖွယ်) ၏ အခြေအနေဆိုင်ရာမွမ်းမံမှု (situational modifier) ကို ခံစားရသည်။

#### သံချပ်ကာကို ကျော်ဖြတ်ခြင်း

Critical တွင် တိုက်ခိုက်သူသည် ကာကွယ်သူ၏ သဘာဝ သို့မဟုတ် ဝတ်ဆင်ထားသော သံချပ်ကာတွင် ကွက်လပ်တစ်ခုကို ရှာတွေ့သည်။ ကာကွယ်သူသည် သဘာဝကာကွယ်မှုအပြင် သံချပ်ကာဝတ်ဆင်ထားပါက တိုက်ခိုက်သူသည် နှစ်ခုထဲမှ မည်သည့်အရာကို ကျော်ဖြတ်မည်ကို ဆုံးဖြတ်ရမည်။ ဤအကျိုးသက်ရောက်မှုကို နှစ်ခုလုံးကို ကျော်ဖြတ်ရန်အတွက် စုပုံထားနိုင်သည်။ ဤအကျိုးသက်ရောက်မှု၏ ရည်ရွယ်ချက်အတွက် မှော်ပညာမှ ရရှိသော ရုပ်ပိုင်းဆိုင်ရာ ကာကွယ်မှုကို ဝတ်ဆင်ထားသော သံချပ်ကာအဖြစ် မှတ်ယူသည်။ သံချပ်ကာကို ကျော်ဖြတ်ခြင်း (Bypass Armor) ကို မည်သည့်တိုက်ခိုက်မှုပုံစံနှင့်မဆို အသုံးပြုနိုင်သော်လည်း အချို့သော ခဲယမ်းအမျိုးအစားများကို ဤအကျိုးသက်ရောက်မှုကို ရယူခွင့်မပြုနိုင်ပါ (ပြန့်ကျဲသော ကျည်ဆန်များကြောင့်) သို့မဟုတ် ၎င်းတို့၏ အခြေခံစွမ်းဆောင်ရည်တွင် ပါဝင်ပြီး အနိုင်ရရှိသည့် အခြားအထူးအကျိုးသက်ရောက်မှုများအပြင် အပိုဆုကြေးအဖြစ် ပေးနိုင်သည် (Teleportation rounds)။ ဂိမ်းမာစတာက လက်နက်ကို မည်သို့ပုံစံထုတ်ချင်သည်အပေါ် များစွာမူတည်သည်။

#### နေရာရွေးချယ်ခြင်း

လက်ချင်းထိတိုက်ခိုက်လက်နက်များကို အသုံးပြုသည့်အခါ တိုက်ခိုက်သူသည် ထိုးနှက်ချက်ကျရောက်သည့် နေရာကို လွတ်လပ်စွာ ရွေးချယ်နိုင်သည်၊ ထိုနေရာသည် ပုံမှန်အားဖြင့် လက်လှမ်းမီသမျှ ဖြစ်ရမည်။ သေနတ်များအပါအဝင် အဝေးပစ်လက်နက်များကို အသုံးပြုသည့်အခါ နေရာရွေးချယ်ခြင်း (Choose Location) သည် Critical Success (ပြင်းထန်အောင်မြင်ခြင်း) အတွက်သာ ဖြစ်သည်၊ ပစ်မှတ်သည် Close Range (အနီးကပ်အကွာအဝေး) အတွင်းရှိပြီး ငြိမ်သက်နေခြင်း သို့မဟုတ် တိုက်ခိုက်သူကို သတိမထားမိခြင်းမှလွဲ၍ ဖြစ်သည်။ သေနတ်များအတွက် တယ်လီစကုပ်ပစ်မှတ်ချိန်ကိရိယာများက ဤအကွာအဝေးကို တိုးချဲ့နိုင်သည်။

#### ကာဗာကိုပတ်သွားခြင်း

ထိုးစစ်၊ နည်းပညာမြင့်သေနတ်များသာ။

#### ပိတ်ဆို့မှုကို ပတ်သွားခြင်း

Critical တွင် တိုက်ခိုက်သူသည် မည်သို့ပင်ဖြစ်စေ အောင်မြင်သော Parry (ပိတ်ဆို့ခြင်း) ကို လုံးဝကျော်ဖြတ်နိုင်သည်။

#### လက်နက်ကို ပျက်စီးစေခြင်း

တိုက်ခိုက်မှု သို့မဟုတ် ပိတ်ဆို့မှု၏ အစိတ်အပိုင်းအနေဖြင့် ၎င်း၏ပြိုင်ဘက်၏လက်နက်ကို ပျက်စီးစေရန် ဇာတ်ကောင်ကို ခွင့်ပြုသည်။ တိုက်ခိုက်ပါက ဇာတ်ကောင်သည် ကာကွယ်သူ၏ ပိတ်ဆို့လက်နက်ကို သီးသန့်ပစ်မှတ်ထားပြီး ၎င်းကိုင်ဆောင်သူကို တိုက်ခိုက်ခြင်းထက် ၎င်း၏ထိခိုက်မှု (damage) လှိမ့်ခြင်းကို လက်နက်ပေါ်တွင် သက်ရောက်သည်။ ပစ်မှတ်ထားသော လက်နက်သည် ထိခိုက်မှုကို ခုခံရန်အတွက် ၎င်း၏ကိုယ်ပိုင် Armor Points (သံချပ်ကာမှတ်) ကို အသုံးပြုသည်။ Hit Points (ထိမှန်မှတ်များ) သုညအထိ လျှော့ချခံရပါက လက်နက်ကျိုးပဲ့သည်။ သေနတ်များအတွက်၊ အကွာအဝေးမှ လက်ကိုင်လက်နက်တစ်ခုကို ထိမှန်ခြင်းသည် သီးခြားကိုယ်ခန္ဓာအစိတ်အပိုင်းတစ်ခုကို ပစ်မှတ်ထားခြင်းထက် ပိုမိုခက်ခဲသောကြောင့် ၎င်းကို ပုံမှန်အားဖြင့် Critical Success (ပြင်းထန်အောင်မြင်ခြင်း) များအတွက်သာ ထားရှိသင့်သည်။ သို့သော် ခြွင်းချက်များစွာရှိသည်၊ ပထမအချက်မှာ ဒိုင်းတစ်ခုသည် ၎င်း၏သဘာဝအရ ပိုမိုလွယ်ကူစွာ ထိမှန်နိုင်သင့်သည်။ ဒုတိယအချက်မှာ ၎င်း၏သက်ရောက်မှုဧရိယာအတွင်းရှိ လက်ကိုင်အရာဝတ္ထုများကို ပစ်မှတ်ထားနိုင်သော ကန်ပုံသဏ္ဍာန် beam ကို ပစ်ခတ်သည့် ထူးခြားသောလက်နက်များနှင့် ပတ်သက်သည်။ ဥပမာအားဖြင့် သတ္တုအရာဝတ္ထုများကို အပူပေးသည့် induction gun (လျှပ်စစ်သံလိုက်သေနတ်)။

#### ပြိုင်ဘက်လက်နက်ချခြင်း

ဇာတ်ကောင်သည် ပြိုင်ဘက်၏လက်နက်ကို ၎င်း၏လက်မှ ခေါက်ချခြင်း၊ ဆွဲထုတ်ခြင်း သို့မဟုတ် လိမ်ဖယ်ခြင်း။ ပြိုင်ဘက်သည် ၎င်း၏ Combat Style (တိုက်ခိုက်ရေးပုံစံ) ၏ Opposed Roll (ပြိုင်ဆိုင်လှိမ့်ခြင်း) ကို ဇာတ်ကောင်၏ မူလလှိမ့်ခြင်း (original roll) နှင့် ဆန့်ကျင်၍ ပြုလုပ်ရမည်။ လက်နက်ချခံရသူ ရှုံးပါက ၎င်း၏လက်နက်သည် လက်နက်ချသူ၏ Damage Modifier (ထိခိုက်မှုမွမ်းမံမှု) အရ မီတာများဖြင့် ဝေးကွာသော အကွာအဝေးသို့ လွင့်စင်သွားသည်။ Damage Modifier မရှိပါက လက်နက်သည် လက်နက်ချခံရသူ၏ ခြေရင်းတွင် ကျသည်။ လက်နက်များ၏ နှိုင်းယှဉ်အရွယ်အစားသည် လှိမ့်ခြင်းကို သက်ရောက်သည်။ လက်နက်ချသူ၏ လက်နက်သည် ပိုကြီးသော အဆင့်တစ်ခုစီတိုင်းသည် ပြိုင်ဘက်၏လှိမ့်ခြင်း အခက်အခဲကို တစ်ဆင့်တိုးမြှင့်သည်။ ဆန့်ကျင်ဘက်အားဖြင့် လက်နက်ချသူ၏ လက်နက်သည် ပိုသေးငယ်သော အဆင့်တစ်ခုစီတိုင်းသည် အခက်အခဲကို တစ်ဆင့်ပိုမိုလွယ်ကူစေသည်။ လက်နက်ချခြင်းသည် တိုက်ခိုက်သူ၏ STR နှစ်ဆအထိရှိသော သတ္တဝါများတွင်သာ အကျိုးသက်ရောက်သည်။

#### ရန်သူကို လဲကျစေခြင်း

ထိုးစစ်၊ သေနတ်များသာ။ ပစ်မှတ်သည် ထိုးနှက်ချက်မှ အနည်းဆုံး ဒဏ်ရာအသေးစားတစ်ခု ခံစားရသည်ဟု ယူဆပါက ၎င်းတို့သည် ၎င်းတို့၏ Endurance (ခံနိုင်ရည်) ၏ Opposed Test (ပြိုင်ဆိုင်စမ်းသပ်မှု) ကို တိုက်ခိုက်သူ၏ ထိမှန်မှုလှိမ့်ခြင်း (hit roll) နှင့် ဆန့်ကျင်၍ ပြုလုပ်ရမည်။ ရှုံးနိမ့်ခြင်းသည် ပစ်မှတ်သည် ထိတ်လန့်ခြင်းနှင့် နာကျင်ခြင်းတို့ကြောင့် အားအင်ကုန်ခမ်းကာ ဆက်လက်တိုက်ခိုက်နိုင်ခြင်းမရှိတော့ကြောင်း ပြသသည်။ အားအင်ကုန်ခမ်းခြင်းမှ ပြန်လည်သက်သာခြင်းကို အောင်မြင်သော First Aid check (ရှေးဦးသူနာပြုစစ်ဆေးမှု) သို့မဟုတ် ကမ်ပိန်းတွင် တည်ရှိပါက နည်းပညာ သို့မဟုတ် မူးယစ်ဆေးဝါးသုံး အားမြှင့်ဆေးတစ်မျိုးမျိုးဖြင့် လုပ်ဆောင်နိုင်သည်။ သို့မဟုတ်ပါက ယာယီအားအင်ကုန်ခမ်းခြင်းသည် ပစ်မှတ်၏ Healing Rate (ကျန်းမာရေးပြန်လည်ကောင်းမွန်နှုန်း) ဖြင့် တစ်နာရီကို ပိုင်းခြားထားသည့် ကာလတစ်ခုကြာရှည်မည်ဖြစ်သည်။

#### နောက်ပြန်ငုံ့ခြင်း

ထိုးစစ်၊ သေနတ်များသာ။ ဤအထူးအကျိုးသက်ရောက်မှုသည် သေနတ်ပစ်သူအား ၎င်းတို့၏ နောက်အလှည့်တွင် Take Cover (ခိုလှုံရာယူခြင်း) လုပ်ဆောင်ချက်ကို အသုံးပြုရန် မစောင့်ဆိုင်းဘဲ ခိုလှုံရာထဲသို့ ချက်ချင်းငုံ့ပြန်ဝင်နိုင်စေသည်။ ဇာတ်ကောင်သည် Duck Back (နောက်ပြန်ငုံ့ခြင်း) ကို အသုံးပြုရန်အတွက် ခိုလှုံရာတစ်ခု၏ဘေးတွင် မတ်တပ်ရပ်လျက် သို့မဟုတ် ဒူးထောက်လျက် ရှိနေရမည်။

#### ပိတ်ဆို့မှုကို မြှင့်တင်ခြင်း

Critical တွင် ကာကွယ်သူသည် ၎င်း၏လက်နက်အရွယ်အစား မည်သို့ပင်ဖြစ်စေ တိုက်ခိုက်မှုတစ်ခု၏ စွမ်းအားတစ်ခုလုံးကို လွှဲပြောင်းပစ်နိုင်ခြင်း။

#### ရှုပ်ထွေးစေခြင်း

ကြိမ်လုံး သို့မဟုတ် ကွန်ရက်ကဲ့သို့သော ရှုပ်ထွေးစေသည့်လက်နက် (entangling weapon) ကို ကိုင်ဆောင်ထားသော ဇာတ်ကောင်အား ထိမှန်သောနေရာကို မလှုပ်မယှက်ဖြစ်စေရန် ခွင့်ပြုသည်။ ရှုပ်ထွေးနေသော လက်မောင်းသည် ၎င်းကိုင်ဆောင်ထားသည့်အရာကို အသုံးမပြုနိုင်၊ ကြိုးနွယ်နေသော ခြေထောက်သည် ပစ်မှတ်ကို ရွေ့လျားခြင်းမှ ကာကွယ်ပေးသည်၊ ဦးခေါင်း၊ ရင်ဘတ် သို့မဟုတ် ဝမ်းဗိုက်တို့ ရှုပ်ထွေးနေပါက ကျွမ်းကျင်မှုလှိမ့်ခြင်း (skill rolls) အားလုံးကို တစ်ဆင့်ပိုမိုခက်ခဲစေသည်။ ၎င်း၏နောက်အလှည့်တွင် ကိုင်ဆောင်သူသည် အလိုအလျောက် Trip Opponent (ပြိုင်ဘက်ကို ချော်လဲစေခြင်း) ကြိုးပမ်းမှုကို ပြုလုပ်ရန် လုပ်ဆောင်ချက်မှတ် (Action Point) တစ်မှတ်ကို သုံးစွဲနိုင်သည်။ ရှုပ်ထွေးနေသော သားကောင်သည် ၎င်း၏အလှည့်တွင် Brawn (ကြွက်သားအား) ကို အသုံးပြု၍ ရုန်းထွက်ရန် ပြိုင်ဆိုင်လှိမ့်ခြင်း (Opposed Roll) ဖြင့် သို့မဟုတ် အထူးအကျိုးသက်ရောက်မှု (Special Effect) တစ်ခုကို ရရှိပြီး Damage Weapon (လက်နက်ပျက်စီးစေခြင်း)၊ Disarm Opponent (ပြိုင်ဘက်လက်နက်ချခြင်း) သို့မဟုတ် Slip Free (လွတ်မြောက်ခြင်း) ကို ရွေးချယ်ခြင်းဖြင့် မိမိကိုယ်ကို လွတ်မြောက်အောင် ကြိုးပမ်းနိုင်သည်။

#### မအောင်မြင်စေရန် တွန်းအားပေးခြင်း

ပြိုင်ဘက် Fumbles (ချော်လဲခြင်း) ဖြစ်သောအခါ အသုံးပြုသည်။ ဇာတ်ကောင်သည် Force Failure (မအောင်မြင်စေရန် တွန်းအားပေးခြင်း) ကို Opposed Roll (ပြိုင်ဆိုင်လှိမ့်ခြင်း) လိုအပ်သော အခြားအထူးအကျိုးသက်ရောက်မှု (Special Effect) တစ်ခုခုနှင့် ပေါင်းစပ်အသုံးပြုနိုင်သည်။ Force Failure သည် ပြိုင်ဘက်အား ၎င်း၏ခုခံမှုလှိမ့်ခြင်းကို ပုံသေအားဖြင့် မအောင်မြင်စေသည် - ထို့ကြောင့် အလိုအလျောက် လက်နက်ချခံရခြင်း၊ ချော်လဲခြင်း စသည်တို့ ဖြစ်ပေါ်စေသည်။

#### ဆုပ်ကိုင်ခြင်း

ပြိုင်ဘက်သည် တိုက်ခိုက်သူ၏ လက်နက်မဲ့တိုက်ခိုက်မှုအကွာအဝေးအတွင်း ရှိနေပါက ၎င်းတို့သည် လက်ဗလာ (သို့မဟုတ် ဆုပ်ကိုင်နိုင်သည့် အလားတူအင်္ဂါ) ကို အသုံးပြု၍ ပြိုင်ဘက်ကို ဆုပ်ကိုင်ထားနိုင်ပြီး တိုက်ခိုက်ခြင်းမှ ထွက်ခွာခြင်းကို တားဆီးနိုင်သည်။ ပြိုင်ဘက်သည် ၎င်း၏အလှည့်တွင် လွတ်မြောက်ရန် ကြိုးပမ်းနိုင်ပြီး Brawn (ကြွက်သားအား) သို့မဟုတ် Unarmed (လက်နက်မဲ့) ၏ Opposed Roll (ပြိုင်ဆိုင်လှိမ့်ခြင်း) ကို ဆုပ်ကိုင်သူက ပိုမိုနှစ်သက်ရာ ကျွမ်းကျင်မှုနှင့် ဆန့်ကျင်၍ ပြုလုပ်ရမည်။

#### ထိုးဖောက်ခြင်း

လက်နက်ထိခိုက်မှု (weapon damage) ကို နှစ်ကြိမ်လှိမ့်ပြီး တိုက်ခိုက်သူက တိုက်ခိုက်မှုအတွက် ရလဒ်နှစ်ခုထဲမှ တစ်ခုကို ရွေးချယ်သည်။ လက်နက်သည် လက်ချင်းထိတိုက်ခိုက်လက်နက်ဖြစ်သည်ဟု ယူဆပါက သံချပ်ကာကို ထိုးဖောက်ပြီး ဒဏ်ရာရစေပါက တိုက်ခိုက်သူသည် လက်နက်ကို ဒဏ်ရာထဲတွင် ထားခဲ့ရန် သို့မဟုတ် ၎င်း၏နောက်အလှည့်တွင် ဆွဲထုတ်ရန် ရွေးချယ်ခွင့်ရှိသည်။ လက်နက်ကို ဒဏ်ရာထဲတွင် ထားခဲ့ခြင်းသည် သားကောင်၏ နောင်ကျွမ်းကျင်မှုကြိုးပမ်းမှုများတွင် အခက်အခဲအဆင့် (Difficulty Grade) ကို ဖြစ်ပေါ်စေသည်။ ဒဏ်ကြေး၏ ပြင်းထန်မှုသည် Impale Effects Table (ထိုးဖောက်မှုအကျိုးသက်ရောက်မှုဇယား) တွင် ဖော်ပြထားသည့်အတိုင်း သတ္တဝါနှင့် ၎င်းကိုထိုးဖောက်ထားသော လက်နက်နှစ်ခုစလုံး၏ အရွယ်အစားပေါ်မူတည်သည်။ ရိုးရှင်းစေရန်အတွက် တူညီသောအရွယ်အစားရှိ လက်နက်ဖြင့် နောက်ထပ်ထိုးဖောက်မှုများသည် နောက်ထပ်ဒဏ်ကြေးများ မဖြစ်ပေါ်စေပါ။ လက်ချင်းထိတိုက်ခိုက်မှုအတွင်း ထိုးဖောက်ထားသော လက်နက်ကို ဆွဲထုတ်ရန် Ready Weapon Combat Action (လက်နက်ပြင်ဆင်ခြင်းတိုက်ခိုက်ရေးလုပ်ဆောင်ချက်) ကို အသုံးပြုရန် လိုအပ်သည်။ ကိုင်ဆောင်သူသည် Brawn roll (ကြွက်သားအားလှိမ့်ခြင်း) ကို မပြိုင်ဆိုင်ဘဲ အောင်မြင်ရမည် (သို့မဟုတ် ပြိုင်ဘက်က ခုခံပါက Opposed Brawn roll (ပြိုင်ဆိုင်ကြွက်သားအားလှိမ့်ခြင်း) ကို အနိုင်ရရမည်)။ အောင်မြင်ပါက လက်နက်ကို ဆွဲထုတ်နိုင်ပြီး ထိုနေရာတွင် ပုံမှန်ထိခိုက်မှုလှိမ့်ခြင်း၏ တစ်ဝက်နှင့် ညီမျှသော နောက်ထပ်ဒဏ်ရာ (Damage Modifier မပါဘဲ) ဖြစ်စေသည်။ မအောင်မြင်ပါက လက်နက်သည် ဒဏ်ရာထဲတွင် ဆက်လက်စိုက်နေပြီး အခြားအကျိုးသက်ရောက်မှုမရှိ၊ သို့သော် ကိုင်ဆောင်သူသည် ၎င်း၏နောက်အလှည့်တွင် ထပ်မံကြိုးစားနိုင်သည်။ အထူးသဖြင့် ဆူးပါသော လက်နက်များ (ဥပမာ- လှံကြီးများ) သည် ပုံမှန်ထိခိုက်မှု (damage) ကို ဖြစ်ပေါ်စေသည်။ သံချပ်ကာသည် ဆွဲထုတ်ဒဏ်ရာကို မလျှော့ချပါ။ ၎င်းသည် ထိုးဖောက်လျက်ရှိနေစဉ် တိုက်ခိုက်သူသည် ၎င်း၏ထိုးဖောက်လက်နက်ကို ပိတ်ဆို့ရန် (Parrying) အသုံးမပြုနိုင်။

#### ပစ်ခတ်ကျွမ်းကျင်သူ

သေနတ်ပစ်သူအား ၎င်း၏ပစ်ခတ်မှုကြောင့် ထိမှန်သည့်နေရာ (Hit Location) ကို တစ်ဆင့်ရွှေ့၍ ချက်ချင်းကပ်လျက် ကိုယ်ခန္ဓာအစိတ်အပိုင်းတစ်ခုသို့ ရွှေ့ရန် ခွင့်ပြုသည်။ ဇီဝကမ္မဗေဒသည် ပြန်လည်ပစ်မှတ်ထားနိုင်သည့်အရာများအပေါ် သက်ရောက်မှုရှိပြီး သာမန်အသိဉာဏ်ကို အသုံးပြုသင့်သည်။ ထို့ကြောင့် လူသားတစ်ဦးအပေါ် ဤအထူးအကျိုးသက်ရောက်မှုကို အသုံးပြုခြင်းသည် ခြေထောက်ကိုထိမှန်သော တိုက်ခိုက်သူအား ၎င်းကို ဝမ်းဗိုက်သို့ ရွှေ့ရန် ခွင့်ပြုနိုင်သည်။ ဆန့်ကျင်ဘက်အားဖြင့် ကျီးကန်းကို ရင်ဘတ်ကို ပစ်ခတ်ခြင်းသည် ခြေထောက်၊ တောင်ပံ သို့မဟုတ် ဦးခေါင်းကို ရွေးချယ်ခွင့်ပြုနိုင်သည်။

#### ထိခိုက်မှုကို အမြင့်ဆုံးမြှင့်တင်ခြင်း

Critical တွင် ဇာတ်ကောင်သည် ၎င်း၏လက်နက်၏ ထိခိုက်မှုအံစာတုံးများထဲမှ တစ်ခုကို ၎င်း၏အပြည့်အဝတန်ဖိုးဖြင့် အစားထိုးနိုင်သည်။ ဥပမာအားဖြင့်၊ ပုံမှန်အားဖြင့် 1d6 ထိခိုက်မှုပေးသော Hand Axe (လက်ပုဆိန်) ကို ၆ အဖြစ် မှတ်ယူမည်ဖြစ်ပြီး 2d6 ထိခိုက်မှုပေးသော great club ကို 1d6+6 ထိခိုက်မှုဖြစ်စေမည်။ ဤအထူးအကျိုးသက်ရောက်မှုကို စုပုံထားနိုင်သည်။ ၎င်းကို သဘာဝလက်နက်များအတွက်လည်း အသုံးပြုနိုင်သော်လည်း Maximize Damage (ထိခိုက်မှုကို အမြင့်ဆုံးမြှင့်တင်ခြင်း) သည် တိုက်ခိုက်သူ၏ Damage Modifier (ထိခိုက်မှုမွမ်းမံမှု) ကို မသက်ရောက်ဘဲ ပုံမှန်အတိုင်း လှိမ့်ရမည်။

#### ကျော်လွန်ထိုးဖောက်ခြင်း

Critical၊ သေနတ်များသာ။ တန်းစီနေသော ပြိုင်ဘက်များကို သို့မဟုတ် လူအုပ်ကြီးထဲသို့ ပစ်ခတ်ပါက ဤအထူးအကျိုးသက်ရောက်မှုသည် ပထမသားကောင်၏ ကိုယ်ကာကွယ်ဝတ်စုံကို ကျော်လွှားပါက ပထမသားကောင်ကို လုံးဝဖြတ်သန်းကာ ၎င်းတို့နောက်မှ ဒုတိယသားကောင်ကို ထိမှန်စေသည်။ သို့သော် ဒုတိယသားကောင်သည် ထိုးနှက်ချက်၏ အားလျော့ခြင်း သို့မဟုတ် နှေးကွေးခြင်းကြောင့် ထိခိုက်မှုတစ်ဝက်ကိုသာ ခံစားရသည်။ ကျော်လွန်ထိုးဖောက်ခြင်းသည် ပျက်စီးမှုများစွာ ဖြစ်ပေါ်စေသော သို့မဟုတ် သံချပ်ကာဖောက်နိုင်သည့် စွမ်းရည်အချို့ရှိသော စွမ်းအားမြင့်လက်နက်များတွင် ပိုမိုအသုံးဝင်သည်။ ပထမပစ်မှတ်အပေါ် ဖြစ်ပေါ်သော မည်သည့်အထူးအကျိုးသက်ရောက်မှုများကိုမျှ ဒုတိယပစ်မှတ်အပေါ် အသုံးမပြုပါ။

#### ပိတ်ဆို့ဖိထားခြင်း

သေနတ်များသာ။ Press Advantage (အသာစီးရယူရန်ဖိအားပေးခြင်း) နှင့် ဆင်တူစွာ၊ ဤအထူးအကျိုးသက်ရောက်မှုသည် ပစ်မှတ်အား ၎င်းတို့၏ Willpower (စိတ်စွမ်းအား) ၏ Opposed Test (ပြိုင်ဆိုင်စမ်းသပ်မှု) ကို တိုက်ခိုက်သူ၏ hit roll (ထိမှန်မှုလှိမ့်ခြင်း) နှင့် ဆန့်ကျင်၍ ပြုလုပ်စေသည်။ မအောင်မြင်ခြင်းသည် ပစ်မှတ်သည် ရရှိနိုင်သော မည်သည့်ခိုလှုံရာနောက်တွင်မဆို ပုန်းအောင်းနေပြီး ၎င်းတို့၏ နောက်အလှည့်တွင် ပြန်မပစ်နိုင်တော့ကြောင်း ဆိုလိုသည်။ Pin Down (ပိတ်ဆို့ဖိထားခြင်း) သည် ပစ်မှတ်အပေါ် တကယ်တမ်း ထိခိုက်မှုမရှိပါကပင် (အောင်မြင်စွာ ရှောင်တိမ်းနိုင်ခြင်း သို့မဟုတ် ၎င်းတို့၏ ခိုလှုံရာကို ထိမှန်ခြင်းကြောင့်) အလုပ်လုပ်နိုင်သည်၊ အဘယ်ကြောင့်ဆိုသော် ၎င်းသည် အနီးကပ်ဖြတ်သွားသော သေနတ်ကျည်ဆန်များ၏ ခြိမ်းခြောက်မှုအကျိုးသက်ရောက်မှုအပေါ် မှီခိုနေသောကြောင့် ဖြစ်သည်။

ပိတ်ဆို့ခံထားရသော သားကောင်သည် လိုအပ်သောအချိန်အတွက် ပြန်မပစ်နိုင်သော်လည်း ၎င်းတို့သည် မိမိကိုယ်ကို ပစ်ခတ်ခံရခြင်းမှ မထိတွေ့စေဘဲ အခြားလုပ်ဆောင်ချက်များကို လုပ်ဆောင်နိုင်သည်၊ ဥပမာ- ခိုလှုံရာအသစ်သို့ တွားသွားခြင်း၊ အခြားသူများနှင့် ဆက်သွယ်ခြင်း၊ လက်နက်ပြန်လည်တင်ဆောင်ခြင်း စသည်တို့။

#### တန်ပြန်ရန် ပြင်ဆင်ခြင်း

ကာကွယ်သူသည် ၎င်း၏ရန်သူ၏ပုံစံများကို ဖတ်ရှုပြီး သီးခြားအထူးအကျိုးသက်ရောက်မှု (Special Effect) ကို တန်ပြန်ရန် ပြင်ဆင်သည် (၎င်းကို လျှို့ဝှက်မှတ်သားထားသင့်သည်)။ ၎င်း၏ပြိုင်ဘက်က တိုက်ပွဲအတွင်း ရွေးချယ်ထားသော အထူးအကျိုးသက်ရောက်မှုကို ၎င်းအပေါ် သက်ရောက်စေရန် ကြိုးပမ်းပါက ကာကွယ်သူသည် တိုက်ခိုက်သူ၏ အကျိုးသက်ရောက်မှုကို ၎င်း၏ကိုယ်ပိုင်အကျိုးသက်ရောက်မှုတစ်ခုဖြင့် ချက်ချင်းအစားထိုးပြီး ၎င်းသည် အလိုအလျောက် အောင်မြင်သည်။

#### လျင်မြန်စွာ ပြန်လည်တင်ဆောင်ခြင်း

အဝေးပစ်လက်နက် (Ranged Weapon) ကို အသုံးပြုသည့်အခါ တိုက်ခိုက်သူသည် နောက်ထပ်ပစ်ခတ်မှုအတွက် ပြန်လည်တင်ဆောင်ချိန်ကို တစ်ဆင့်လျှော့ချပေးသည်။ ဤအကျိုးသက်ရောက်မှုကို စုပုံထားနိုင်သည်။ သေနတ်များအတွက်၊ black powder သို့မဟုတ် single round (တစ်ချက်ပစ်) သေနတ်များအတွက် အသုံးဝင်သော်လည်း ဤအကျိုးသက်ရောက်မှုသည် အလိုအလျောက်လက်နက်များတွင် သိသာထင်ရှားသော အသုံးချမှု နည်းပါးသည်၊ ထိုလက်နက်များတွင် နောက်ဆုံးပစ်ခတ်မှုကို ချက်ချင်း chamber (သို့မဟုတ် charge) လုပ်ပြီးသား ဖြစ်သည်။ သို့သော် Rapid Reload (လျင်မြန်စွာ ပြန်လည်တင်ဆောင်ခြင်း) ကို လက်နက်မဂ္ဂဇင်းများကို လျင်မြန်စွာ ပြောင်းလဲရန်အတွက် အသုံးပြုနိုင်သည်။ capacitor များကို ပြန်လည်အားသွင်းရန် သို့မဟုတ် barrel ကို အအေးခံရန် အချိန်ကြာမြင့်သော cycle time (လည်ပတ်ချိန်) ရှိသည့် စွမ်းအင်လက်နက်များအတွက်လည်း အသုံးဝင်နိုင်သည်။

#### ရန်သူကို အမာရွတ်ရစေခြင်း

တိုက်ခိုက်သူသည် ၎င်း၏ပြိုင်ဘက်အား ၎င်း၏ဘဝတစ်လျှောက်လုံး ပြောဆိုဆွေးနွေးစရာဖြစ်မည့် အမာရွတ်တစ်ခု ဖြစ်ပေါ်စေသည် - ဥပမာ- လည်ပင်းနီးပါးပြတ်ရှရာ သို့မဟုတ် ရင်ဘတ်တွင် 'Z' အက္ခရာကို လိမ္မာပါးနပ်စွာ ရေးထွင်းခြင်း။

#### ပစ်မှတ်ရွေးချယ်ခြင်း

တိုက်ခိုက်သူ Fumbles (ချော်လဲခြင်း) ဖြစ်သောအခါ ကာကွယ်သူသည် ထိုးနှက်ချက်ကို အနီးအနားရှိ ဘေးနားရှိသူတစ်ဦးကို ထိမှန်စေမည့်ပုံစံဖြင့် လှုပ်ရှားခြင်း သို့မဟုတ် လွှဲပြောင်းပစ်ခြင်း။ ၎င်းသည် တိုက်ခိုက်သူ၏ လက်ချင်းထိတိုက်ခိုက်လက်နက် (close combat weapon) ရောက်နိုင်သည့်အကွာအဝေးအတွင်း ပစ်မှတ်အသစ်ရှိရန် လိုအပ်သည်၊ သို့မဟုတ် အဝေးပစ်တိုက်ခိုက်မှု (ranged attack) ဖြစ်ပါက ပစ်ခတ်သည့်လမ်းကြောင်းတစ်လျှောက်တွင် ရပ်နေရမည်။ ပစ်မှတ်အသစ်သည် မမျှော်လင့်ဘဲ ဖြစ်ပေါ်လာသော မတော်တဆမှုကြောင့် လုံးဝအံ့အားသင့်သွားပြီး တိုက်ခိုက်မှုကို ရှောင်ရှားရန် အခွင့်အရေးမရှိဘဲ အလိုအလျောက် ထိမှန်သည်။ သို့သော် လျော်ကြေးအနေဖြင့် ၎င်းတို့သည် မည်သည့်အထူးအကျိုးသက်ရောက်မှု (Special Effect) ကိုမျှ ခံစားရမည်မဟုတ်ပေ။

#### လွတ်မြောက်ခြင်း

Critical တွင် ကာကွယ်သူသည် Entangled (ရှုပ်ထွေးနေခြင်း)၊ Gripped (ဆုပ်ကိုင်ခံရခြင်း) သို့မဟုတ် Pinned (ပိတ်မိနေခြင်း) မှ အလိုအလျောက် လွတ်မြောက်နိုင်သည်။

#### နေရာကို ထုံစေခြင်း

တိုက်ခိုက်သူသည် ရိုက်နှက်လက်နက်ကို အသုံးပြု၍ ထိမှန်သော ကိုယ်ခန္ဓာအစိတ်အပိုင်းကို ယာယီ ထုံစေနိုင်သည်။ ထိုးနှက်ချက်က သံချပ်ကာမှတ် (Armor Points) ကို ကျော်လွှားပြီး ပစ်မှတ်ကို ဒဏ်ရာရစေပါက ကာကွယ်သူသည် Endurance ၏ Opposed Roll (ပြိုင်ဆိုင်လှိမ့်ခြင်း) ကို မူလတိုက်ခိုက်မှုလှိမ့်ခြင်း (original attack roll) နှင့် ဆန့်ကျင်၍ ပြုလုပ်ရမည်။ ကာကွယ်သူ Fails (မအောင်မြင်) ပါက ထိမှန်သည့်နေရာ (Hit Location) သည် ထိခိုက်မှုဖြစ်ပေါ်စေသော အရေအတွက်နှင့် ညီမျှသော အလှည့်များ (Turns) အတွက် အားအင်ကုန်ခမ်း (Incapacitated) သည်။ ကိုယ်ခန္ဓာကို ထိမှန်ခြင်းသည် ကာကွယ်သူအား အသက်ရှုကျပ်ပြီး ယိုင်ထိုးစေကာ ခုခံကာကွယ်ရန်သာ စွမ်းဆောင်နိုင်စေသည်။ ဦးခေါင်းကို ပစ်ခတ်ခြင်းသည် ရန်သူအား ခဏတာ သတိမေ့သွားပြီး လုပ်ဆောင်နိုင်ခြင်းမရှိတော့ပေ။

#### ဖြတ်တောက်ခြင်း

တိုက်ခိုက်သူသည် ပုဆိန်ကဲ့သို့သော သင့်လျော်သော လက်နက်ကို အသုံးပြု၍ ပြိုင်ဘက်၏ သံချပ်ကာ သို့မဟုတ် သဘာဝကာကွယ်မှုကို ပျက်စီးစေနိုင်သည်။ ပိတ်ဆို့ခြင်း သို့မဟုတ် မှော်ပညာကြောင့် လျှော့ချပြီးနောက် မည်သည့်လက်နက်ထိခိုက်မှုမဆို ကာကွယ်မှု၏ သံချပ်ကာမှတ် (Armor Point) တန်ဖိုးကို ဆန့်ကျင်၍ သက်ရောက်သည်။ ၎င်း၏ Armor Points ထက် ပိုလျှံသော ထိခိုက်မှုများကို ထိုသံချပ်ကာ (သို့မဟုတ် သံချပ်ကာဝတ်ထားသော) နေရာ၏ Armor Point တန်ဖိုးကို လျှော့ချရန် အသုံးပြုသည် - ကြိုးများကို ဆွဲဖြဲခြင်း၊ ကွင်းများကို ပေါက်ပြဲစေခြင်း၊ ပန်းကန်ပြားများကို ကွေးကောက်စေခြင်း သို့မဟုတ် ဘီလူးများ၏ အရေခွံ၊ အကြေးခွံများ သို့မဟုတ် chitin ကို ဆွဲဖြဲခြင်း။ ကာကွယ်မှုကို သုည AP အထိ လျှော့ချပြီးနောက် ထိခိုက်မှုများ ကျန်ရှိနေပါက ၎င်းသည် ထိမှန်သည့်နေရာ၏ Hit Points (ထိမှန်မှတ်များ) သို့ ရောက်ရှိသွားသည်။ သေနတ်များအတွက်၊ Sunder (ဖြတ်တောက်ခြင်း) သည် ablative armors (အပူခံသံချပ်ကာများ) သို့မဟုတ် gravity wave guns (ဆွဲငင်အားလှိုင်းသေနတ်များ) ကဲ့သို့သော ပိုမိုထူးခြားသည့် အယူအဆများကို ဖျက်ဆီးရန် ဒီဇိုင်းထုတ်ထားသော စွမ်းအင်အခြေခံသေနတ်များအတွက် သင့်လျော်မည်ဖြစ်သည်။

#### ပြိုင်ဘက်ကို ချော်လဲစေခြင်း

ဇာတ်ကောင်သည် ၎င်း၏ပြိုင်ဘက်ကို ချိန်ခွင်လျှာပျက်အောင် သို့မဟုတ် မြေပြင်သို့ ပစ်ချရန် ကြိုးပမ်းသည်။ ပြိုင်ဘက်သည် ၎င်း၏ Brawn (ကြွက်သားအား)၊ Evade (ရှောင်တိမ်း) သို့မဟုတ် Acrobatics (กายกรรม) ၏ Opposed Roll (ပြိုင်ဆိုင်လှိမ့်ခြင်း) ကို ဇာတ်ကောင်၏ မူလလှိမ့်ခြင်း (original roll) နှင့် ဆန့်ကျင်၍ ပြုလုပ်ရမည်။ ပစ်မှတ် မအောင်မြင်ပါက ၎င်းတို့သည် လဲကျသည်။ ခြေလေးချောင်းရှိသော ပြိုင်ဘက်များ (သို့မဟုတ် ခြေထောက်ပိုများသော သတ္တဝါများ) သည် ၎င်းတို့၏ Athletics (အားကစား) ကျွမ်းကျင်မှုကို Evade (ရှောင်တိမ်း) အတွက် အစားထိုးနိုင်ပြီး လှိမ့်ခြင်းကို အခက်အခဲအဆင့် (Difficulty Grade) တစ်ဆင့် ပိုမိုလွယ်ကူသည်ဟု မှတ်ယူသည်။ သေနတ်များအတွက်၊ သေနတ်သည် ထုထည်ရှိသော ခဲယမ်းများကို ပစ်ခတ်သည်ဟု ယူဆပါက ဤအကျိုးသက်ရောက်မှုကို ပြိုင်ဘက်၏ ခြေထောက်များကို ပစ်ခတ်သည့်အခါ အသုံးပြုနိုင်ခြင်းမရှိသည့် အကြောင်းပြချက်မရှိပါ၊ ဒဏ်ရာမရှိသည့်တိုင်အောင်ပင်။

#### လက်နက်ချို့ယွင်းခြင်း

တိုက်ခိုက်သူ Fumbles (ချော်လဲခြင်း)၊ သေနတ်များသာ။ တိုက်ခိုက်သူ၏ သေနတ်သည် ချို့ယွင်းမှုဖြစ်ပေါ်ပြီး ၎င်းကို ဖြုတ်တပ်ကာ သန့်ရှင်းရေးလုပ်ရန် အချိန်ယူနိုင်သည်အထိ အသုံးမဝင်တော့ပေ။

#### ဆုတ်ခွာခြင်း

ကာကွယ်သူသည် အလိုအလျောက် ရောက်ရှိရာမှ ဆုတ်ခွာပြီး ထိုသီးခြားပြိုင်ဘက်နှင့် ထိတွေ့မှုကို ဖြတ်တောက်နိုင်သည်။

## လက်ချင်းထိတိုက်ခိုက်ခြင်း

လက်ချင်းထိတိုက်ခိုက်ခြင်း (Close combat) သည် တိုက်ခိုက်သူများ လက်ကိုင်လက်နက်များ သို့မဟုတ် သဘာဝလက်နက်များဖြင့် အပြန်အလှန် တိုက်ခိုက်သည့် ဂန္ထဝင်လက်ချင်းထိတိုက်ခိုက်မှုမျိုး ဖြစ်သည်။ အောက်ပါ လမ်းညွှန်ချက်များသည် လက်ချင်းထိတိုက်ခိုက်ခြင်းကို ပိုမိုစိတ်ဝင်စားဖွယ်ဖြစ်စေရန်နှင့် ဤအခန်း၏ အစောပိုင်းတွင် ဖော်ပြခဲ့သော အယူအဆအချို့ကို ပိုမိုရှင်းလင်းစေရန် ကူညီပေးသည်။

### အခြေအနေဆိုင်ရာ မွမ်းမံမှုများ

အချို့သော အခြေအနေများတွင် တိုက်ခိုက်သည့်အခါ သို့မဟုတ် အထူးအကျိုးသက်ရောက်မှု (Special Effect) ရလဒ်အဖြစ် အခြေအနေဆိုင်ရာ မွမ်းမံမှုများကို အသုံးပြုနိုင်သည်။ ဥပမာအားဖြင့် အမှောင်ထဲတွင် တိုက်ခိုက်ခြင်းသည် Herculean (ကြီးမားသော) အခက်အခဲအဆင့် (Difficulty Grade) ကို ဖြစ်ပေါ်စေနိုင်သည်။ အခြားနည်းဖြင့် မဖော်ပြထားပါက မွမ်းမံမှုများကို ဂိမ်းမာစတာက ဆုံးဖြတ်သည်။ အခြေအနေနှစ်ခု သို့မဟုတ် ထို့ထက်ပို၍ ဇာတ်ကောင်နှင့် သက်ဆိုင်ပါက အပြင်းထန်ဆုံးကို အသုံးပြုပါ။

အချို့သော မွမ်းမံမှုများသည် အခြေအနေအပေါ်မူတည်သော်လည်း အခက်အခဲအဆင့်ကို အသုံးပြုခြင်းအစား တိုက်ခိုက်ရေးကျွမ်းကျင်မှု (Combat Skill) ကို သက်ဆိုင်ရာကျွမ်းကျင်မှုဖြင့် ကန့်သတ်ထားသည်။ ဥပမာအားဖြင့် မြင်းကျောပေါ်မှ တိုက်ခိုက်ခြင်းသည် တိုက်ခိုက်သူ၏ Combat Skill ကို ၎င်းတို့၏ Ride skill (မြင်းစီးကျွမ်းကျင်မှု) ဖြင့် ကန့်သတ်မည်ဖြစ်ပြီး ရေထဲတွင် တိုက်ခိုက်ခြင်းသည် ၎င်းကို ဇာတ်ကောင်၏ Swim (ရေကူး) တန်ဖိုးဖြင့် ကန့်သတ်နိုင်မည်ဖြစ်သည်။

### ထိတွေ့မှု

ဇာတ်ကောင်တစ်ဦးသည် ပြိုင်ဘက်၏ လက်ချင်းထိလက်နက် (melee weapon) အကွာအဝေးအတွင်း ရှိနေပါက ထိတွေ့မှု (Engaged) ရှိသည်ဟု မှတ်ယူသည်။ ၎င်းသည် တိုက်ခိုက်သူနှစ်ဦးစလုံး အချင်းချင်းရောက်နိုင်ရန် မလိုအပ်ပါ။ ၎င်းတို့ထဲမှ တစ်ဦးက အခြားတစ်ဦးကို တိုက်ခိုက်နိုင်ခြေရှိလျှင်သာ ဖြစ်သည်။ ဇာတ်ကောင်တစ်ဦးသည် ပြိုင်ဘက်နှင့် ထိတွေ့ပြီးသည်နှင့် ထိုတိုက်ပွဲမှ လွတ်လပ်စွာ ထွက်ခွာနိုင်တော့မည်မဟုတ်ပေ (ကျော်ဖြတ်ပြေးခြင်း (charging past) သည် အထူးခြွင်းချက်တစ်ခုဖြစ်သည်)။ ရန်သူနှင့် အဆက်အသွယ်ဖြတ်ရန်အတွက် တမင်တကာလုပ်ဆောင်ချက်တစ်မျိုးမျိုး လိုအပ်ပြီး ပုံမှန်အားဖြင့် Outmaneuver (အသာစီးရယူခြင်း) ကဲ့သို့သော တိုက်ခိုက်ရေးလုပ်ဆောင်ချက် (Combat Action) သို့မဟုတ် Withdraw (ဆုတ်ခွာခြင်း) Special Effect (အထူးအကျိုးသက်ရောက်မှု) ကို အသုံးပြုခြင်းဖြင့် လုပ်ဆောင်သည်။

### တိုက်ခိုက်ရန်ပြေးခြင်း (Charging)

တိုက်ခိုက်ရန်ပြေးခြင်း (Charge action) သည် တိုက်ခိုက်မှု၏ စွမ်းအားနှင့် သက်ရောက်မှုကို တိုးမြှင့်ရန်အတွက် လျင်မြန်စွာ ရွေ့လျားခြင်းဖြစ်သည်။ တိုက်ခိုက်ရန်ပြေးခြင်းအတွက် တိုက်ခိုက်သူသည် Run (ပြေးနှုန်း) သို့မဟုတ် Sprint Gait (အပြေးမြန်နှုန်း) ဖြင့် ရွေ့လျားခြင်း (အထက်ပါ [Movement Rate](0002_Characters.md?id=movement-rate) အပိုင်းကို ကြည့်ပါ) ကို ရွေးချယ်ရမည်။ ရွေ့လျားမှုစည်းမျဉ်းများကို _Mythras_ စည်းမျဉ်းစာအုပ်တွင် အပြည့်အစုံ ဖော်ပြထားသည်။

၎င်းသည် ၎င်းတို့အား ပြိုင်ဘက်နှင့် ထိတွေ့စေပါက ၎င်းတို့သည် ထိုနေရာတွင် ရပ်တန့်ပြီး တိုက်ခိုက်ရန်ပြေးခြင်းအတွက် ပြင်ဆင်ထားသည့်အတိုင်း တိုက်ခိုက်မှုလုပ်ဆောင်ချက် (Attack action) ကို လုပ်ဆောင်ရမည်။ တိုက်ခိုက်ရန်ပြေးခြင်းမပြုမီ ဖုံးအုပ်ထားရမည့် အနည်းဆုံးအကွာအဝေးမရှိပါ၊ အကြောင်းမှာ ရွေ့လျားသည့်အကွာအဝေးသည် ရရှိသည့်အရှိန်အဟုန်ထက် အရေးပါမှုနည်းသောကြောင့်ဖြစ်သည်။ ထို့ကြောင့် ပြေးခြင်း သို့မဟုတ် Sprinting (အပြေးမြန်ခြင်း)။

တိုက်ခိုက်ရန်ပြေးနေစဉ် တိုက်ခိုက်မှုလှိမ့်ခြင်း (attack roll) သည် Hard (ခက်ခဲသော) အခက်အခဲအဆင့် (Difficulty Grade) ကို ခံစားရသည်။ အပြန်အလှန်အားဖြင့် တိုက်ခိုက်ရန်ပြေးခြင်းသည် တိုက်ခိုက်လက်နက်၏ အရွယ်အစား (Size) ကို တစ်ဆင့်တိုးမြှင့်ပြီး ခြေနှစ်ဖက်ဖြင့် တိုက်ခိုက်သူ၏ Damage Modifier (ထိခိုက်မှုမွမ်းမံမှု) ကို တစ်ဆင့်တိုးမြှင့်သည်၊ သို့မဟုတ် ခြေလေးချောင်းရှိသော သတ္တဝါ (သို့မဟုတ် ခြေထောက်ပိုများပါက) နှစ်ဆင့်တိုးမြှင့်သည်။ မြင်းစီးဇာတ်ကောင်များသည် ၎င်းတို့၏ Damage Modifier ကို ၎င်းတို့၏ စီးနင်းသူ၏ Damage Modifier ဖြင့် အစားထိုးနိုင်သည်။ တိုက်ခိုက်ရန်ပြေးနေသော ဇာတ်ကောင်၏ နောက်ဆုံးအနေအထားသည် လဲလှယ်မှု၏ ရလဒ်နှင့် တိုက်ခိုက်သူက ရပ်တန့်လိုခြင်း သို့မဟုတ် ကျော်ဖြတ်ဆက်သွားပြီး ချက်ချင်းထိတွေ့မှုကို ဖြတ်တောက်လိုခြင်းအပေါ် မူတည်သည်။

Parrying (ပိတ်ဆို့ခြင်း) သို့မဟုတ် Evading (ရှောင်တိမ်းခြင်း) အစား တိုက်ခိုက်ရန်ပြေးခြင်းကို လက်ခံရရှိသူသည် တိုက်ခိုက်ရန်ပြေးနေသော တိုက်ခိုက်သူကို တစ်ပြိုင်နက်တည်း တန်ပြန်တိုက်ခိုက်နိုင်သည်။ ဤကိစ္စတွင် ပိုရှည်သော လက်နက်ကို ကိုင်ဆောင်သူက ဦးစွာတိုက်ခိုက်သည်။ လက်နက်ကို တိုက်ခိုက်ရန်ပြေးခြင်းကို လက်ခံရန် သတ်မှတ်နိုင်ပါက (လှံကဲ့သို့) လက်ခံရရှိသူသည် ၎င်း၏ကိုယ်ပိုင် Damage Modifier အစား တိုက်ခိုက်ရန်ပြေးသူ၏ Damage Modifier ကို အသုံးပြုနိုင်သည်။

### ကာဗာ

ပြိုင်ဘက်များသည် ၎င်းတို့ကို တိုက်ခိုက်ခြင်းမှ ကာကွယ်ရန် ကာဗာကို အသုံးပြုသည်၊ ဝင်ရောက်စွက်ဖက်နေသော ပစ္စည်း၏ ခံနိုင်ရည်အားကြောင့် ထိုးနှက်ချက်ကို ရုပ်ပိုင်းဆိုင်ရာ ပိတ်ဆို့ခြင်းဖြင့် သို့မဟုတ် ရန်သူ မည်သည့်နေရာတွင် ရှိသည်ကို ဝိုးတဝါးဖြစ်စေခြင်းဖြင့် ဖြစ်သည်။ ကာဗာ၏ တိကျသောတန်ဖိုးသည် ၎င်း၏ကာကွယ်မှု၏ အထူနှင့် ၎င်း၏ဖုံးအုပ်မှုအတိုင်းအတာပေါ် မူတည်သည်။

ဖုံးအုပ်ထားသော ထိမှန်သည့်နေရာ (Hit Location) တွင် ကျရောက်သော ပစ်မှတ်ကို တိုက်ခိုက်ခြင်းသည် ကာဗာ၏ မွေးရာပါ ကာကွယ်မှုအတိုင်းအတာအထိ ပိတ်ဆို့ခံရမည်။ ကာဗာကို Choose Location Special Effect (နေရာရွေးချယ်ခြင်း အထူးအကျိုးသက်ရောက်မှု) ကို အသုံးပြု၍ ပစ်မှတ်၏ မြင်နိုင်သော အစိတ်အပိုင်းများကို ပစ်မှတ်ထားခြင်းဖြင့် သို့မဟုတ် လက်နက်သည် ၎င်းကို ထိုးဖောက်နိုင်ပါက ကာဗာကို ဖြတ်၍ တိုက်ခိုက်ခြင်းဖြင့် တစ်စိတ်တစ်ပိုင်း ပယ်ဖျက်နိုင်သည်။

ကာဗာသည် လုံးဝကာကွယ်ထားသော်လည်း တိုက်ခိုက်သူက ၎င်း၏တိုက်ခိုက်မှုသည် ထိုးဖောက်နိုင်မည်ဟု ယုံကြည်ပါက ၎င်းတို့သည် ပုန်းအောင်းနေသော ပစ်မှတ်ကို မျက်ကွယ်ပစ်ခတ်နိုင်သည်။ ထိုသို့သောကိစ္စများတွင် တိုက်ခိုက်မှုလှိမ့်ခြင်း (attack roll) သည် တစ်ဆင့်ပိုမိုခက်ခဲပြီး အထူးအကျိုးသက်ရောက်မှု (Special Effect) ရရှိပါက Choose Location (နေရာရွေးချယ်ခြင်း) ကို ယူဆောင်နိုင်မည်မဟုတ်ပေ။ ၎င်းသည် တိုက်ခိုက်သူက ကာဗာနောက်ရှိ ပစ်မှတ်၏အနေအထားကို တိကျစွာ သိရှိသည်ဟု ယူဆသည်။

### ရှောင်တိမ်းခြင်း

တိုက်ခိုက်ခြင်းတွင် ရှောင်တိမ်းခြင်းသည် ပြင်းထန်သော တိုက်ခိုက်မှုတစ်ခုမှ မိမိကိုယ်ကို လွတ်အောင် ခုန်ချခြင်းဖြစ်သည်။ ၎င်းအတွက် ကာကွယ်သူ၏ Evade skill (ရှောင်တိမ်းကျွမ်းကျင်မှု) နှင့် တိုက်ခိုက်သူ၏ သက်ဆိုင်ရာလှိမ့်ခြင်း (roll) တို့၏ Opposed Roll (ပြိုင်ဆိုင်လှိမ့်ခြင်း) လိုအပ်သည်။ ၎င်းသည် ရွေ့လျားနေသော စစ်တိုက်ကားတစ်စီး၏ Drive skill (မောင်းနှင်ကျွမ်းကျင်မှု) မှ မှော်ဆရာတစ်ဦး၏ စာလုံးပေါင်းရွတ်ဆိုကျွမ်းကျင်မှု (spell casting skill) အထိ မည်သည့်အရာမဆို ဖြစ်နိုင်သည်။ တိုက်ခိုက်သူ အနိုင်ရပါက ၎င်းတို့သည် ပုံမှန်အတိုင်း ထိခိုက်မှု (damage) ကို ဖြစ်ပေါ်စေသည်။ ကာကွယ်သူ အနိုင်ရပါက ထိခိုက်မှုကို လုံးဝရှောင်ရှားနိုင်သည်။

Opposed Roll (ပြိုင်ဆိုင်လှိမ့်ခြင်း) ၏ အနိုင်ရသူသည် ၎င်း၏ပြိုင်ဘက်အပေါ် တစ်ဆင့် သို့မဟုတ် ထို့ထက်ပိုသော Success (အောင်မြင်မှု) အဆင့်များကို ရရှိပါက ပုံမှန်တိုက်ခိုက်ခြင်းအတိုင်း သင့်လျော်သော Special Effects (အထူးအကျိုးသက်ရောက်မှုများ) ကို ရွေးချယ်နိုင်သည်။ မည်သို့ပင် ရလဒ်ဖြစ်စေ၊ ရှောင်တိမ်းသည့် လှုပ်ရှားမှုသည် ကာကွယ်သူအား လဲကျနေအောင် ဖြစ်စေသည် - ပုံမှန်အားဖြင့် ဇာတ်ကောင်သည် ၎င်း၏ နောက်အလှည့်တွင် Regain Footing Action (ခြေထောက်ပြန်ထိန်းမတ်ခြင်းလုပ်ဆောင်ချက်) ကို ပြန်လည်မတ်တပ်ရပ်ရန် လုပ်ဆောင်ရန် လိုအပ်သည် - Arise Special Effect (ထခြင်း အထူးအကျိုးသက်ရောက်မှု) ကို အသုံးမပြုနိုင်ပါက။

### နောက်ပြန်လွင့်ကျခြင်း

လက်ခံရရှိသူ၏ SIZ (အရွယ်အစား) ထက် ပိုမိုထိခိုက်မှုပေးသော တိုက်ခိုက်မှုသည် ပုံသေအားဖြင့် ဇာတ်ကောင်အား နောက်ပြန်လွင့်ကျစေသည်။ ဤအခြေအနေရှိ ထိခိုက်မှုသည် ပိတ်ဆို့ခြင်း သို့မဟုတ် သံချပ်ကာကြောင့် မည်သည့်လျှော့ချမှုမှ မပြုမီဖြစ်သည်။ ထိုသို့သော ထိုးနှက်ချက်ကို ခံရသောအခါ လက်ခံရရှိသူသည် လဲကျခြင်းမှ ရှောင်ရှားရန် Easy Acrobatics သို့မဟုတ် Standard Athletics လှိမ့်ခြင်း (roll) ကို အောင်မြင်ရမည်။ ၎င်းတို့သည် ၎င်းတို့၏ SIZ ထက် ပိုလျှံသော ထိခိုက်မှု (damage) ရမှတ်ငါးမှတ် (သို့မဟုတ် ၎င်း၏ အပိုင်းအစ) တိုင်းအတွက် တစ်မီတာ နောက်သို့ တွန်းပို့ခံရသည်။ Brace Combat Action (ထောက်ခြင်း တိုက်ခိုက်ရေးလုပ်ဆောင်ချက်) ကို တက်ကြွစွာ အသုံးပြုခြင်းသည် နောက်ပြန်လွင့်ကျခြင်း၏ အကျိုးသက်ရောက်မှုများကို လျှော့ချနိုင်သည် သို့မဟုတ် လုံးဝ ပယ်ဖျက်နိုင်သည်။

### ခုန်ပေါက်တိုက်ခိုက်ခြင်း

ခုန်ပေါက်တိုက်ခိုက်မှုများကို အခြေအနေအမျိုးမျိုးမှ စတင်နိုင်သည်။ အများစုမှာ ချုံခိုတိုက်ခိုက်မှု သို့မဟုတ် တိုက်ခိုက်ရန်ပြေးခြင်း (charge) အဆုံးတွင် ဖြစ်ပေါ်သော်လည်း အချို့သတ္တဝါများသည် ပြေးဆွဲရန် သို့မဟုတ် အမြင့်ထက်သာလွန်ရန် မလိုအပ်ဘဲ ပြိုင်ဘက်တစ်ဦး၏ ထိပ်သို့ ခုန်ပေါက်နိုင်သည်။

ခုန်ပေါက်တိုက်ခိုက်မှုကို ခုန်ပေါက်သူ၏ Athletics skill (အားကစားကျွမ်းကျင်မှု) နှင့် ကာကွယ်သူ၏ Brawn (ကြွက်သားအား) သို့မဟုတ် Evade skill (ရှောင်တိမ်းကျွမ်းကျင်မှု) တို့၏ Opposed Roll (ပြိုင်ဆိုင်လှိမ့်ခြင်း) ဖြင့် ဖြေရှင်းသည်။ ခြေလေးချောင်းရှိသော ပစ်မှတ်များသည် တိုက်ခိုက်မှုလှိမ့်ခြင်း (attack roll) ကို အခက်အခဲအဆင့် (Difficulty Grade) တစ်ဆင့် ပိုမိုခက်ခဲစေသည်။ ခုန်ပေါက်သူ အနိုင်ရပါက ကာကွယ်သူသည် အလိုအလျောက် လဲကျသွားပြီး တိုက်ခိုက်သူသည် ၎င်းတို့၏ကိုယ်ပေါ်တွင် စီးနင်းလျက်ရှိသည်။ မအောင်မြင်ပါက ကာကွယ်သူသည် သက်ရောက်မှုကို ခံနိုင်ရည်ရှိခဲ့သည် သို့မဟုတ် ရှောင်တိမ်းခဲ့သည်ဟု ဆိုလိုသည်။ Opposed Roll (ပြိုင်ဆိုင်လှိမ့်ခြင်း) ၏ အနိုင်ရသူသည် ၎င်း၏ပြိုင်ဘက်အပေါ် တစ်ဆင့် သို့မဟုတ် ထို့ထက်ပိုသော Success (အောင်မြင်မှု) အဆင့်များကို ရရှိပါက ပုံမှန်တိုက်ခိုက်ခြင်းအတိုင်း သင့်လျော်သော Special Effects (အထူးအကျိုးသက်ရောက်မှုများ) ကို ရွေးချယ်နိုင်သည်။

ခုန်ပေါက်ခြင်း၏ အစိတ်အပိုင်းအဖြစ် မည်သည့်ထိခိုက်မှုကိုမျှ မဖြစ်ပေါ်စေပါ။ သို့သော် နောက်ပိုင်းတွင် လဲကျနေသော သားကောင်သည် ၎င်း၏တိုက်ခိုက်သူ ထွက်ခွာသွားသည်အထိ သို့မဟုတ် Arise (ထခြင်း) ကို ခွင့်ပြုသည့် Special Effect (အထူးအကျိုးသက်ရောက်မှု) ကို နောက်ဆုံးတွင် အနိုင်ရသည်အထိ ၎င်း၏ခြေထောက်ကို ပြန်မထူနိုင်ပါ။ ထို့အပြင် ခုန်ပေါက်တိုက်ခိုက်မှုများသည် တိုက်ခိုက်သူ၏ SIZ (အရွယ်အစား) နှစ်ဆအထိရှိသော ပြိုင်ဘက်များကိုသာ တိုက်ခိုက်နိုင်သည်။ ပစ်မှတ်သည် လာမည့် ခုန်ပေါက်တိုက်ခိုက်မှုကို သိရှိပါက Brace Combat Action (ထောက်ခြင်း တိုက်ခိုက်ရေးလုပ်ဆောင်ချက်) ကို တက်ကြွစွာ အသုံးပြုခြင်းဖြင့် မိမိကိုယ်ကို ချော်လဲရန် ပိုမိုခက်ခဲအောင် ပြုလုပ်နိုင်သည်။

### အသာစီးရယူခြင်း

ပြိုင်ဘက်များစွာနှင့် ရင်ဆိုင်နေရသော ဇာတ်ကောင်တစ်ဦးသည် ၎င်းကို မည်သည့်အချိန်တွင်မဆို တိုက်ခိုက်နိုင်သည့် အရေအတွက်ကို ကန့်သတ်ရန် ရွေ့လျားမှုကို အသုံးပြုနိုင်သည်၊ အဆက်မပြတ် အနေအထားပြောင်းလဲနေပြီး ရန်သူများကို အချင်းချင်း အနှောင့်အယှက်ဖြစ်စေရန် တွန်းအားပေးသည်။ အသာစီးရယူခြင်း (Outmaneuvering) အတွက် ဇာတ်ကောင်သည် လှုပ်ရှားရန် နေရာရှိရန် လိုအပ်ပြီး ကန့်သတ်ထားသော နေရာတွင် ပိတ်မိနေခြင်း မရှိရပါ။

အသာစီးရယူခြင်း (Outmaneuvering) အတွက် ဇာတ်ကောင်သည် ၎င်း၏ပြိုင်ဘက်များကို Evade skills (ရှောင်တိမ်းကျွမ်းကျင်မှု) ၏ အုပ်စုလိုက် ပြိုင်ဆိုင်လှိမ့်ခြင်း (Opposed Roll) တွင် ပါဝင်ရမည်။ ပါဝင်သူတိုင်း၊ လှုပ်ရှားနေသော ဇာတ်ကောင်နှင့် ၎င်းကို ထောင့်စွန်းသို့ ပိတ်ဆို့လိုသော ရန်သူများ နှစ်ဦးစလုံးသည် လုပ်ဆောင်ချက်မှတ် (Action Point) တစ်မှတ်စီ သုံးစွဲရမည်။ ထို့နောက် ၎င်းတို့ တစ်ဦးစီက တစ်ကြိမ်လှိမ့်ပြီး လှုပ်ရှားနေသော ဇာတ်ကောင်၏လှိမ့်ခြင်းကို မကျော်နိုင်သူများသည် ထိုတိုက်ခိုက်ရေးအချီ (Combat Round) ၏ ကျန်ရှိသောကာလအတွက် ၎င်းကို တိုက်ခိုက်နိုင်မည်မဟုတ်ပေ၊ ၎င်းတို့၏ မဟာမိတ်များ သို့မဟုတ် မြေပြင်အနေအထားများကြောင့် ပိတ်ဆို့ခံရသောကြောင့် ဖြစ်သည်။

လှုပ်ရှားနေသော ဇာတ်ကောင်သည် ၎င်း၏ပြိုင်ဘက်အားလုံးကို အနိုင်ရပါက ကျန်ရှိသော အချီအတွက် ပြိုင်ဘက်တစ်ဦးတည်းကို ဘေးကင်းစွာ ထိတွေ့ရန် သို့မဟုတ် တိုက်ပွဲမှ လုံးဝဆုတ်ခွာရန် ရွေးချယ်ခွင့်ရှိသည်။

### ကာကွယ်မှုဖြင့် ပိတ်ဆို့ခြင်း (Passive Blocking)

ကာကွယ်မှုဖြင့် ပိတ်ဆို့ခြင်း (Passive Blocking) သည် လက်နက်ကိုင် ဇာတ်ကောင်တစ်ဦးအား ၎င်းကို ရွေးချယ်ထားသော နေရာတစ်ခု (သို့မဟုတ် ဒိုင်းကို အသုံးပြုသောအခါ နေရာများစွာ) ကို ဖုံးအုပ်ရန် ကိုင်ထားနိုင်စေသော်လည်း ၎င်းဖြင့် တက်ကြွစွာ ပိတ်ဆို့နိုင်စွမ်း (Parry) (Ward Location ကိုကြည့်ပါ) ကို ဆုံးရှုံးစေသည်။ ထိုနေရာများကို ထိမှန်သော မည်သည့်တိုက်ခိုက်မှုမဆို လက်နက် သို့မဟုတ် ဒိုင်း၏ အကျိုးကျေးဇူးကို အလိုအလျောက် ရရှိပြီး ထိခိုက်မှု (damage) ကို ပုံမှန်အတိုင်း လျှော့ချပေးသည်။ ဤနည်းစနစ်ကို စစ်သည်တော်တစ်ဦးက အားနည်းသောနေရာ သို့မဟုတ် ဒဏ်ရာရသောနေရာကို ကာကွယ်လိုသည့်အခါ အများအားဖြင့် အသုံးပြုသည်။ လက်နက်နှစ်ခု သို့မဟုတ် လက်နက်နှင့် ဒိုင်းကိုင်ဆောင်ထားသော တိုက်ခိုက်သူတစ်ဦးအား ၎င်း၏ အခြားလက်နက်ကို တက်ကြွစွာ ပိတ်ဆို့ခြင်းမှ တားဆီးထားခြင်းမရှိပါ။

Passive Blocking (ကာကွယ်မှုဖြင့် ပိတ်ဆို့ခြင်း) သည် cover (ခိုလှုံရာ) နှင့် တူညီစွာ အလုပ်လုပ်သောကြောင့် Close Combat (လက်ချင်းထိတိုက်ခိုက်ခြင်း) တွင် ဤပိတ်ဆို့မှုကို ကျော်ဖြတ်ရန် Choose Location Special Effect (နေရာရွေးချယ်ခြင်း အထူးအကျိုးသက်ရောက်မှု) ကို အသုံးပြု၍ မရနိုင်သော်လည်း ပြိုင်ဘက်များသည် ပိတ်ဆို့မထားသော အခြားနေရာများကို တိုက်ခိုက်ရန် ထိုအကျိုးသက်ရောက်မှုကို ဆက်လက်အသုံးပြုနိုင်သည်။ ဒိုင်းနောက်တွင် ဒူးထောက်ခြင်းသည် ဇာတ်ကောင်အား Passive Blocking လုပ်နေစဉ် ကာကွယ်ထားသော နေရာအရေအတွက်ကို နှစ်ဆတိုးစေသည်။

### အံ့အားသင့်ခြင်း

အံ့အားသင့်ခြင်းသည် တိုက်ခိုက်သူ၏ ရှိနေခြင်း သို့မဟုတ် ရည်ရွယ်ချက်ကို သတိမထားမိသော ပြိုင်ဘက်များကို မမျှော်လင့်ဘဲ တိုက်ခိုက်မှုတစ်ခု စတင်သောအခါ ဖြစ်ပေါ်သည်။ ချုံခိုတိုက်ခိုက်ခြင်းသည် ပထမအမျိုးအစား၏ ဥပမာတစ်ခုဖြစ်ပြီး ချစ်ခင်ရင်းနှီးသော စကားပြောဆိုမှုအတွင်း သတိမထားမိသော မဟာမိတ်တစ်ဦးကို သစ္စာဖောက် တိုက်ခိုက်ခြင်းသည် ဒုတိယအမျိုးအစားကို ဖော်ပြသည်။

ပစ်မှတ်အပေါ် အံ့အားသင့်မှု၏ အကျိုးသက်ရောက်မှုများမှာ ပြင်းထန်သည်။

- ပစ်မှတ်သည် ရှေ့ဦးတည်ချက် (Initiative) တွင် –၁၀ ဒဏ်ကြေးခံစားရသည်။
- ၎င်းတို့၏ ရှေ့ဦးတည်ချက်ရောက်သည်အထိ ၎င်းတို့အား ခြေလှမ်းမမှန်ဘဲ ခုခံနိုင်စွမ်းမရှိဟု မှတ်ယူသည်။
- ပစ်မှတ်အပေါ် ပထမဆုံးတိုက်ခိုက်မှုသည် အောင်မြင်ပါက အပို Special Effect (အထူးအကျိုးသက်ရောက်မှု) တစ်ခု ရရှိသည်။
- အချီ၏ ကျန်ရှိသောကာလအတွက် ၎င်းတို့သည် မည်သည့်ထိုးစစ်လုပ်ဆောင်ချက် (offensive Action) ကိုမျှ မလုပ်ဆောင်နိုင်။

### လှည်းပတ်တိုက်ခိုက်ခြင်းများ

လှည်းပတ်တိုက်ခိုက်ခြင်းများ (Sweep attacks) သည် ဂိမ်းမာစတာ၏ ဆုံးဖြတ်ချက်ဖြင့် ဖြစ်ပေါ်သည်၊ လက်နက်များ သို့မဟုတ် ပုံမှန်မဟုတ်သော အရွယ်အစားရှိ သတ္တဝါများက အနီးကပ်စုဝေးနေသော ပြိုင်ဘက်အုပ်စုတစ်ခုကို တိုက်ခိုက်သောအခါ ဖြစ်သည် - ဥပမာအားဖြင့် ဧရာမနဂါးတစ်ကောင်၏ လှံစွပ်ကဲ့သို့ အမြီး သို့မဟုတ် ဧရာမ triceratops တစ်ကောင်၏ ရပ်တန့်မရနိုင်သော တိုက်ခိုက်ရန်ပြေးခြင်း (charge) ကဲ့သို့ - ရန်သူများစွာကို တစ်ပြိုင်နက်တည်း ထိမှန်ခြင်း။

လှည်းပတ်တိုက်ခိုက်မှုကို လက်နက် သို့မဟုတ် သတ္တဝါ၏ တိုက်ခိုက်မှုလှိမ့်ခြင်း (attack roll) တစ်ခုတည်းကို ၎င်း၏လမ်းကြောင်းရှိ ပစ်မှတ်အားလုံးအပေါ် အသုံးပြု၍ ပြုလုပ်သည်။ ကာကွယ်သူတစ်ဦးစီသည် တိုက်ခိုက်မှု၏ အကျိုးသက်ရောက်မှုများကို သီးခြားစီ ဖြေရှင်းရမည်ဖြစ်ပြီး တိုက်ခိုက်သူအပေါ် ဖြစ်ပေါ်လာသော မည်သည့်အထူးအကျိုးသက်ရောက်မှုများကိုမဆို တစ်ပြိုင်နက်တည်း ဖြစ်ပေါ်ခဲ့သည်ဟု မှတ်ယူသည်။

## အဝေးပစ်တိုက်ခိုက်ခြင်း

အဝေးပစ်တိုက်ခိုက်ခြင်း (Ranged combat) သည် ၎င်းတို့၏ ပစ်မှတ်ကို ထိမှန်ရန် ပစ်ခတ်ရမည့်၊ ပစ်ပေါက်ရမည့် သို့မဟုတ် လှုပ်ခါပစ်ရမည့် လက်နက်အမျိုးအစားအားလုံးကို ပေါင်းစပ်ထားသည်။ အဝေးပစ်တိုက်ခိုက်မှုများကို လက်ချင်းထိတိုက်ခိုက်မှု (close combat) နှင့် အတူတူပင် ဖြေရှင်းသည်။ သို့သော် အဝေးပစ်လက်နက်များကို ပုံမှန်အားဖြင့် ဒိုင်းများဖြင့်သာ ပိတ်ဆို့နိုင်သည်၊ ဒိုင်းမရှိသူများသည် သဘာဝကာဗာကို အားကိုးရမည် သို့မဟုတ် ပစ်ခတ်မှုလမ်းကြောင်းမှ ရှောင်တိမ်းရန် Evade (ရှောင်တိမ်းခြင်း) ကို အသုံးပြုရမည်။ ထို့ကြောင့် သံချပ်ကာအနည်းငယ်သာ ဝတ်ဆင်ထားသော ရန်သူများအတွက် အဝေးပစ်လက်နက်များသည် ကြောက်မက်ဖွယ်ကောင်းသော အဟန့်အတားများ ဖြစ်နိုင်သည်။

အဝေးပစ်လက်နက်တစ်ခုစီတွင် ၎င်း၏ ထိရောက်မှုကို ဆုံးဖြတ်သည့် သီးခြားစရိုက်လက္ခဏာများစွာရှိသည်။ တိုက်ခိုက်ခြင်းအပေါ် အဓိကသက်ရောက်မှုရှိသော အရာများမှာ အောက်ပါအတိုင်းဖြစ်သည်။

- **စွမ်းအား (Force):** Ranged Weapon (အဝေးပစ်လက်နက်) ၏ Size (အရွယ်အစား) နှင့် ညီမျှသည်။ ၎င်းသည် ထိုးနှက်ချက်က (ဒိုင်း) ပိတ်ဆို့မှုကို ကျော်လွှားနိုင်ခြင်းရှိမရှိ ဆုံးဖြတ်ရန်အတွက် လက်နက် သို့မဟုတ် ၎င်း၏ခဲယမ်း၏ ထိုးဖောက်နိုင်စွမ်းကို စိတ္တဇတိုင်းတာမှုတစ်ခုဖြစ်သည်။
- **ထိခိုက်မှုမွမ်းမံမှု (Damage Modifier):** ဤစရိုက်လက္ခဏာသည် တိုက်ခိုက်သူ၏ Damage Modifier ကို Ranged Weapon ၏ ထိခိုက်မှုလှိမ့်ခြင်း (damage roll) ကို မြှင့်တင်ရန်အတွက် အသုံးပြုနိုင်ခြင်းရှိမရှိ ဖော်ပြသည်။ ယေဘုယျအားဖြင့် ကိုယ်တိုင်ဆွဲသောလေးများနှင့် ပစ်ပေါက်လက်နက်များသာ အသုံးပြုသူအား ၎င်းတို့၏ Damage Modifier ကို အသုံးပြုခွင့်ပြုသည်။
- **အကွာအဝေး (Range):** Slash များဖြင့် ပိုင်းခြားထားသော နံပါတ်သုံးလုံးသည် လက်နက် သို့မဟုတ် ၎င်း၏ခဲယမ်း၏ အမြင့်ဆုံး Close (အနီးကပ်)၊ Effective (ထိရောက်သော) နှင့် Long (ဝေးကွာသော) အကွာအဝေးများကို ကိုယ်စားပြုသည်။ Close Range သည် Choose Location Special Effect (နေရာရွေးချယ်ခြင်း အထူးအကျိုးသက်ရောက်မှု) ကို အသုံးပြုနိုင်သည့် အကွာအဝေးဖြစ်သည်၊ ပစ်မှတ်သည် ငြိမ်သက်နေခြင်း သို့မဟုတ် လာမည့်တိုက်ခိုက်မှုကို သတိမထားမိခြင်းမှလွဲ၍ ဖြစ်သည်။ Effective Range တွင် သိသာထင်ရှားသော မွမ်းမံမှုများမရှိပါ။ Long Range တွင် လက်နက်သည် ဒဏ်ရာဖြစ်စေနိုင်သေးသော်လည်း ထိခိုက်မှုပမာဏသည် တစ်ဝက်လျှော့ချပြီး Force (စွမ်းအား) ကို တစ်ဆင့်လျှော့ချသည်။
- **တင်ဆောင်ခြင်း (Load):** ခဲယမ်းပစ်ခတ်သော လက်နက်တစ်ခုကို တင်ဆောင်ရန် သို့မဟုတ် ပြန်လည်တင်ဆောင်ရန် ကြာမြင့်သည့် အလှည့်များ (Turns)။ ဇာတ်ကောင်တစ်ဦးသည် Rapid Reload Special Effect (လျင်မြန်စွာ ပြန်လည်တင်ဆောင်ခြင်း အထူးအကျိုးသက်ရောက်မှု) ကို အသုံးပြုခြင်းဖြင့် တင်ဆောင်ခြင်း သို့မဟုတ် ပြင်ဆင်ခြင်းအတွက် အချိန်ကို လျှော့ချနိုင်သည်။
- **ထိုးဖောက်အရွယ်အစား (Impale Size):** ထိုးဖောက်ထားသော လက်နက်၏ အရွယ်အစားသည် ၎င်းတိုက်ခိုက်သည့် Force (စွမ်းအား) နှင့် ကွာခြားသည်။ အဝေးပစ်ထိုးဖောက်လက်နက်များတွင် Impale Special Effect (ထိုးဖောက်ခြင်း အထူးအကျိုးသက်ရောက်မှု) ကြောင့် ဖြစ်ပေါ်လာသော အဟန့်အတားများကို ထည့်သွင်းစဉ်းစားသည့်အခါ ၎င်းတို့၏ တကယ့်အရွယ်အစားကို ဖော်ပြသည့် အထူးကော်လံတစ်ခု ရှိသည်။

### အခြေအနေဆိုင်ရာ မွမ်းမံမှုများ

လက်ချင်းထိတိုက်ခိုက်မှု (close combat) ကဲ့သို့ပင် အဝေးပစ်လက်နက် (Ranged Weapons) များကို သီးခြားအခြေအနေများတွင် အသုံးပြုသည့်အခါ အခြေအနေဆိုင်ရာ မွမ်းမံမှုများကို အသုံးပြုနိုင်သည်။ ဇာတ်ကောင်၏ Combat Skill (တိုက်ခိုက်ရေးကျွမ်းကျင်မှု) ကို မြင်းစီးနေစဉ် လှံပစ်ခြင်းကဲ့သို့သော ခက်ခဲသောအခြေအနေများတွင် လက်နက်ကို အသုံးပြုသည့်အခါ ကန့်သတ်ထားနိုင်သည်။

| ပစ်မှတ် လုံးဝဖုံးကွယ်ထားခြင်း | ကြီးမားသော |
| ပစ်မှတ် လုံးဝမြင်မရခြင်း | မဖြစ်နိုင်\*\* |
| ပစ်မှတ် လဲကျနေခြင်း | ကြောက်မက်ဖွယ် |
| တိုက်ခိုက်သူ လဲကျနေခြင်း\*\*\* | ကြီးမားသော |
| တိုက်ခိုက်သူသည် မတည်ငြိမ်သော မြေပြင်ပေါ်တွင်ရှိခြင်း | ခက်ခဲ |

_\* လေတိုက်နှုန်းအားနည်းသော ဒုံးပျံလက်နက် သို့မဟုတ် ခဲယမ်းများသည် လေတိုက်နှုန်းကြောင့် ထိခိုက်နိုင်သည်ဟု ယူဆပြီး ပုံမှန်ကျွမ်းကျင်မှုများအပေါ် လေတိုက်နှုန်းကြောင့် ဖြစ်ပေါ်သော ပုံမှန်ဒဏ်ကြေးကို ကျော်လွှားသည်။_

_\*\* ပစ်မှတ်၏ယေဘုယျတည်နေရာကို ဖော်ပြထားခြင်း သို့မဟုတ် သိရှိထားခြင်းမရှိပါက ကြီးမားသော (Herculean) အဖြစ် မှတ်ယူပါ။_

_\*\*\* ပြင်ဆင်ထားသောအနေအထားမှ crossbow သို့မဟုတ် သေနတ်ကို အသုံးပြုပါက ဒဏ်ကြေးကို ဖယ်ရှားနိုင်သည်။_

### ပစ်မှတ်ထားခြင်း

အဝေးပစ်လက်နက် (Ranged Weapon) ကို ပစ်မှတ်ထားရန် အချိန်ပိုသုံးစွဲခြင်းဖြင့် ဇာတ်ကောင်သည် ထိမှန်ရန် ၎င်း၏အခွင့်အရေးကို တိုးမြှင့်နိုင်သည်။ ပစ်မှတ်ထားခြင်းအတွက် လက်နက်ကို မြဲမြံစွာ ထိန်းထားပြီး အကောင်းဆုံး အခွင့်အရေးကို စောင့်ဆိုင်းရန်အတွက် တိုက်ခိုက်ရေးအချီ (Combat Round) တစ်ခုလုံး လိုအပ်သည်၊ ဥပမာ- လေတိုက်နှုန်း ခဏတာ ငြိမ်သက်ချိန်အထိ သို့မဟုတ် ပစ်မှတ်သည် အတားအဆီးနှစ်ခုကြား ရွေ့လျားသည်အထိ လေးမြှားကို မပစ်ဘဲ ထိန်းထားခြင်း။ ပစ်မှတ်ထားခြင်းဖြင့် ဇာတ်ကောင်သည် Range (အကွာအဝေး) သို့မဟုတ် Situational Modifier (အခြေအနေဆိုင်ရာ မွမ်းမံမှု) ၏ အခက်အခဲကို တစ်ဆင့်လျှော့ချနိုင်သည်။ နောက်ထပ်အချီများ ပစ်မှတ်ထားခြင်းသည် မည်သည့်အပိုအားသာချက်ကိုမျှ မပေးပါ။

### အလိုအလျောက်ပစ်ခတ်ခြင်း

အချို့သော သေနတ်များသည် ယခင်ပစ်ခတ်မှုမှ အညစ်အကြေးဓာတ်ငွေ့များ၊ နောက်ပြန်ကန်အား သို့မဟုတ် စွမ်းအင်ကို အသုံးပြု၍ နောက်ထပ်ပစ်ခတ်မှုကို အလိုအလျောက် chamber (သို့မဟုတ် charge) လုပ်သည်။ ပြန်လည်မဆွဲတင်ရဘဲ ဤစွမ်းရည်သည် လက်နက်အား မတူညီသော ပစ်ခတ်မှုနှုန်းများစွာကို ပေးသည်၊ ၎င်းတို့အားလုံးသည် အသုံးပြုသူ ပစ်ခတ်လိုသော ခဲယမ်းပမာဏပေါ် မူတည်သည်။

ဤပစ်ခတ်မှုပုံစံများကို Semi-Automatic (တစ်ချက်ချင်းအလိုအလျောက်)၊ Burst (ကျည်ဆန်အုပ်စုလိုက်) နှင့် Full-Automatic (အပြည့်အဝအလိုအလျောက်) ဟု သိကြသည်။ ဤစွမ်းရည်သည် သေနတ်အချို့ကို ကြောက်မက်ဖွယ်ကောင်းသော ထိခိုက်မှုများ ပေးနိုင်သော်လည်း နောက်ပြန်ကန်သောလက်နက်များဖြင့် တိကျမှုကို ထိန်းသိမ်းရန်မှာ ပြဿနာရှိသည်။ အောက်ပါစည်းမျဉ်းများသည် ကျည်ဆန်ပစ်ခတ်သော အလိုအလျောက်လက်နက်များအတွက် အဓိကအားဖြင့် သက်ဆိုင်သည်ဟု ယူဆပါ။

#### တစ်ချက်ချင်းအလိုအလျောက်

၎င်းသည် တကယ်တမ်းတွင် သေနတ်အများစု၏ ပုံသေပစ်ခတ်မှုနှုန်းဖြစ်ပြီး ခလုတ်ဆွဲလိုက်တိုင်း တစ်ချက်ပစ်ခတ်ပြီး နောက်ကျည်ကို တင်ပေးသည်။

#### ကျည်ဆန်အုပ်စုလိုက်

ကျည်ဆန်အုပ်စုလိုက်ပစ်ခတ်ခြင်းအတွက် ခလုတ်ဆွဲလိုက်တိုင်း ကျည်ဆန်တိုအချို့ကို ထုတ်လွှတ်သည်၊ ပုံမှန်အားဖြင့် လက်နက်အပေါ်မူတည်၍ သုံးချက်မှ ငါးချက်အထိသာ ရှိသည်။ ၎င်း၏မူလရည်ရွယ်ချက်မှာ ကန့်သတ်ထားသော ထိုးစစ်ပစ်ခတ်မှု (suppression fire) ကို ခွင့်ပြုရန်ဖြစ်ပြီး အလွန်အကျွံခဲယမ်းအသုံးပြုမှုကို ကန့်သတ်ရန်ဖြစ်သည်။ ၎င်းသည် သေနတ်ပစ်သူ၏ ထိမှန်ရန်အခွင့်အရေးကို မြှင့်တင်ရန် ရည်ရွယ်ခြင်းမဟုတ်ပါ၊ အကြောင်းမှာ မြန်ဆန်သော၊ ဆက်တိုက်နောက်ပြန်ကန်အားသည် လက်နက်ကို အဆက်မပြတ် လမ်းကြောင်းပြောင်းစေသောကြောင့်ဖြစ်သည်။ ကျည်ဆန်အုပ်စုလိုက်ပစ်ခတ်ခြင်းသည် တိုက်ခိုက်မှုလှိမ့်ခြင်း (attack roll) တွင် Hard (ခက်ခဲသော) ဒဏ်ကြေးအဆင့်ကို သက်ရောက်သည်။ တိုက်ခိုက်မှု အောင်မြင်ပါက သေနတ်ပစ်သူသည် ထိုအုပ်စုလိုက်ပစ်ခတ်မှုမှ ကျည်ဆန်မည်မျှ ပစ်မှတ်ကို ထိမှန်သည်ကို ကြည့်ရှုရန် အံစာတုံးတစ်ခုကို လှိမ့်သည်။ ထို့ကြောင့် သုံးချက်ပစ်ခတ်မှုသည် 1d3 ကို အသုံးပြုမည်ဖြစ်သည်။ ထိမှန်သော ကျည်ဆန်တစ်ခုစီအတွက် နေရာနှင့် ထိခိုက်မှုတို့ကို သီးခြားလှိမ့်ပါ။

#### အပြည့်အဝအလိုအလျောက်

ပစ်မှတ်များစွာကို ဖိအားပေးရန်အတွက် အပြည့်အဝထိုးစစ်ပစ်ခတ်မှု (suppressing fire) အတွက် ပုံမှန်အားဖြင့် အသုံးပြုသော်လည်း၊ ရန်သူအုပ်စုတစ်ခုထဲသို့ စုစည်းပစ်ခတ်ပါက အပြည့်အဝအလိုအလျောက်ပစ်ခတ်ခြင်းသည် အလွန်အမင်း ထိခိုက်စေနိုင်သည်။ သေနတ်ပစ်သူသည် ခဲယမ်းမည်မျှပစ်ခတ်မည်ကို ဆုံးဖြတ်သည် (လက်နက်၏ cyclic rate ကန့်သတ်ချက်အထိ) ၎င်းနောက် ပစ်ခတ်မှုလမ်းကြောင်းတွင် ပါဝင်သော ပစ်မှတ်အားလုံးသို့ ညီတူညီမျှ ခွဲဝေပေးသည်။ ကျန်ရှိသောကျည်ဆန်များကို ပစ်မှတ်ပြောင်းချိန်တွင် ဆုံးရှုံးသည်ဟု မှတ်ယူသည် (ဥပမာ- ပစ်မှတ် ၆ ခုကို ကျည် ၁၅ တောင့်ပစ်ခတ်ပါက တစ်ခုလျှင် ကျည်နှစ်တောင့်စီ ဖြစ်သည်)။ ပစ်မှတ်တစ်ခုစီအတွက် သီးခြားတိုက်ခိုက်မှုလှိမ့်ခြင်း (attack roll) ကို ပြုလုပ်သော်လည်း ဆက်တိုက်ပစ်ခတ်မှု၏ နောက်ပြန်ကန်အားကြောင့် ကြောက်မက်ဖွယ် (Formidable) အခက်အခဲအဆင့်ဖြင့် ပြုလုပ်သည်။ ထိမှန်သော ပစ်မှတ်များသည် ကျည်ဆန်အုပ်စုလိုက်ပစ်ခတ်မှု (Burst Fire) အတိုင်း ကျပန်းကျည်ဆန်အရေအတွက်ကို ခံစားရသည်။

ကျည်ဆန်အုပ်စုလိုက်ပစ်ခတ်ခြင်း သို့မဟုတ် အပြည့်အဝအလိုအလျောက်ပစ်ခတ်ခြင်း (full automatic spray) အတွင်း ထိမှန်သော ပထမဆုံးကျည်ဆန်တစ်ချက်သာ အထူးအကျိုးသက်ရောက်မှုများ (special effects) မှ အကျိုးကျေးဇူးရရှိနိုင်ကြောင်း သတိပြုပါ။ ထို့ကြောင့် အလိုအလျောက်ပုံစံဖြင့် ပစ်ခတ်နေစဉ် critical attack (ပြင်းထန်သောတိုက်ခိုက်မှု) ကို လှိမ့်ခြင်းသည် Maximize Damage (ထိခိုက်မှုကို အမြင့်ဆုံးမြှင့်တင်ခြင်း) ကို ရွေးချယ်ခွင့်ပြုသော်လည်း ပထမဆုံးကျည်ဆန်ဖြင့်သာ ဖြစ်သည်၊ ကျန်ရှိသောကျည်ဆန်များ အားလုံးထိမှန်ပါကပင် ဖြစ်သည်။

#### အလိုအလျောက်ပစ်ခတ်ခြင်းကို အဘယ်ကြောင့် ပြင်းထန်စွာ ဒဏ်ခတ်ထားသနည်း။

အလိုအလျောက်ပစ်ခတ်ခြင်းအတွက် စည်းမျဉ်းများသည် ခေတ်မီသေနတ်များ၏ ထင်ရှားသော တိကျမှုနှင့် နှိုင်းယှဉ်ပါက မသင့်လျော်စွာ ပြင်းထန်သည်ဟု ထင်ရနိုင်သည်။ ၎င်းအတွက် အကြောင်းပြချက်များစွာရှိပြီး အဓိကအချက်များမှာ စနစ်ကို ရိုးရှင်းစေခြင်းနှင့် ဂိမ်းချိန်ခွင်လျှာဖြစ်သည်။ ၎င်းတို့၏ဇာတ်ကောင်၏ တိကျမှုမရှိခြင်းကို ဝေဖန်သော ကစားသမားများကို စည်းမျဉ်းများကို မည်သည့်ပြင်ဆင်မှုမဆို ၎င်းတို့အပေါ်တွင်လည်း အသုံးချမည်ဖြစ်ပြီး လျင်မြန်စွာပစ်ခတ်မှု၏ အန္တရာယ်ကို ထည့်သွင်းစဉ်းစားပါက သတိပေးချက်တစ်ခု ဖြစ်သင့်သည်။

ဟုတ်ပါတယ်၊ ဤနောက်ပြန်ကန်ဒဏ်ကြေးအချို့ကို လျှော့ချနိုင်သည်၊ ဥပမာအားဖြင့် အလိုအလျောက်သေနတ်များကို ထောက်ရန် Steady Weapon (လက်နက်ကို တည်ငြိမ်အောင် ထိန်းခြင်း) ကို အသုံးပြုခြင်းဖြင့် ဖြစ်သည်။ သို့သော် အမှန်တကယ်တွင် သေနတ်ပစ်ခတ်မှုများသည် သေနတ်ပစ်ကွင်းတွင် ပစ်မှတ်ပစ်ခြင်းနှင့် အလွန်ကွာခြားသည်။ ကျည်ဆန်များပျံသန်းနေစဉ် ကာဗာယူရန် စိတ်ဓာတ်၊ ရွေ့လျားနေရန် လိုအပ်ခြင်းနှင့် ပစ်ခတ်မှုတိုင်းက ၎င်း၏သေနတ်ခေါင်းမီး သို့မဟုတ် အသံဖြင့် သေနတ်ပစ်သူကို ဖော်ထုတ်ခြင်းတို့သည် ပစ်မှတ်ထားရန် သို့မဟုတ် ထောက်ရန် အခွင့်အရေးကို မကြာခဏ တားဆီးထားသည်။ 'Spray and Pray' (ပစ်ပြီး ဆုတောင်း) ဟူသော စကားစုကို အကြောင်းပြချက်ခိုင်မာစွာဖြင့် ဖန်တီးခဲ့ခြင်းဖြစ်သည်။

### လက်နက်နှစ်ခုသုံး သေနတ်များ

ဇာတ်ကောင်များသည် ၎င်းတို့၏အလှည့်တွင် လက်နက်နှစ်ခုလုံးကို တစ်ပြိုင်နက်တည်း ပစ်ခတ်ခြင်းဖြင့် ပိုမိုမြန်ဆန်သော ပစ်ခတ်မှုနှုန်းကို ရရှိရန် လက်နက်နှစ်ခုသုံး သေနတ်များကို အသုံးပြုနိုင်သည်။ သို့သော် ၎င်းသည် တိကျမှုတွင် ဒဏ်ကြေးနှင့် လာသည်။ လက်တစ်ဖက်စီတွင် ပစ္စတိုတစ်လက်စီ ပစ်ခတ်ခြင်းသည် ပစ်မှတ်ကို ထိမှန်ရန် Hard (ခက်ခဲသော) ဒဏ်ကြေးခံစားရသည်။ လက်တစ်ဖက်စီတွင် carbine (ကာဗိုင်) သို့မဟုတ် SMG (Submachine Gun) ဖြင့် တစ်ပြိုင်နက်တည်း ပစ်ခတ်ရန် ကြိုးစားသူများသည် Formidable (ကြောက်မက်ဖွယ်) ဒဏ်ကြေးခံစားရသည်။

### လူအုပ်ထဲသို့ ပစ်ခတ်ခြင်း။

လက်ချင်းထိတိုက်ခိုက်မှု၏ ရွေ့လျားနေသော လူအုပ်ထဲသို့ ပစ်ခတ်ခြင်းသည် အန္တရာယ်ရှိသော လုပ်ရပ်တစ်ခုဖြစ်သည်၊ ထို့ကြောင့် မူလပစ်မှတ်အပြင် အခြားတစ်ဦးဦးကို မတော်တဆ ထိမှန်နိုင်သည့် အန္တရာယ် အမြဲရှိသည်။ လူအုပ် သို့မဟုတ် လက်ချင်းထိတိုက်ခိုက်မှု၏ အစွန်းရှိ သီးခြားပစ်မှတ်တစ်ခုကို ပစ်ခတ်သည့်အခါ တိုက်ခိုက်မှုသည် Hard (ခက်ခဲသော) အခက်အခဲအဆင့် (Difficulty Grade) ကို ခံစားရသည်။ ၎င်းကို ဖြတ်၍ ပစ်ခတ်ရန် ကြိုးစားပါက Formidable (ကြောက်မက်ဖွယ်) အထိ မြှင့်တင်သင့်သည်။ အန္တရာယ်ကို လျှော့ချရန်အတွက် ပစ်မှတ်ထားခြင်းကို ခွင့်ပြုထားသေးသည်။

သေနတ်ပစ်ကျွမ်းကျင်သူသည် အခက်အခဲဒဏ်ကြေးရှိသော်လည်း ၎င်း၏တိုက်ခိုက်မှုလှိမ့်ခြင်း (attack roll) ကို အောင်မြင်ပါက ၎င်းသည် ၎င်း၏ရည်ရွယ်ထားသော ပစ်မှတ်ကို တိကျစွာ ထိမှန်မည်ဖြစ်သည်။ သို့သော် သေနတ်ပစ်ကျွမ်းကျင်သူသည် လှိမ့်ခြင်းကို မအောင်မြင်သော်လည်း 'လူအုပ်ထဲသို့ ပစ်ခတ်ခြင်း' ဒဏ်ကြေးမရှိပါက အောင်မြင်နိုင်မည်ဆိုလျှင် ဘေးချင်းကပ်သားကောင်တစ်ဦးကို ထိမှန်မည် (၎င်းသည် ပုံမှန်အတိုင်း Evade (ရှောင်တိမ်း) သို့မဟုတ် Parry (ပိတ်ဆို့) နိုင်သည်။

ပစ်မှတ်တစ်ခုထက်ပို၍ ပစ်ခတ်လမ်းကြောင်းတွင် ရှိနေပါက ဂိမ်းမာစတာက သားကောင်ကို ကျပန်းဆုံးဖြတ်သင့်သည်။ တိုက်ခိုက်မှု၏ အစိတ်အပိုင်းအဖြစ် အနိုင်ရရှိသည့် မည်သည့်အထူးအကျိုးသက်ရောက်မှုများကိုမျှ မူလပစ်မှတ်အပေါ်သာ သက်ရောက်ပြီး မတော်တဆထိမှန်သော ဘေးနားရှိသူအပေါ် မသက်ရောက်ပါ။

### သေနတ်များဖြင့် ချော်လဲခြင်း

သေနတ်ပစ်ခတ်မှုကို ပုံမှန်အားဖြင့် ပြိုင်ဘက်ကျွမ်းကျင်မှုဖြင့် မခုခံသောကြောင့် သေနတ်ပစ်သူသည် ၎င်းတို့၏တိုက်ခိုက်မှုလှိမ့်ခြင်း (attack roll) ကို ချော်လဲသောအခါ အထူးပြင်ဆင်ချက်တစ်ခု ပြုလုပ်သင့်သည်။ ဤကိစ္စတွင် ရည်ရွယ်ထားသောပစ်မှတ် မည်သူမဆို ချော်လဲမှုနှင့် သက်ဆိုင်သော အထူးအကျိုးသက်ရောက်မှု (fumble specific special effect) ကို ရွေးချယ်ခွင့်ပြုပြီး ၎င်းကို သေနတ်ပစ်သူက ခံစားရမည်။ ချော်လဲမှုအတွက်သာရှိသော အထူးအကျိုးသက်ရောက်မှု - Weapon Malfunction (လက်နက်ချို့ယွင်းခြင်း) ကိုလည်း ကြည့်ပါ။

### ဒိုင်းများဖြင့် လက်နက်ပစ်ခတ်မှုကို ပိတ်ဆို့ခြင်း။

ယခင်က ဖော်ပြခဲ့သည့်အတိုင်း အသံထက်မြန်သော ကျည်ဆန် သို့မဟုတ် စွမ်းအင် Beam ကို တမင်တကာ ပိတ်ဆို့ရန်မှာ မဖြစ်နိုင်သလောက်ပင်။ သို့သော် ဒိုင်းများသည် အမှန်တကယ်တွင် ၎င်းတို့သည် သယ်ဆောင်နိုင်သော 'ကာဗာ' ဖြစ်သောကြောင့် အတိုင်းအတာတစ်ခုအထိ ကာကွယ်မှုကို ဆက်လက်ပေးသည်။ သေနတ်ပစ်ခတ်မှုတွင် ဒိုင်းကို အသုံးပြုရန်အတွက် အထက်ပါ [Close Combat](0005_Combat.md?id=passive-blocking) အပိုင်းမှ Passive Blocking (ကာကွယ်မှုဖြင့် ပိတ်ဆို့ခြင်း) စည်းမျဉ်းများကို အသုံးပြုပါ။

ပိုမိုခေတ်မီသော သေနတ်များ သို့မဟုတ် သိပ္ပံစိတ်ကူးယဉ်သေနတ်များနှင့် ပတ်သက်၍ ဂိမ်းမာစတာသည် ဒိုင်းကို Size (အရွယ်အစား) သတ်မှတ်ခြင်းအစား ၎င်းပြုလုပ်ထားသော ပစ္စည်းပေါ်မူတည်၍ Armor Point (သံချပ်ကာမှတ်) တန်ဖိုးပေးလိုပေမည်။ ဥပမာအားဖြင့် သစ်သားဒိုင်းကို Armor Points ၄ မှတ်၊ Kevlar နည်းဗျူဟာဒိုင်းကို ၈ AP၊ Ulysses 31 လက်ကောက်စွမ်းအားနယ်ပယ်ဒိုင်းကို ၁၅ AP အဖြစ် မှတ်ယူနိုင်သည်။

### တယ်လီစကုပ်ပစ်မှတ်ချိန်ကိရိယာများ

တယ်လီစကုပ်ပစ်မှတ်ချိန်ကိရိယာ (Telescopic Sights) သည် အသုံးပြုသူက ကောင်းစွာချိန်ရွယ်ရန် အချိန်ယူပါက (Aiming ကိုကြည့်ပါ) ပိုမိုဝေးကွာသော အကွာအဝေးများတွင် သေနတ်၏ တိကျမှုကို မြှင့်တင်ပေးသည်။ အခြေခံအကျဆုံး တယ်လီစကုပ်ပစ်မှတ်ချိန်ကိရိယာများမှာ ရိုးရိုးမှန်ဘီလူးချဲ့ထွင်ကိရိယာများသာ ဖြစ်သော်လည်း နည်းပညာ တိုးတက်လာသည်နှင့်အမျှ ၎င်းတို့သည် ပေါင်းစပ်အကွာအဝေးတိုင်းတာကိရိယာများ၊ ပစ်မှတ်လမ်းကြောင်းတွက်ချက်ကိရိယာများ၊ လေထုလှုပ်ရှားမှုအာရုံခံကိရိယာများ၊ လေထုအလင်းရောင်လျော်ကြေးပေးစနစ်များ၊ အနီအောက်ရောင်ခြည် သို့မဟုတ် အလင်းအားနည်းသည့် အားဖြည့်စနစ်များနှင့် အခြားသော ကိရိယာအများအပြားကို ပေါင်းစပ်ပါဝင်လာပြီး ၎င်း၏ ထိရောက်သော 'ထိမှန်မှု' အကွာအဝေးကို မြှင့်တင်ပေးသည်။

တယ်လီစကုပ်ပစ်မှတ်ချိန်ကိရိယာများသည် ၎င်း၏ရှုပ်ထွေးမှုအပေါ်မူတည်၍ ကျွမ်းကျင်မှုလှိမ့်ခြင်း (skill rolls) များကို တစ်ဆင့် သို့မဟုတ် ထို့ထက်ပို၍ လွယ်ကူစေသည် ([Modifying Skills](0003_Skills.md?id=modifying-skills) ကိုကြည့်ပါ)။ မည်သည့် sight ကမျှ ကျွမ်းကျင်မှုလှိမ့်ခြင်းကို Automatic success (အလိုအလျောက်အောင်မြင်မှု) ဖြစ်အောင် လုပ်ဆောင်နိုင်မည်မဟုတ်ပေ၊ ထို့ကြောင့် ရရှိနိုင်သည့် အကောင်းဆုံး Grade (အဆင့်) မှာ Very Easy (အလွန်လွယ်ကူ) ဖြစ်သည်။
