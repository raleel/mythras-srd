# د سټایل لارښود سرلیک (h1)

د اصلي څپرکي سرلیک لپاره h1 سرلیکونه (`# Heading Title`) وکاروئ. ټول hx سرلیکونه به په سایډبار کې د بک مارکونو په توګه څرګند شي.

که سرلیک h1 سرلیک نه وي، سایډبار به د نقل غلطی ولري چې پاڼه او بیا h2 'سرلیک' سرلیک ښیي.

---
## برخې سرلیکونه (h2)

د هرې نوې اصلي برخې لپاره h2 سرلیکونه (`## Section Heading`) وکاروئ چې د یوې ویشونکي (`---`) لخوا مخکې وي.

### فرعي برخې سرلیکونه چې h-کچه یې 1 زیاتیږي

د هرې نوې فرعي برخې لپاره h3 سرلیکونه (`### Subsection Heading`) وکاروئ، او د هرې نوې فرعي-فرعي برخې لپاره h4 سرلیکونه (`#### Sub-Subsection Heading`) وکاروئ.

### د فرعي برخې h-پرګریشن: 2 > 3 > 4

فرعي سرلیکونه په سایډبار کې ګډوډ کیږي که h-پرګریشن په سمه توګه اداره نه شي.

د سرلیک نښه باید د یو یا هیڅ شي په واسطه زیاته شي.

h2 چې وروسته h2 یا h3 وي ښه ده. h3 چې وروسته h3 یا h4 وي هم ښه ده.

### د فرعي برخې غلطۍ
h2 چې وروسته h4 وي ستونزې رامینځته کوي. سایډبار فرعي سرلیک د محتویاتو په جدول کې د سرلیک پورته ځای په ځای کوي.

### د h-کچو کمول

دا کومه ستونزه نه بریښي. د بیلګې په توګه، h4 ته h3 یا h2 ښه ده.

## د فارمیټ کولو کنوانسیونونه

لاندې سټایلونه په Markdown فارمیټ کې د دې _Imperative_ SRDs لپاره معمول دي.

### تور، ایټالیک او ویشونکي

- د **غټ** متن لپاره دوه ستوري (`**Bold**`) وکاروئ.
- د _ایټالیک_ متن لپاره انډرسکورونه (`_Italic_`) وکاروئ.
   - _نوټ: د ایټالیک لپاره د واحد ستوري کارولو څخه ډډه وکړئ!_
- د یو کرکټر 'تیښتې' لپاره بیک سلیشونه (`\_`) وکاروئ: \_ایټالیک نه\.
- د `code blocks` جوړولو لپاره زاویه شوي نرخونه (``` `code` ```) وکاروئ.
- د افقي ویشونکي کرښې داخلولو لپاره درې هایفنونه (`---`) وکاروئ.

### د غټ او ایټالیک لپاره کنوانسیونونه

هڅه وکړئ چې لاندې نمونو ته ودرېږئ...

#### غټ سرلیکونه

د جادو، اغیزو او نورو تشریح کولو لپاره غټ سرلیکونه وکاروئ.
- **جادو سرلیک:** تشریح.

#### ایټالیک سرلیکونه

د متغیراتو جادو او نورو 'فرعي لیستونو' لپاره ایټالیک سرلیکونه وکاروئ.
- **جادو:** تشریح.
  - **_جادو ډول-A:_** تشریح.
  - **_جادو ډول-B:_** تشریح.

#### د ایټالیک سرلیک حوالې

کله چې د _Mythras Imperative_ یا د _Mythras_ اصلي قواعدو په څیر کتاب ته حواله ورکوئ، ایټالیک سرلیکونه وکاروئ.

## جدولونه

جدولونو ته یو فرعي سرلیک ورکړئ ترڅو دوی په سایډبار کې د بک مارک په توګه څرګند شي، د خپل مورنۍ برخې څخه یوې کچې ته انډینټ شوي.

~د سرلیک قطار غټ متن جوړ کړئ.~
د سرلیک قطار غټ کول په حقیقت کې دا په عادي متن کې وړاندې کوي. اصلي CSS هغه قطار غټ کړی دی.

### د جدول متن تنظیمول

د محتویاتو تنظیمولو لپاره قطار 2 وکاروئ: `:--`، `:-:`، یا `--:`.
دا به ټوله ستونزه اغیزه وکړي.

#### د جدولونو اساسي ترتیب

| **ستون 1** | **ستون 2** |
| :-: | :-- |
| 1 | الفا |
| 2 | بیټا |
| 3 | ګاما |

### د متن تنظیمولو کنوانسیونونه

- د شمیرو یا ورته ارزښت لیستونو لپاره د مرکز تنظیمول وکاروئ.
- د متن جملو لپاره کیڼ تنظیمول وکاروئ.

#### د تنظیمولو مثال جدول

| **شمیره** | **تشریح** |
| :-: | :-- |
| 1 | تشریحي متن |
| 2 | تشریحي متن |
| 3 | تشریحي متن |

#### مثال: د اور شدت جدول

| **شدت** | **مثالونه** | **د اور اخیستلو وخت** | **زیان** |
| :-: | :-- | :-: | :-: |
| 1 | شمع | 1d4 | 1d2 |
| 2 | مشال | 1d3 | 1d4 |
| 3 | د کمپ اور | 1d2 | 1d6 |
| 4 | کوټه ډکوونکی اور | 1d2 | 2d6 |
| 5 | د اورغورځوونکي لاوا | فوري | 3d6 |

## لینکونه

په SRD کې د لینکونو لپاره ډیری کارونې شتون لري او دلته یوازې یو څو اصلي قضیې پوښل شوي دي.

### د لینکونو فارمیټ کول

د لینک فارمیټ کول د بریکټ شوي متن سره ترسره کیږي. د ښکاره لینک متن لپاره مربع بریکټونه وکاروئ، او د منزل URL لپاره منحني بریکټونه: `[Link Text](URL)`.

په دې اسنادو کې تر ټولو عام کارول شوی بهرنی لینک به [Mythras Encounter Generator](https://mythras.skoll.xyz) وي.
```
[Mythras Encounter Generator](https://mythras.skoll.xyz)
```

### بهرني vs داخلي لینکونه

بهرني لینکونه د انټرنیټ له لارې نورو پاڼو ته ځي او یو `http://example.com` URL ته به اړتیا ولري او، په ډیفالټ، دوی به تل په نوي براوزر ټب کې خلاص شي.

د سند لوستلو پرمهال د ډیری ټبونو له ګډوډولو څخه مخنیوي لپاره، داخلي لینکونه او بک مارکونه د نسبي URLs سره جوړیږي چې د GitHub ریپوزټوري کې ښکاره شوي مارکډاون فایلونو ته اشاره کوي. د نوي ټب پرځای، دا داخلي لینکونه به په چټکۍ سره وصل شوي برخې ته سکرول کړي.

#### د داخلي لینک مثالونه

- **د څپرکي لینکونه:** دا د اصلي (h1) برخې پیل ته اشاره کوي. د بیلګې په توګه، د [Combat](0005_Combat.md) څپرکی.

  - د لینک فارمیټ: `[Combat](0005_Combat.md)`

- **د فرعي برخې لینکونه:** دا په یوه څپرکي کې یو سرلیک ته اشاره کوي چې پورته سند ته اشاره کوي مګر د `?id=` وروسته سمدستي د منزل سرلیک سره. د بیلګې په توګه [Leaping Attacks](0005_Combat.md?id=leaping-attacks)

  - د لینک فارمیټ: `[Leaping Attacks](0005_Combat.md?id=leaping-attacks)`

#### بهرني لینکونه فرعي برخو ته

کیدای شي یو بهرني سند ته د یوې پیژندل شوې برخې حواله ورکولو فرصت وي، د بیلګې په توګه، په بل SRD کې یو ځانګړی جدول. فارمیټ کول د فرعي برخې لینکونو په څیر دي، یوازې د بهرني HTTP URL په کارولو سره.

د مثال په توګه، دلته د [Classic Fantasy Imperative SRD](https://cfi-srd.mythras.net/) څخه [د امپراطوري څخه میټریک ته د بدلولو جدول](https://cfi-srd.mythras.net/#/Appendix_B_Conversion_Tables?id=imperial-weights-and-measures-to-metric-conversion) دی.

```
[Imperial to Metric conversions table](https://cfi-srd.mythras.net/#/Appendix_B_Conversion_Tables?id=imperial-weights-and-measures-to-metric-conversion)
```

## د جادو او ځواکونو لیستونه

د موجودو جادوګانو، ځواکونو او نورو لیستونو لپاره یو 'پاک' برخه د h2 سرلیک سره پیل کړئ؛ بیا د هر لیست توکي لپاره h3.

جادوګانې او ځواکونه معمولا د جادو ډول (عوامي جادو، تییزم او نور) له مخې په یو واحد فلیټ لیست کې وړاندې کیږي مګر که لیست خپلو فرعي برخو ته اړتیا ولري، لکه د ښوونځي یا فرقې تخصصونه، د سرلیک کچه په مناسبه توګه لوړه کړئ. د بیلګې په توګه، د تییزم معجزو لیست لپاره h2، د فرقې: میتراس لپاره h3، او د هر میتراس فرقې معجزې لپاره h4.

په ځانګړې توګه د جادوګانو په حالت کې، مګر په هر تشریح کې چې د کلیدي کلمو ځانګړتیاوې کاروي، دوی په ایټالیک متن کې لیست کړئ.

تشریحي متن او ځانګړي قواعد عادي متن کاروي.

### د جادو لیست فارمیټ کول

```
## h2 List name

### List item

_Attributes_

Description text.
```

## د وسایطو احصایې بلاکونه
د لیستونو لپاره یو 'پاک' برخه د h2 سرلیک سره پیل کړئ، بیا د کټګوریو او لیست توکو لپاره د معمول په څیر h-کچه زیاته کړئ.

هر لیست شوی مثال د احصایو او تشریحاتو په توګه د بلټ ټکي لیست په توګه د غټو ټکو سرلیکونو سره وړاندې کوي.

### د وسایطو د احصایې بلاک فارمیټ کول

```
## h2 List name

### Category

#### List Item

- **Stat 1:**  [Value]

- **Stat 2:**  [Value]

- **Description:** Descriptive text.
```

## د مخلوق احصایې بلاکونه
د لیستونو لپاره یو 'پاک' برخه د h2 سرلیک سره پیل کړئ. د مخلوق احصایې بلاکونه جدولونه کاروي او د فارمیټ کولو او ځای اړتیاو له مخې ډیر پیچلي دي. د وضاحت لپاره، له فرعي برخو څخه ډډه وکړئ او د لیست توکو ترمنځ ویشونکي اضافه کړئ.

هر لیست شوی مثال باید لاندې ولري...

- **د مخلوق سرلیک:** یو ویشونکی چې وروسته د مخلوق نوم لپاره h3 سرلیک.

- **MEG ننوتنه:** د [Mythras Encounter Generator](https://mythras.skoll.xyz/) کې مشخص شوي مخلوق ته بهرنی لینک.

- **اصلي جزئیات:** د مخلوق نوم/ډول، ځانګړتیاوې، مهارتونه او هر ډول ځانګړي تشریحات یا د کرکټر یادښتونه لکه څنګه چې اړتیا وي.

- **د وسلو جزئیات:** لنډ شوي تشریحات د وسلو نوم او لاندې ځانګړتیاو لپاره د ستونونو سره: اندازه (دلته ځواک شامل کړئ)، رسېدل (دلته رینج شامل کړئ)، زیان، AP/HP، او هر ډول ځانګړي قواعد.

- **د زیان موقعیتونه او زغره:** د موقعیت جدولونه به د مختلف مخلوق اناتومي پراساس توپیر ولري مګر باید حتی د دودیزو انسانانو لپاره د زغرې او زیان تعقیبولو لپاره شامل شي.

### د مخلوق د احصایې بلاک فارمیټ کول

```
## Bestiary/List Name

Descriptive text. 

---

[MEG Entry](https://mythras.skoll.xyz/enemy_template/8626/)

| Creature Name | Example Creature  |
| :-- | :-- |
| Key Attributes | STR 11, CON 14, SIZ 11, DEX 11, INT 11, POW 11, CHA 7 |
| Other notes | Values and details | 
```

## ټولنیز اصلاحات

### د ټایپوګرافیکي غلطۍ او ګډوډي

د لیکوالانو متن 'پاکولو' لپاره سمونونه باید لږ تر لږه وساتل شي مګر ټایپوګرافیکي غلطۍ او نورې تېروتنې باید یوازې د کومې نښې پرته په لیکه کې بدلې شي.

### د اصلاحاتو معلوماتي پینلونه

چیرې چې یو ورکړل شوی قاعده خورا مبهم، نیمګړې، یا په ساده ډول غلطه وي، یو د سقوط وړ ټولنیز اصلاح یادداشت اضافه کړئ.

مسله په ایټالیک کې یوازې د مسلې سرلیک لپاره غټ ایټالیک په کارولو سره تشریح کړئ - _د h(x) سرلیکونه نه!_ - ترڅو ګډوډي سایډبار ننوتنې رامینځته نه کړي.

که بدلیدونکی متن اضافه شي، دا په غټ ایټالیک کې په دوه نقل قولونو کې ځای په ځای کړئ، چې د "_ټولنیز اصلاح:_" لخوا په عادي وزن ایټالیک کې مخکې کیږي.

اضافي قواعد، که اړتیا وي، باید په عادي وزن متن کې وړاندې شي.

## متفرقه اخطارونه

### خالي کرښې / د کرښې راستنېدل

د مختلف ډوله ننوتونو ترمنځ خالي کرښه پریږدئ. سرلیکونه، ویشونکي، جدولونه او نور. دا به په لوستلو لید کې نه ښکاري مګر د دوی نشتوالی کولی شي د فارمیټ کولو ستونزې رامینځته کړي.

### غټ یا ایټالیک څو-پراګراف متن

هر پراګراف باید د غټ یا ایټالیک متن په توګه په نښه شي. د پراګرافونو ترمنځ نوې کرښه به فارمیټ مات کړي چې د یتیم فارمیټ کولو نښې به پریږدي.

### نړیوال املاوې

رجعتي او د غندلو وړ قدرتونو پریکړه کړې چې ټول برتانوي املاوې د هدف له مخې ضعیف امریکایي نسخو په ګټه لرې کړي. د برتانوي -OUR کلمو په لټه کې اوسئ: _armour_، _favour_، او نور.
